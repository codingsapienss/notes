## 📖 Documentation Structure

Documentation follows a **hybrid approach** of manual category definitions and auto-generated sidebars.

### A. Sidebar Hierarchy (`sidebars.js`)

`sidebars.js` defines high-level categories and delegates content discovery to Docusaurus automatically:

```js
// Example from sidebars.js
{
  type: 'category',
  label: 'C++ Fundamentals',
  items: [{type: 'autogenerated', dirName: 'dsa/basics/cpp-fundamentals'}],
}
```

### B. Folder Metadata (`_category_.json`)

Each folder contains a `_category_.json` to control how it appears in the sidebar (its display name and order):

```json
{
  "label": "TypeScript Fundamentals",
  "position": 1
}
```

### C. Document Metadata (Frontmatter)

Every `.md` file begins with a **YAML frontmatter** block to define its sidebar identity:

```markdown
---
sidebar_label: 'Binary Fundamentals'  # Name shown in the sidebar
sidebar_position: 1                   # Order within the folder
---

# Page Title
Content goes here...
```

### D. URL Routing

Docusaurus maps file paths to URLs automatically:

```
docs/web-dev/react/intro.md  →  yoursite.com/notes/docs/web-dev/react/intro
```

## 🛠️ Local Development

### Prerequisites
- Node.js `>= 20.0`
- npm

### 1. Install Dependencies

```bash
npm install
```

### 2. Start Development Server

```bash
npm run start
```

The site will be available at `http://localhost:3000/notes/`.

### 3. Build Production Site

```bash
npm run build
```

The output will be generated in the `build/` directory.

### 4. Serve Built Site Locally

```bash
npm run serve
```

## 🚢 Deployment

The site is hosted on **GitHub Pages** under the `gh-pages` branch.

| Branch | Purpose |
|---|---|
| `main` | Source code — where you write and commit |
| `gh-pages` | Built static files — auto-managed by Docusaurus deploy |

### Site Configuration (`docusaurus.config.js`)

```js
const config = {
  url: 'https://codingsapienss.github.io',
  baseUrl: '/notes/',
  organizationName: 'codingsapienss', // GitHub User/Org name
  projectName: 'notes',               // Repository name
  trailingSlash: false,
};
```

### Deploy Command (PowerShell)

```powershell
# Set identity environment variables (required for this setup)
$env:GIT_USER="codingsapienss"
$env:GIT_COMMITTER_NAME="codingsapienss"
$env:GIT_COMMITTER_EMAIL="contactprsant@gmail.com"
$env:GIT_AUTHOR_NAME="codingsapienss"
$env:GIT_AUTHOR_EMAIL="contactprsant@gmail.com"

# Run the deploy script (builds + pushes to gh-pages)
npm run deploy
```

This command:
1. Runs `docusaurus build` to generate the static site
2. Pushes the `build/` output to the `gh-pages` branch on GitHub
3. GitHub Pages automatically serves the content from `gh-pages`

## ✏️ Contributing / Adding Content

### Adding a New DSA Topic

1. Create a folder under `docs/dsa/` (e.g., `docs/dsa/sorting/`)
2. Add a `_category_.json` to name the section:
   ```json
   { "label": "Sorting Algorithms", "position": 2 }
   ```
3. Add your `.md` files with proper frontmatter:
   ```markdown
   ---
   sidebar_label: 'Bubble Sort'
   sidebar_position: 1
   ---
   ```
4. If needed, add a new category entry in `sidebars.js` pointing to the new folder.

### Adding a New Web Dev Section

1. Create a folder under `docs/web-dev/` (e.g., `docs/web-dev/next-js/`)
2. The `autogenerated` directive in `sidebars.js` will automatically pick it up.
3. Add a `_category_.json` for display control.

## 💡 Best Practices

> **Pro Tip:** Always start documentation directly under `docs/` (e.g., `docs/dsa/...`) to keep URLs clean. Avoid nesting a `docs/` folder inside `docs/`.

1. **Initialize new projects:** `npx create-docusaurus@latest my-website classic --typescript`
2. **Clean up:** Remove default tutorial folders and update `docusaurus.config.js`
3. **Structure:** Create folders under `docs/` and add `_category_.json` to each
4. **Sidebar:** Use `autogenerated` in `sidebars.js` for content-heavy folders
5. **Deploy:** Link to a GitHub repo and use `npm run deploy` with your credentials

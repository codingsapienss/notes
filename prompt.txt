  3. How Documentation is Structured
  You use a hybrid approach of Manual and Autogenerated sidebars.

  A. The Sidebar Hierarchy (sidebars.js)
  Your sidebars.js defines high-level categories (like "DSA Basics", "DevOps") and then tells Docusaurus to automatically find the files within
  specific folders.

    1 // Example from your sidebars.js
    2 {
    3   type: 'category',
    4   label: 'C++ Fundamentals',
    5   items: [{type: 'autogenerated', dirName: 'dsa/basics/cpp-fundamentals'}],
    6 }

  B. Folder Metadata (_category_.json)
  To control how a folder appears in the sidebar (its name and position) without editing sidebars.js, you place a _category_.json file
  inside the folder:

    1 {
    2   "label": "TypeScript Fundamentals",
    3   "position": 1
    4 }

  C. Document Metadata (Frontmatter)
  Every .md file starts with a "Frontmatter" block to define its identity:

    1 ---
    2 sidebar_label: 'Binary Fundamentals'  # Name shown in the sidebar
    3 sidebar_position: 1                   # Order within the folder
    4 ---
    5
    6 # Page Title
    7 Content goes here...

  4. How the Pages are Built
    1. Markdown to HTML: Docusaurus reads your .md and .mdx files. It converts Markdown into optimized HTML and React components.
    2. Routing: The file path determines the URL.
        * docs/web-dev/react/intro.md becomes yoursite.com/notes/docs/web-dev/react/intro.
    3. Homepage: The landing page is a custom React component located at src/pages/index.js. It uses Docusaurus's <Layout> and <Link>
       components to provide a consistent look.

  6. Backdating Commits
  When you want to create commits with historical dates (e.g., to reflect actual writing progress over time):

    python create_backdated_commits.py --start-date YYYY-MM-DD --end-date YYYY-MM-DD --min-per-day N --max-per-day M

  Rules:
  - This script only commits REAL file changes already present in the working tree. It does NOT create empty/fake commits.
  - Each commit corresponds to an actual staged file (new, modified, or untracked).
  - Commit timestamps are randomized between 09:00 and 22:00 IST for natural look.
  - You specify the date range and commit frequency per day.
  - Example: python create_backdated_commits.py --start-date 2026-06-26 --end-date 2026-07-20 --min-per-day 5 --max-per-day 8

  7. Helper Scripts
  - scripts/fix_headings.py: Normalizes H1/H2 structure in docs/dsa/basics/hashing and docs/web-dev/javascript/Prototypes & Inheritance. Run only if headings look broken.
  - scripts/fix_devops_frontmatter.py: Auto-generates _category_.json for every devops subfolder and injects clean sidebar_label/sidebar_position frontmatter into all devops markdown files.
  - scripts/fix_frontmatter_v2.py: Strips malformed frontmatter from markdown files and replaces it with clean sidebar_label + sidebar_position. Useful when bulk-importing docs.

  8. Best Practices for New Projects
  To use this setup for a new project:
    1. Initialize: npx create-docusaurus@latest my-website classic --typescript.
    2. Clean Up: Remove the default tutorial folders and update docusaurus.config.js.
    3. Structure: Create folders inside docs/ and add _category_.json to each.
    4. Sidebar: Set up sidebars.js to use autogenerated for your main content folders.
    5. Deploy: Link to a GitHub repo and use npm run deploy with your credentials.

  > Pro Tip: Your current project has some duplication (a docs folder inside the docs folder). For future projects, ensure all
  documentation starts directly under the root docs/ directory to keep URLs clean (e.g., docs/DSA/... instead of docs/docs/DSA/...).

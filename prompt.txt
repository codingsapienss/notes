  📚 Knowledge Base Architecture Guide

  1. Project Overview & Tech Stack
   * Framework: Docusaurus 3 (https://docusaurus.io/) (Classic Preset).
   * Purpose: A structured handbook for DSA (C++) and Web Development.
   * Key Features: Automatic sidebar generation, MDX support (React in Markdown), and GitHub Pages deployment.

  2. Code & Folder Structure
  The project follows the standard Docusaurus layout with a few custom organizational choices:

    1 .
    2 ├── docs/                   # 📄 The "Heart" of the project (Markdown files)
    3 │   ├── DSA/                # Data Structures & Algorithms content
    4 │   ├── WebDev/             # Web Development content (TypeScript, React)
    5 │   └── _category_.json     # Metadata for folder-based sidebar items
    6 ├── src/                    # 🎨 React components and custom styling
    7 │   ├── css/                # Global styles (custom.css)
    8 │   └── pages/              # Custom React pages (Home page: index.js)

   10 ├── docusaurus.config.js    # ⚙️ Main site configuration
   11 ├── sidebars.js             # 🗺️ Navigation structure definition
   12 └── package.json            # 📦 Dependencies and scripts

  3. How Documentation is Structured
  You use a hybrid approach of Manual and Autogenerated sidebars.

  A. The Sidebar Hierarchy (sidebars.js)
  Your sidebars.js defines high-level categories (like "DSA Basics") and then tells Docusaurus to automatically find the files within
  specific folders.

   1 // Example from your sidebars.js
   2 {
   3   type: 'category',
   4   label: 'C++ Fundamentals',
   5   items: [{type: 'autogenerated', dirName: 'DSA/Basics/C++ Fundamentals'}],
   6 }

  B. Folder Metadata (_category_.json)
  To control how a folder appears in the sidebar (its name and position) without editing sidebars.js, you place a _category_.json file
  inside the folder:

   1 {
   2   "label": "TypeScript Fundamentals",
   3   "position": 1
   4 }

  C. Document Metadata (Frontmatter)
  Every .md file starts with a "Frontmatter" block to define its identity:

   1 ---
   2 sidebar_label: 'Binary Fundamentals'  # Name shown in the sidebar
   3 sidebar_position: 1                   # Order within the folder
   4 ---
   5
   6 # Page Title
   7 Content goes here...

  ---

  4. How the Pages are Built
   1. Markdown to HTML: Docusaurus reads your .md and .mdx files. It converts Markdown into optimized HTML and React components.
   2. Routing: The file path determines the URL.
       * docs/WebDev/React/intro.md becomes yoursite.com/notes/docs/WebDev/React/intro.
   3. Homepage: The landing page is a custom React component located at src/pages/index.js. It uses Docusaurus's <Layout> and <Link>
      components to provide a consistent look.

  5. Deployment Workflow
  The site is hosted on GitHub Pages under the gh-pages branch.

  The Deployment Branch
   * Source Branch: main (where you write code).
   * Deployment Branch: gh-pages (where the built static files live).

  Configuration (docusaurus.config.js)

   1 const config = {
   2   url: 'https://codingsapienss.github.io',
   3   baseUrl: '/notes/',
   4   organizationName: 'codingsapienss', // GitHub User/Org name
   5   projectName: 'notes',               // Repository name
   6   trailingSlash: false,
   7 };

  Deployment Command
  To deploy, you use the following command which builds the site and pushes it to GitHub:

   1 # Set identity env vars (required for your specific setup)
   2 $env:GIT_USER="codingsapienss"
   3 $env:GIT_COMMITTER_NAME="codingsapienss"
   4 $env:GIT_COMMITTER_EMAIL="contactprsant@gmail.com"
   5 $env:GIT_AUTHOR_NAME="codingsapienss"
   6 $env:GIT_AUTHOR_EMAIL="contactprsant@gmail.com"
   7
   8 # Run the deploy script
   9 npm run deploy

  6. Best Practices for New Projects
  To use this setup for a new project:
   1. Initialize: npx create-docusaurus@latest my-website classic --typescript.
   2. Clean Up: Remove the default tutorial folders and update docusaurus.config.js.
   3. Structure: Create folders inside docs/ and add _category_.json to each.
   4. Sidebar: Set up sidebars.js to use autogenerated for your main content folders.
   5. Deploy: Link to a GitHub repo and use npm run deploy with your credentials.

  > Pro Tip: Your current project has some duplication (a docs folder inside the docs folder). For future projects, ensure all
  documentation starts directly under the root docs/ directory to keep URLs clean (e.g., docs/DSA/... instead of docs/docs/DSA/...).

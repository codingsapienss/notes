# AGENTS.md

## What this is
Docusaurus 3 documentation site (Classic Preset). Not a code app — content lives in `docs/`.

## Prereqs
- Node.js >= 20.0
- npm

## Commands
```bash
npm install        # first time or after lockfile changes
npm run start      # dev server at http://localhost:3000/notes/
npm run build      # output to build/
npm run serve      # preview built site locally
npm run deploy     # pushes build/ to gh-pages branch
```

Deploy requires these env vars (PowerShell example in README.md:178):
`GIT_USER`, `GIT_COMMITTER_NAME`, `GIT_COMMITTER_EMAIL`, `GIT_AUTHOR_NAME`, `GIT_AUTHOR_EMAIL`

## Content rules (agents miss these often)
- Every `.md` file needs YAML frontmatter with at least `sidebar_label` and `sidebar_position`
- Each docs folder needs a `_category_.json` with `label` and `position`
- `sidebars.js` defines the real hierarchy — do NOT add new top-level categories here without updating it
- Web Dev uses `autogenerated`; DSA uses explicit categories with nested `autogenerated` dirs
- Folders containing spaces in their names (e.g., `Prototypes & Inheritance`) must match `sidebars.js` `dirName` exactly

## Do not add
- Tests, CI, lint, or typecheck config — none exist and none are needed
- Code outside `docs/`, `src/css/`, `src/pages/`, `static/`

## One-off utilities
- `scripts/fix_headings.py` — normalizes H1/H2 structure in specific directories; only run if headings are broken

Typing APIs (fetch, Axios, AxiosResponse & Error Handling)

In real-world applications, one of the most common tasks is:

Communicating With APIs

Examples:

  • Fetching Users
  • Loading Products
  • Authentication
  • Payments
  • Dashboard Data

TypeScript helps us make API communication:

Type Safe
Predictable
Easier To Maintain

Why API Typing Matters​

Consider:

const response = await fetch("/users");

const data = await response.json();

console.log(data.name);

Problem:

TypeScript has no idea
what data looks like.

Type becomes:

any;

or

unknown;

(depending on configuration)

This means:

data.xyz.abc.def;

may compile.

But fail at runtime.

The Goal​

Instead of:

Unknown Data

we want:

Strongly Typed Data

Example

{
  id: number;
  name: string;
  email: string;
}

TypeScript can then:

Validate Fields
Provide Autocomplete
Catch Errors Early

Creating API Models​

Suppose API returns:

{
  "id": 1,
  "name": "Prashant",
  "email": "test@gmail.com"
}

Create a Type

interface User {
  id: number;

  name: string;

  email: string;
}

This becomes:

Source Of Truth

for API data.

Typing fetch()​

Basic Fetch

const response = await fetch("/users/1");

const data = await response.json();

Problem

data;

has weak typing.

Type Assertion Approach​

const response =
  await fetch("/users/1");

const data =
  await response.json()
  as User;

Now:

data.name;

works.

data.email;

works.

data.salary;

Error.

Because:

salary;

does not exist in:

User;

Complete Example​

interface User {
  id: number;

  name: string;

  email: string;
}
async function getUser() {

  const response =
    await fetch("/users/1");

  const user =
    await response.json()
    as User;

  return user;
}

Usage

const user = await getUser();

console.log(user.name);

TypeScript knows:

user;

is:

User;

Typing Array Responses​

Suppose API returns:

[
  {
    "id": 1,
    "name": "A"
  },
  {
    "id": 2,
    "name": "B"
  }
]

Model

interface User {
  id: number;

  name: string;
}

Fetch

const users =
  await response.json()
  as User[];

Type

User[]

Usage

users[0].name;

Autocomplete available.

Generic API Response Model​

Very common.

Backend Response

{
  "success": true,
  "data": {
    "id": 1,
    "name": "Prashant"
  }
}

Create Generic Type

interface ApiResponse<T> {
  success: boolean;

  data: T;
}

User Model

interface User {
  id: number;

  name: string;
}

Usage

type UserResponse = ApiResponse<User>;

Generated Type

{
  success: boolean;

  data: User;
}

Very common in enterprise applications.

Introduction to Axios​

Axios is one of the most popular HTTP libraries.

Installation

npm install axios

Simple Request

import axios from "axios";

const response = await axios.get("/users");

Question:

How do we type the response?

AxiosResponse​

Axios provides:

AxiosResponse<T>;

Syntax

AxiosResponse<User>;

Meaning:

Response Data
contains User

Example​

User Model

interface User {
  id: number;

  name: string;
}

Request

const response = await axios.get<User>("/users/1");

Type of:

response.data;

becomes:

User;

Usage

console.log(response.data.name);

Autocomplete available.

How Axios Generics Work​

Request

axios.get<User>("/users/1");

Internally

AxiosResponse<User>;

Result

response.data;

↓

User;

Array Response Example​

Model

interface User {
  id: number;

  name: string;
}

Request

const response = await axios.get<User[]>("/users");

Type

response.data;

↓

User[]

Usage

response.data[0].name;

Generic API Service​

Common enterprise pattern.

Response Model

interface ApiResponse<T> {
  success: boolean;

  data: T;
}

User Model

interface User {
  id: number;

  name: string;
}

Request

const response = await axios.get<ApiResponse<User>>("/users/1");

Type

response.data.data;

↓

User;

Very common pattern.

Typing POST Requests​

Model

interface CreateUser {
  name: string;

  email: string;
}

Request

await axios.post("/users", {
  name: "Prashant",
  email: "test@gmail.com",
});

Better

const payload: CreateUser = {
  name: "Prashant",

  email: "test@gmail.com",
};
await axios.post("/users", payload);

Now payload is validated.

API Error Handling​

One of the most important topics.

Bad

const response = await axios.get("/users");

If request fails:

Application Crashes

Try-Catch​

try {
  const response = await axios.get("/users");
} catch (error) {
  console.log(error);
}

Problem​

Type of:

error;

is:

unknown;

Why?

Because:

Anything can be thrown

Narrowing Error Type​

try {
} catch (error) {
  if (error instanceof Error) {
    console.log(error.message);
  }
}

Now:

error.message;

is safe.

Axios Error Handling​

Axios provides:

AxiosError;

Example

import { AxiosError } from "axios";
try {
  await axios.get("/users");
} catch (error) {
  if (error instanceof AxiosError) {
    console.log(error.response);
    console.log(error.message);
  }
}

Very common in production.

Real World API Wrapper​

import axios from "axios";
interface ApiResponse<T> {
  success: boolean;

  data: T;
}
interface User {
  id: number;

  name: string;
}
async function getUser(id: number) {
  const response = await axios.get<ApiResponse<User>>(`/users/${id}`);

  return response.data.data;
}

Usage

const user = await getUser(1);

Type

User;

Fully type-safe.

Common Mistakes​

Using any​

Bad

const data: any;

Prefer

User;

or

ApiResponse<User>;

Ignoring Error Types​

Bad

catch(error) {

  console.log(
    error.message
  );
}

Error is:

unknown;

Use

instanceof Error

Not Creating API Models​

Bad

response.data.name;

with no types.

Always create:

interface User

Interview Questions​

Q1​

Why type API responses?

Answer​

Type Safety
Autocomplete
Compile-Time Validation

Q2​

What is AxiosResponse?

Answer​

A generic response type
provided by Axios.

Q3​

How do you type an API response?

Answer​

axios.get<User>();

Q4​

Why is catch(error) typed as unknown?

Answer​

Anything can be thrown.

Q5​

What is a common generic API pattern?

Answer​

interface ApiResponse<T>

Cheat Sheet​

interface User {}
await response.json()
as User
axios.get<User>();
AxiosResponse<User>;
ApiResponse<T>;
User[]
error instanceof Error;
error instanceof AxiosError;

Key Takeaways​

  • Always create models for API data.
  • fetch() responses should be typed.
  • Axios provides generic typing through AxiosResponse<T>.
  • axios.get<User>() creates strongly typed responses.
  • Generic API wrappers improve reusability.
  • ApiResponse<T> is a very common enterprise pattern.
  • Error handling should use proper type narrowing.
  • catch(error) is typed as unknown.
  • AxiosError helps handle HTTP-specific failures.
  • Strongly typed APIs significantly reduce runtime bugs.

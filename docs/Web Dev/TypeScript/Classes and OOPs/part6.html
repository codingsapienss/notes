Abstract Classes

In previous chapters, we learned:

  • Inheritance (extends)
  • Interfaces (implements)

However, there is a middle ground between:

Interface

and

Normal Class

Sometimes we want:

  • Some common implementation
  • Some methods that MUST be implemented by child classes

This is where:

abstract class

becomes useful.

Abstract Classes are one of the most common OOP interview topics.

Why Do We Need Abstract Classes?​

Suppose we are building an Animal System.

Every animal can:

Eat
Sleep

But every animal makes a different sound.

Dog  → Woof
Cat  → Meow
Lion → Roar

Creating a normal parent class:

class Animal {
  eat() {
    console.log("Eating...");
  }

  makeSound() {
    // ???
  }
}

Problem:

Animal itself does not know
how to make a sound.

We need:

Force child classes
to provide implementation.

This is exactly why:

abstract;

exists.

What is an Abstract Class?​

An Abstract Class is:

A class that cannot
be instantiated directly.

It acts as:

A Base Blueprint

for child classes.

Syntax

abstract class Animal {}

Important Rule​

You CANNOT create an object of an abstract class.

Example

abstract class Animal {}

Wrong

const animal = new Animal();

Error

Cannot create an instance
of an abstract class.

Why?

Because:

Animal is incomplete.

Only child classes can be instantiated.

Abstract Methods​

The biggest feature of abstract classes.

An Abstract Method:

Has Declaration
But No Implementation

Syntax

abstract methodName(): void;

Example

abstract class Animal {
  abstract makeSound(): void;
}

Notice:

abstract makeSound(): void;

No body.

No implementation.

Why?​

Because:

Every child class
will implement it differently.

Implementing Abstract Methods​

Abstract Class

abstract class Animal {
  abstract makeSound(): void;
}

Child Class

class Dog extends Animal {
  makeSound() {
    console.log("Woof!");
  }
}

Usage

const dog = new Dog();

dog.makeSound();

Output

Woof!

What Happens If Child Class Doesn't Implement?​

Example

abstract class Animal {
  abstract makeSound(): void;
}

Wrong

class Dog extends Animal {}

Error

Non-abstract class
'Dog' does not implement
inherited abstract member.

Because:

Contract Broken

Abstract Class with Normal Methods​

Abstract classes can contain:

Normal Methods

and

Abstract Methods

together.

Example

abstract class Animal {
  eat() {
    console.log("Eating...");
  }

  abstract makeSound(): void;
}

Child

class Dog extends Animal {
  makeSound() {
    console.log("Woof!");
  }
}

Usage

const dog = new Dog();

dog.eat();

dog.makeSound();

Output

Eating...
Woof!

Notice:

eat()

was inherited.

makeSound()

was implemented.

Real World Mental Model​

Think of:

Vehicle

Every vehicle:

Start Engine
Stop Engine

But:

Car
Bike
Truck

all start differently.

Abstract Class

abstract class Vehicle {
  abstract start(): void;

  stop() {
    console.log("Stopped");
  }
}

Car

class Car extends Vehicle {
  start() {
    console.log("Car Started");
  }
}

Usage

const car = new Car();

car.start();

car.stop();

Output

Car Started
Stopped

Abstract Properties​

Not only methods.

Properties can also be abstract.

Example

abstract class Animal {
  abstract name: string;
}

Child

class Dog extends Animal {
  name = "Tommy";
}

Valid.

Multiple Abstract Methods​

Example

abstract class Shape {
  abstract area(): number;

  abstract perimeter(): number;
}

Implementation

class Rectangle extends Shape {
  constructor(
    public width: number,
    public height: number,
  ) {
    super();
  }

  area() {
    return this.width * this.height;
  }

  perimeter() {
    return 2 * (this.width + this.height);
  }
}

Usage

const rect = new Rectangle(10, 5);

Output

rect.area();
50

Constructors in Abstract Classes​

Abstract classes can have constructors.

Example

abstract class Animal {
  constructor(public name: string) {}
}

Child

class Dog extends Animal {
  constructor(name: string) {
    super(name);
  }
}

Usage

const dog = new Dog("Tommy");

Output

console.log(dog.name);
Tommy

Abstract Class as a Type​

Abstract classes can be used as types.

Example

abstract class Animal {
  abstract makeSound(): void;
}
class Dog extends Animal {
  makeSound() {
    console.log("Woof");
  }
}

Valid

let animal: Animal;

animal = new Dog();

Why?

Because:

Dog IS-A Animal

Polymorphism with Abstract Classes​

Abstract Class

abstract class Animal {
  abstract makeSound(): void;
}

Children

class Dog extends Animal {
  makeSound() {
    console.log("Woof");
  }
}
class Cat extends Animal {
  makeSound() {
    console.log("Meow");
  }
}

Function

function play(animal: Animal) {
  animal.makeSound();
}

Usage

play(new Dog());

Output

Woof

Usage

play(new Cat());

Output

Meow

This is:

Runtime Polymorphism

Abstract Class vs Interface​

One of the most common interview questions.

Interface​

interface Animal {
  makeSound(): void;
}

Provides:

Contract Only

Cannot contain:

Implementation
State
Constructors

(ignoring advanced edge cases)

Abstract Class​

abstract class Animal {
  eat() {
    console.log("Eating");
  }

  abstract makeSound(): void;
}

Provides:

Contract
+
Implementation

Comparison Table​

FeatureInterfaceAbstract Class
Methods Declaration✅✅
Method Implementation❌✅
Properties✅✅
Constructors❌✅
Access Modifiers❌✅
State Storage❌✅
Multiple Inheritance✅❌
Code Reuse❌✅

When to Use Interface?​

Use Interface when:

Only Contract Needed

Example:

Payment Provider
Logger
Repository
Serializer

When to Use Abstract Class?​

Use Abstract Class when:

Shared Logic Exists

Example:

Vehicle
Animal
Employee
Shape

Common Mistakes​

Instantiating Abstract Class​

Wrong

new Animal();

Error.

Only child classes can be created.

Forgetting Abstract Method Implementation​

Wrong

class Dog extends Animal {}

Must implement:

makeSound();

Using Abstract Class When Interface Is Enough​

Bad

Heavy hierarchy
for simple contracts

Prefer:

interface;

when no shared implementation exists.

Interview Questions​

Q1​

What is an Abstract Class?

Answer​

A class that cannot
be instantiated directly
and may contain
abstract methods.

Q2​

What is an Abstract Method?

Answer​

A method declaration
without implementation.

Q3​

Can Abstract Classes Have Normal Methods?

Answer​

Yes.

Q4​

Can Abstract Classes Have Constructors?

Answer​

Yes.

Q5​

Difference Between Interface and Abstract Class?

Answer​

Interface:
Contract Only

Abstract Class:
Contract + Implementation

Q6​

Can Abstract Classes Be Used as Types?

Answer​

Yes.

Cheat Sheet​

abstract class Animal {}
abstract makeSound(): void;
class Dog extends Animal {}
super(name);
let animal: Animal;
new Animal();

❌ Invalid

Key Takeaways​

  • Abstract Classes are incomplete base classes.
  • Abstract Classes cannot be instantiated directly.
  • Abstract Methods provide declarations without implementation.
  • Child classes must implement all abstract methods.
  • Abstract Classes can contain normal methods and properties.
  • Constructors are allowed in abstract classes.
  • Abstract Classes support code reuse.
  • Interfaces provide contracts only.
  • Abstract Classes provide contracts plus implementation.
  • Abstract Classes are heavily used in enterprise-level OOP designs.

Arrays

Arrays are one of the most commonly used data structures in TypeScript.

They allow us to store:

Multiple values of the same type

TypeScript enhances JavaScript arrays by adding:

  • Type Safety
  • Better IntelliSense
  • Compile-Time Validation
  • Safer Array Operations

Arrays are heavily used in:

  • API Responses
  • Database Records
  • React State
  • Lists and Collections
  • Data Processing

What is an Array?​

An array is a collection of values stored in a single variable.

JavaScript Example​

const numbers = [10, 20, 30, 40];

Instead of:

const num1 = 10;
const num2 = 20;
const num3 = 30;
const num4 = 40;

we can store all values together.

Why Do We Need Typed Arrays?​

Consider JavaScript:

const values = [10, 20, 30];

values.push("hello");

Valid JavaScript.

Result:

[10, 20, 30, "hello"];

This may introduce bugs.

TypeScript prevents this.

Array Type Syntax​

There are two ways to define array types.

Method 1 (Most Common)​

type[]

Example

const numbers: number[] = [10, 20, 30];

Meaning:

Array of Numbers

Method 2 (Generic Syntax)​

Array<type>;

Example

const numbers: Array<number> = [10, 20, 30];

Both are identical.

Number Arrays​

Example

const scores: number[] = [90, 95, 100];

Valid

scores.push(80);

Result

[90, 95, 100, 80];

Invalid

scores.push("hello");

Error:

Argument of type 'string'
is not assignable to type 'number'

String Arrays​

Example

const users: string[] = ["Prashant", "John", "Alex"];

Valid

users.push("David");

Invalid

users.push(100);

Error.

Boolean Arrays​

Example

const flags: boolean[] = [true, false, true];

Valid

flags.push(false);

Invalid

flags.push("true");

Type Inference with Arrays​

TypeScript can infer array types automatically.

Example

const numbers = [10, 20, 30];

Inferred Type

number[]

Now:

numbers.push("hello");

Error.

Mixed Arrays Using Union Types​

Sometimes arrays need multiple types.

Example

const values: (string | number)[] = ["Prashant", 25, "John", 30];

Valid

values.push("Alex");

Valid

values.push(100);

Invalid

values.push(true);

Important Interview Question​

Difference between:

string | number[]

and

(string | number)[]

First​

string | number[]

Means:

A string

OR

An array of numbers

Valid

"hello";

Valid

[1, 2, 3];

Second​

(string | number)[]

Means:

Array containing
strings and numbers

Valid

["hello", 10];

Accessing Array Elements​

Example

const users = ["Prashant", "John"];

Access

console.log(users[0]);

Output

Prashant

TypeScript Knows Element Types​

Example

const users: string[] = ["Prashant"];
users[0].toUpperCase();

Valid.

Why?

Because:

users[0]

is known to be:

string;

Array Length​

Example

const numbers = [10, 20, 30];
console.log(numbers.length);

Output

3

Common Array Methods​

push()​

Add at end.

const numbers = [10, 20];

numbers.push(30);

Result

[10, 20, 30];

pop()​

Remove last element.

numbers.pop();

Result

[10, 20];

shift()​

Remove first element.

numbers.shift();

unshift()​

Add at beginning.

numbers.unshift(5);

includes()​

Check existence.

numbers.includes(20);

Output

true

indexOf()​

Find index.

numbers.indexOf(20);

Output

1

Iterating Arrays​

for Loop​

const numbers = [10, 20, 30];

for (let i = 0; i < numbers.length; i++) {
  console.log(numbers[i]);
}

Output

10
20
30

for...of Loop​

Modern approach.

for (const num of numbers) {
  console.log(num);
}

Output

10
20
30

forEach()​

numbers.forEach((num) => {
  console.log(num);
});

Output

10
20
30

Arrays of Objects​

Very common in real applications.

Example

type User = {
  id: number;
  name: string;
};
const users: User[] = [
  {
    id: 1,
    name: "Prashant",
  },
  {
    id: 2,
    name: "John",
  },
];

Access

console.log(users[0].name);

Output

Prashant

Nested Arrays​

Arrays inside arrays.

Example

const matrix: number[][] = [
  [1, 2, 3],
  [4, 5, 6],
  [7, 8, 9],
];

Access

matrix[1][2];

Output

6

Multidimensional Arrays​

2D Array

number[][]

Example

const grid: number[][] = [
  [1, 2],
  [3, 4],
];

Readonly Arrays​

Sometimes arrays should not be modified.

Example

const numbers: readonly number[] = [10, 20, 30];

Attempt

numbers.push(40);

Error.

Attempt

numbers.pop();

Error.

Alternative Readonly Syntax​

ReadonlyArray<number>;

Equivalent to:

readonly number[]

Example

const users: ReadonlyArray<string> = ["Prashant", "John"];

Real World Example​

API Response

type User = {
  id: number;
  name: string;
};
const users: User[] = await fetchUsers();

TypeScript knows:

users[0].name → string

users[0].id → number

Strong typing throughout the application.

Common Mistakes​

Mixing Types Accidentally​

Wrong

const scores: number[] = [90, 95];

scores.push("100");

Error.

Confusing Array Types​

Wrong

string | number[]

when you intended:

(string | number)[]

Forgetting Readonly Arrays​

If data should never change:

readonly number[]

is often safer.

Interview Questions​

Q1​

How do you define an array of numbers?

Answer​

number[]

Q2​

Alternative syntax for arrays?

Answer​

Array<number>;

Q3​

Difference between:

string | number[]

and

(string | number)[]

Answer​

string | number[]

↓

string OR number array
(string | number)[]

↓

array containing strings and numbers

Q4​

How do you create a readonly array?

Answer​

readonly number[]

or

ReadonlyArray<number>;

Q5​

How do you type an array of objects?

Answer​

User[]

Cheat Sheet​

const nums: number[] = [1, 2, 3];
const nums: Array<number> = [1, 2, 3];
const values: (string | number)[] = ["hello", 10];
const users: User[] = [];
const matrix: number[][] = [
  [1, 2],
  [3, 4],
];
const nums: readonly number[] = [1, 2, 3];

Key Takeaways​

  • Arrays store multiple values of the same type.
  • TypeScript adds compile-time type safety to arrays.
  • Arrays can be typed using type[] or Array<type>.
  • Type inference automatically determines array types.
  • Union types can be used inside arrays.
  • Arrays can contain objects and nested arrays.
  • TypeScript understands element types when accessing array values.
  • Readonly arrays prevent accidental modifications.
  • Arrays are heavily used in APIs, React applications and backend systems.
  • Strongly typed arrays improve maintainability and reduce runtime bugs.

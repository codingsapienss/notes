Generic Interfaces

In previous chapter, we learned how Generic Functions allow us to write reusable and type-safe functions.

However, Generics are not limited to functions.

We can also create:

Generic Interfaces

which allow interfaces to work with multiple data types while maintaining strong type safety.

Generic Interfaces are heavily used in:

  • API Responses
  • React Components
  • Repositories
  • Database Layers
  • Collections
  • Enterprise Applications

The Problem Before Generic Interfaces​

Suppose we want an API response interface.

For Users

interface UserResponse {
  success: boolean;

  data: {
    id: number;
    name: string;
  };
}

For Products

interface ProductResponse {
  success: boolean;

  data: {
    id: number;
    title: string;
    price: number;
  };
}

Problem:

Duplicate Structure

Only:

data

changes.

Everything else remains the same.

Generic Interface Solution​

Instead of repeating:

UserResponse;
ProductResponse;
OrderResponse;

we can create:

interface ApiResponse<T>

What is a Generic Interface?​

A Generic Interface is:

An Interface
that accepts Type Parameters.

Syntax

interface Name<T> {}

Here:

T;

is a:

Type Placeholder

First Generic Interface​

interface ApiResponse<T> {
  success: boolean;

  data: T;
}

Notice:

data: T;

Type of data is now:

Flexible

Using Generic Interface​

User Type

type User = {
  id: number;

  name: string;
};

Response

const response: ApiResponse<User> = {
  success: true,

  data: {
    id: 1,
    name: "Prashant",
  },
};

TypeScript replaces:

T;

with:

User;

Generated Type

{
  success: boolean;

  data: User;
}

Another Example​

Product Type

type Product = {
  id: number;

  title: string;

  price: number;
};

Response

const response: ApiResponse<Product> = {
  success: true,

  data: {
    id: 1,
    title: "Laptop",
    price: 50000,
  },
};

Same interface.

Different data type.

Visualizing Generic Interfaces​

Interface

interface Box<T> {
  value: T;
}

Usage

Box<string>;

becomes:

{
  value: string;
}

Usage

Box<number>;

becomes:

{
  value: number;
}

Generic Interface with Arrays​

Example

interface ApiResponse<T> {
  success: boolean;

  data: T[];
}

User Type

type User = {
  id: number;

  name: string;
};

Usage

const response: ApiResponse<User> = {
  success: true,

  data: [
    {
      id: 1,
      name: "Prashant",
    },
  ],
};

Generated Type

{
  success: boolean;

  data: User[];
}

Multiple Generic Parameters​

Generic Interfaces can have multiple type parameters.

Syntax

interface Pair<T, U> {}

Example

interface Pair<T, U> {
  first: T;

  second: U;
}

Usage

const pair: Pair<string, number> = {
  first: "Age",

  second: 25,
};

Generated Type

{
  first: string;

  second: number;
}

Generic Key-Value Example​

Interface

interface KeyValue<K, V> {
  key: K;

  value: V;
}

Usage

const item: KeyValue<string, number> = {
  key: "age",

  value: 25,
};

Result

{
  key: string;

  value: number;
}

Generic Interfaces with Functions​

Interfaces can define generic function structures.

Example

interface Processor<T> {
  process(value: T): T;
}

Implementation

class StringProcessor implements Processor<string> {
  process(value: string) {
    return value.toUpperCase();
  }
}

Usage

const processor = new StringProcessor();

console.log(processor.process("hello"));

Output

HELLO

Generic Interfaces with Classes​

Very common.

Interface

interface Repository<T> {
  getById(id: number): T;

  save(item: T): void;
}

Implementation

type User = {
  id: number;

  name: string;
};
class UserRepository implements Repository<User> {
  getById(id: number) {
    return {
      id,
      name: "Prashant",
    };
  }

  save(user: User) {
    console.log("Saved", user);
  }
}

This pattern is used extensively in:

Backend Systems
ORMs
Repository Pattern

Real World API Response Model​

One of the most important examples.

Without Generics

interface UserResponse {}
interface ProductResponse {}
interface OrderResponse {}

With Generics

interface ApiResponse<T> {
  success: boolean;

  message: string;

  data: T;
}

User

type User = {
  id: number;

  name: string;
};

Usage

const response: ApiResponse<User> = {
  success: true,

  message: "Success",

  data: {
    id: 1,
    name: "Prashant",
  },
};

Product

type Product = {
  id: number;

  title: string;
};

Usage

const productResponse: ApiResponse<Product> = {
  success: true,

  message: "Success",

  data: {
    id: 1,
    title: "Laptop",
  },
};

Huge code reuse.

Generic Collection Example​

Interface

interface Collection<T> {
  items: T[];

  add(item: T): void;
}

Implementation

class UserCollection implements Collection<string> {
  items: string[] = [];

  add(item: string) {
    this.items.push(item);
  }
}

Usage

const users = new UserCollection();

users.add("Prashant");

Type-safe collection.

Generic Interfaces vs Generic Functions​

Generic Function

function identity<T>(value: T): T;

Purpose:

Reusable Function

Generic Interface

interface Box<T>

Purpose:

Reusable Structure

Generic Interfaces vs Generic Types​

Interface

interface Box<T> {
  value: T;
}

Type Alias

type Box<T> = {
  value: T;
};

Most of the time:

Both behave similarly

Differences will be covered later in:

Interface vs Type

Common Mistakes​

Using any Instead of Generic Interfaces​

Bad

interface ApiResponse {
  data: any;
}

Problem:

Type Safety Lost

Prefer

interface ApiResponse<T> {
  data: T;
}

Creating Separate Interfaces​

Bad

UserResponse;

ProductResponse;

OrderResponse;

When structure is identical.

Use:

ApiResponse<T>;

instead.

Forgetting Generic Arguments​

Wrong

const response: ApiResponse;

Correct

const response: ApiResponse<User>;

Cheat Sheet​

Key Takeaways​

  • Generic Interfaces allow interfaces to work with multiple data types.
  • They improve reusability while preserving type safety.
  • T acts as a type placeholder.
  • Multiple generic parameters are supported.
  • Generic Interfaces are heavily used in API responses and repository patterns.
  • Classes can implement Generic Interfaces.
  • Generic Interfaces eliminate duplicated type definitions.
  • Prefer Generic Interfaces over any when structure remains the same but data types change.
  • Generic Interfaces are a core building block of scalable TypeScript applications.
  • Understanding Generic Interfaces is essential before learning Generic Classes and Constraints.

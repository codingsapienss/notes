Enums

Enums (Enumerations) allow us to define a fixed set of named constants.

Instead of remembering:

0;
1;
2;

we can use:

Status.Pending;
Status.Success;
Status.Failed;

making code:

More Readable
More Maintainable
Less Error-Prone

What is an Enum?

An Enum is a special TypeScript feature that groups related constant values together.

Example​

enum Status {
  Pending,
  Success,
  Failed,
}

Now we can write:

const currentStatus = Status.Success;

instead of:

const currentStatus = 1;

Why Do We Need Enums?

Without Enums:

const STATUS_PENDING = 0;
const STATUS_SUCCESS = 1;
const STATUS_FAILED = 2;

Usage:

if(status === 1)

Question:

What does 1 mean?

Not obvious.

With Enum:

if(status === Status.Success)

Immediately clear.

Numeric Enums

Default enum type.

Example​

enum Status {
  Pending,
  Success,
  Failed,
}

TypeScript automatically assigns:

enum Status {
  Pending = 0,
  Success = 1,
  Failed = 2,
}

Accessing Enum Values

Example

enum Status {
  Pending,
  Success,
  Failed,
}

Usage

console.log(Status.Pending);

Output:

0
console.log(Status.Success);

Output:

1

Custom Numeric Values

You can specify values manually.

Example

enum Status {
  Pending = 100,
  Success = 200,
  Failed = 500,
}

Usage

console.log(Status.Success);

Output:

200

Auto Increment Behavior

Example

enum Status {
  Pending = 10,
  Success,
  Failed,
}

Generated Values

Pending = 10;
Success = 11;
Failed = 12;

TypeScript automatically increments.

Reverse Mapping

Unique feature of Numeric Enums.

Example

enum Status {
  Pending,
  Success,
  Failed,
}

Forward Mapping

Status.Success;

Output:

1

Reverse Mapping

Status[1];

Output:

"Success"

Example

console.log(Status[2]);

Output:

Failed

String Enums

Instead of numbers, use strings.

Example

enum Status {
  Pending = "PENDING",
  Success = "SUCCESS",
  Failed = "FAILED",
}

Usage

console.log(Status.Success);

Output:

SUCCESS

Why String Enums?

More readable.

Compare:

1

vs

SUCCESS

When debugging:

SUCCESS

is much easier to understand.

Real World Example

API Status

enum ApiStatus {
  Loading = "LOADING",
  Success = "SUCCESS",
  Error = "ERROR",
}

Usage

let status = ApiStatus.Loading;

Check

if (status === ApiStatus.Success) {
  console.log("Done");
}

Heterogeneous Enums

Mixing strings and numbers.

Example

enum Result {
  Success = 1,
  Failed = "FAILED",
}

Technically valid.

Not recommended.

Why?

Inconsistent
Confusing
Harder to Maintain

Enum as a Type

Enums create both:

Value

and

Type

Example

enum Status {
  Pending,
  Success,
  Failed,
}

Variable

let currentStatus: Status;

Valid

currentStatus = Status.Success;

Invalid Assignment

currentStatus = 100;

Usually invalid in strict codebases.

Prefer enum values.

Enum in Functions

Example

enum Role {
  Admin,
  User,
  Guest,
}

Function

function login(role: Role) {
  console.log(role);
}

Call

login(Role.Admin);

Valid.

Real World Example

User Roles

enum UserRole {
  Admin,
  Manager,
  Employee,
}

Usage

const role = UserRole.Admin;

Check

if (role === UserRole.Admin) {
  console.log("Full Access");
}

Const Enums

Optimization feature.

Example

const enum Status {
  Pending,
  Success,
  Failed,
}

Usage

const status = Status.Success;

Generated JavaScript:

const status = 1;

No Enum Object Created.

Benefits:

Smaller Bundle
Faster Execution

Normal Enum Compilation

TypeScript

enum Status {
  Pending,
  Success,
}

Generated JavaScript

var Status;

(function (Status) {
  Status[(Status["Pending"] = 0)] = "Pending";

  Status[(Status["Success"] = 1)] = "Success";
})(Status || (Status = {}));

Notice:

Extra JavaScript Object Created

Const Enum Compilation

TypeScript

const enum Status {
  Pending,
  Success,
}

Generated JavaScript

const status = 1;

No runtime object.

Enum vs Union Literal Types

Modern TypeScript often prefers:

type Status = "pending" | "success" | "failed";

instead of:

enum Status {
  Pending,
  Success,
  Failed,
}

Enum Version

enum Status {
  Pending,
  Success,
  Failed,
}

Usage:

Status.Success;

Union Literal Version

type Status = "pending" | "success" | "failed";

Usage:

const status: Status = "success";

Why Many Developers Prefer Literal Unions?

Benefits:

Less Generated Code
Better Type Inference
More Flexible
Works Better With APIs

Modern codebases often use:

Union Literal Types

instead of Enums.

When Should You Use Enums?

Good Use Cases:

Status Codes
User Roles
Application States
Configuration Constants

Examples

Role.Admin;
Role.Manager;
Role.Employee;

When Should You Avoid Enums?

If simple strings work:

type Status = "pending" | "success" | "failed";

Often preferred.

Common Mistakes

Mixing String and Numeric Enums​

Bad

enum Status {
  Pending = 0,
  Success = "SUCCESS",
}

Avoid.

Using Magic Numbers​

Bad

if(status === 1)

Use:

if(
  status ===
  Status.Success
)

Forgetting String Enums Don't Have Reverse Mapping​

Numeric Enum

Status[1];

Works.

String Enum

Status["SUCCESS"];

Does NOT reverse map.

Numeric vs String Enum

FeatureNumeric EnumString Enum
Auto Increment✅❌
Reverse Mapping✅❌
ReadabilityMediumHigh
DebuggingMediumExcellent

Interview Questions

Q1​

What is an Enum?

Answer​

A collection of named constants.

Q2​

What value does this generate?

enum Status {
  Pending,
  Success,
  Failed,
}

Answer​

Pending = 0;
Success = 1;
Failed = 2;

Q3​

What is Reverse Mapping?

Answer​

Status[1];

returns:

"Success"

for numeric enums.

Q4​

Difference between String and Numeric Enums?

Answer​

Numeric Enums:
Auto Increment + Reverse Mapping

String Enums:
More Readable
No Reverse Mapping

Q5​

What is a Const Enum?

Answer​

An enum removed during compilation
for better performance.

Cheat Sheet

enum Status {
  Pending,
  Success,
  Failed,
}
enum Status {
  Pending = "PENDING",
  Success = "SUCCESS",
}
Status.Success;
Status[1];
const enum Status {
  Pending,
  Success,
}
type Status = "pending" | "success" | "failed";

Key Takeaways

  • Enums group related constants together.
  • Numeric enums start from 0 by default.
  • Numeric enums support reverse mapping.
  • String enums are more readable and common in APIs.
  • Const enums improve performance by removing runtime objects.
  • Enums can be used as both values and types.
  • Modern TypeScript often prefers Union Literal Types over Enums.
  • Avoid heterogeneous enums.
  • Use enums when you need named constants with strong semantics.
  • Understanding enums is important for interviews and legacy TypeScript codebases.

Async TypeScript (Promise Types & Async/Await)

Modern applications spend a significant amount of time performing:

  • API Requests
  • Database Queries
  • File Operations
  • Authentication
  • Payment Processing

These operations take time.

We cannot block the application while waiting.

This is why JavaScript and TypeScript provide:

Asynchronous Programming

using:

Promise;
async;
await;

Understanding Promise Types is essential for:

  • React
  • Angular
  • Node.js
  • NestJS
  • Express
  • Backend Development

Why Do We Need Async Programming?

Consider:

const user = fetch("/users/1");

console.log(user);

Output:

Promise { <pending> }

Why?

Because:

Network Requests
Take Time

The browser cannot stop execution and wait.

Instead:

Request Starts
Execution Continues
Response Arrives Later

Synchronous vs Asynchronous

Synchronous​

console.log("A");

console.log("B");

console.log("C");

Output

A
B
C

Each statement waits for the previous one.

Asynchronous​

console.log("A");

setTimeout(() => {
  console.log("B");
}, 1000);

console.log("C");

Output

A
C
B

Because:

setTimeout runs later

What is a Promise?

A Promise represents:

A Future Value

Something that:

Will be available later

Promise States

A Promise can be in three states.

Pending​

Operation Still Running

Fulfilled​

Operation Successful

Rejected​

Operation Failed

Visualization

Promise
   │
   ▼

 Pending
   │
 ┌─┴─┐
 │   │
 ▼   ▼

Success Failure

Fulfilled Rejected

Creating a Promise

Example

const promise = new Promise<string>((resolve, reject) => {
  resolve("Success");
});

Notice:

Promise<string>;

This means:

Promise resolves
to a string

Promise Type Syntax

Promise<T>;

Examples

Promise<string>;
Promise<number>;
Promise<boolean>;
Promise<User>;

Promise Example

const promise: Promise<number> = new Promise((resolve) => {
  resolve(100);
});

Resolved Value

number;

TypeScript knows:

Promise returns number

Consuming Promises with then()

Example

promise.then((value) => {
  console.log(value);
});

Output

100

Type of:

value;

↓

number;

because:

Promise<number>;

Real API Example

fetch("/users/1")
  .then((response) => response.json())
  .then((data) => {
    console.log(data);
  });

Works.

But:

Can become hard to read

for multiple async operations.

Async/Await

Modern TypeScript uses:

async;
await;

These keywords make asynchronous code look synchronous.

async Function

Example

async function greet() {
  return "Hello";
}

Question:

What is the return type?

Answer:

Promise<string>;

Important Rule

Every:

async function

always returns:

Promise;

Even if:

return "Hello";

TypeScript converts it internally to:

Promise.resolve("Hello");

Example

async function getAge() {
  return 25;
}

Return Type

Promise<number>;

Not:

number;

Understanding await

await waits for a Promise to finish.

Example

const result = await Promise.resolve("Hello");

Type

string;

Notice:

Promise<string>;

↓

string;

await unwraps the Promise.

First Async/Await Example

async function getMessage() {
  const result = await Promise.resolve("Hello");

  return result;
}

Return Type

Promise<string>;

Inside Function

result;

↓

string;

API Example Using Fetch

Model

interface User {
  id: number;

  name: string;

  email: string;
}

Function

async function getUser()
: Promise<User> {

  const response =
    await fetch(
      "/users/1"
    );

  const user =
    await response.json()
    as User;

  return user;
}

Return Type

Promise<User>;

Usage

const user = await getUser();

Type

User;

Why Explicit Return Types Matter

Bad

async function getUser() {}

TypeScript infers.

Better

async function getUser(): Promise<User> {}

Benefits

Clear Documentation
Better Safety
Better Refactoring

Async Function Returning Arrays

Model

interface User {
  id: number;

  name: string;
}

Function

async function getUsers(): Promise<User[]> {
  const response = await fetch("/users");

  return (await response.json()) as User[];
}

Return Type

Promise<User[]>;

Usage

const users = await getUsers();

Type

User[]

Generic API Response

Very common.

Response Model

interface ApiResponse<T> {
  success: boolean;

  data: T;
}

User Model

interface User {
  id: number;

  name: string;
}

Function

async function getUser(): Promise<ApiResponse<User>> {
  const response = await fetch("/users/1");

  return (await response.json()) as ApiResponse<User>;
}

Return Type

Promise<ApiResponse<User>>;

Usage

const result = await getUser();

Type

result.data;

↓

User;

Async Error Handling

Bad

const user = await getUser();

If request fails:

Unhandled Exception

Try Catch

try {
  const user = await getUser();
} catch (error) {
  console.log(error);
}

Type of Error

TypeScript treats:

error;

as:

unknown;

Why?

Because:

Anything can be thrown

Proper Error Narrowing

try {
} catch (error) {
  if (error instanceof Error) {
    console.log(error.message);
  }
}

Now:

error.message;

is safe.

Promise.all()

One of the most useful Promise APIs.

Problem

Need multiple requests.

Bad

const users = await getUsers();

const products = await getProducts();

Runs sequentially.

Better

const [users, products] = await Promise.all([getUsers(), getProducts()]);

Runs concurrently.

Types

users;

↓

User[]
products;

↓

Product[]

Promise.all Type Inference

Example

const result = await Promise.all([Promise.resolve("A"), Promise.resolve(100)]);

Type

[string, number];

Very powerful inference.

Promise

Sometimes functions return nothing.

Example

async function saveUser(): Promise<void> {
  console.log("Saved");
}

Meaning:

Promise resolves
with no value

Promise

Rare.

Used when function never completes normally.

Example

async function fail(): Promise<never> {
  throw new Error();
}

Meaning:

Function never
returns a value.

Real World Service Layer

Model

interface User {
  id: number;

  name: string;
}

Service

class UserService {
  async getUser(id: number): Promise<User> {
    const response = await fetch(`/users/${id}`);

    return (await response.json()) as User;
  }
}

Usage

const service = new UserService();

const user = await service.getUser(1);

Type

User;

Common Mistakes

Forgetting await​

Wrong

const user = getUser();

Type

Promise<User>;

Not:

User;

Correct

const user = await getUser();

Returning Wrong Type​

Wrong

async function getUser(): User {}

Correct

async function getUser(): Promise<User> {}

Ignoring Error Handling​

Bad

await fetch("/users");

Always consider:

try/catch

Interview Questions

Q1​

What does an async function return?

Answer​

Always a Promise.

Q2​

What does await do?

Answer​

Waits for a Promise
and unwraps its value.

Q3​

What is Promise?

Answer​

A Promise that resolves
to type T.

Q4​

What is the type of error in catch?

Answer​

unknown;

Q5​

Why use Promise.all()?

Answer​

Run multiple async tasks
concurrently.

Cheat Sheet

Promise<T>;
Promise<User>;
Promise<User[]>;
async function fn();

↓

Promise<...>
await promise;

↓

Resolved Value
Promise.all([...])
Promise<void>;
Promise<never>;

Key Takeaways

  • Promises represent future values.
  • A Promise can be Pending, Fulfilled, or Rejected.
  • Promise<T> represents a promise resolving to type T.
  • Every async function automatically returns a Promise.
  • await unwraps the resolved value of a Promise.
  • API functions should use explicit Promise return types.
  • Generic API models like ApiResponse<T> work well with async code.
  • Errors inside catch are typed as unknown.
  • Promise.all() allows concurrent execution with strong type inference.
  • Async programming is a fundamental skill for modern TypeScript development.

Advanced Interfaces

In previous chapter, we learned how Interfaces define the structure of objects.

In real-world applications, interfaces are used for much more than objects.

They can describe:

  • Functions
  • Dynamic Objects
  • Inheritance Hierarchies
  • API Contracts
  • React Props
  • Class Contracts

This chapter covers the advanced capabilities of interfaces that are heavily used in production TypeScript applications.

Function Interfaces

What is a Function Interface?​

An Interface can describe the structure of a function.

Instead of:

type AddFunction = (a: number, b: number) => number;

we can use:

interface AddFunction {
  (a: number, b: number): number;
}

Example

interface AddFunction {
  (a: number, b: number): number;
}

Implementation

const add: AddFunction = (a, b) => {
  return a + b;
};

Usage

console.log(add(10, 20));

Output

30

Why Use Function Interfaces?

Useful when:

Many functions share
the same signature.

Example

interface Calculator {
  (a: number, b: number): number;
}
const add: Calculator = (a, b) => a + b;

const multiply: Calculator = (a, b) => a * b;

Both follow the same contract.

Function Interface with Object Properties

Interfaces can describe:

Functions + Properties

simultaneously.

Example

interface Counter {
  (start: number): string;

  count: number;
}

Meaning:

Counter is callable
and
contains a count property.

This pattern is used in advanced libraries.

Index Signatures

One of the most important interface features.

Problem

Normally:

interface User {
  name: string;
  age: number;
}

Only these properties are allowed.

But what if keys are dynamic?

Example:

{
  math: 90,
  science: 95,
  english: 85
}

Property names are not known beforehand.

Solution

Use Index Signatures.

Syntax​

interface InterfaceName {
  [key: string]: Type;
}

Example

interface Scores {
  [subject: string]: number;
}

Usage

const marks: Scores = {
  math: 95,
  science: 90,
  english: 88,
};

Valid.

Why Does This Work?

Interface says:

Any string key
must contain a number value.

Therefore:

math: 95;

Valid.

science: 90;

Valid.

Invalid Example

const marks: Scores = {
  math: 95,
  english: "A",
};

Error.

Because:

Value must be number.

Number Index Signatures

Example

interface NumberMap {
  [index: number]: string;
}

Usage

const users: NumberMap = {
  0: "Prashant",
  1: "John",
};

Valid.

Real World Example

API Dictionary

interface Users {
  [id: string]: {
    name: string;
  };
}

Usage

const users: Users = {
  "101": {
    name: "Prashant",
  },

  "102": {
    name: "John",
  },
};

Very common in backend systems.

Mixing Explicit Properties with Index Signatures

Allowed.

Example

interface User {
  id: number;

  [key: string]: string | number;
}

Usage

const user: User = {
  id: 1,
  name: "Prashant",
  city: "Delhi",
};

Valid.

Important Rule

Explicit properties must be compatible with the index signature.

Example

interface User {
  id: number;

  [key: string]: string | number;
}

Valid because:

id;

is:

number;

which belongs to:

string | number;

Interface Extension

One of the most powerful interface features.

Why Extension?

Without extension:

interface User {
  id: number;
  name: string;
}
interface Admin {
  id: number;
  name: string;
  permissions: string[];
}

Repeated properties.

Extension eliminates duplication.

Syntax

interface Child extends Parent {}

Example

interface User {
  id: number;
  name: string;
}
interface Admin extends User {
  permissions: string[];
}

Result

{
  id: number;
  name: string;
  permissions: string[];
}

Usage

const admin: Admin = {
  id: 1,

  name: "Prashant",

  permissions: ["read", "write"],
};

Visual Representation

User
 ├─ id
 └─ name

Admin
 ├─ id
 ├─ name
 └─ permissions

Multiple Interface Extension

Interfaces can inherit from multiple interfaces.

Example

interface Person {
  name: string;
}
interface Employee {
  employeeId: number;
}
interface Admin {
  permissions: string[];
}

Combined

interface SuperAdmin extends Person, Employee, Admin {}

Result

{
  name: string;
  employeeId: number;
  permissions: string[];
}

Usage

const admin: SuperAdmin = {
  name: "Prashant",

  employeeId: 101,

  permissions: ["all"],
};

Interface Inheritance Hierarchy

Real projects often create inheritance chains.

Example

interface Entity {
  id: number;
}
interface User extends Entity {
  name: string;
}
interface Admin extends User {
  permissions: string[];
}

Result

Entity
   ↓
 User
   ↓
 Admin

Very common.

Declaration Merging

This is the biggest difference between:

Interface

and

Type Alias

What is Declaration Merging?

TypeScript automatically merges interfaces having the same name.

Example

interface User {
  name: string;
}
interface User {
  age: number;
}

Result

interface User {
  name: string;
  age: number;
}

TypeScript combines them.

Usage

const user: User = {
  name: "Prashant",
  age: 25,
};

Valid.

Why Is Declaration Merging Useful?

Large libraries often extend existing interfaces.

Example

interface Window {
  myAppVersion: string;
}

Now:

window.myAppVersion;

becomes available everywhere.

This is heavily used in:

Browser APIs
Node.js Typings
Third-party Libraries

Type Alias Comparison

Interfaces

interface User {
  name: string;
}

interface User {
  age: number;
}

Valid.

Type Alias

type User = {
  name: string;
};
type User = {
  age: number;
};

Error.

Duplicate Identifier

Real World Example

API Models

Base Interface

interface ApiResponse {
  success: boolean;
}

Extended Interface

interface UserResponse extends ApiResponse {
  user: {
    id: number;
    name: string;
  };
}

Usage

const response: UserResponse = {
  success: true,

  user: {
    id: 1,
    name: "Prashant",
  },
};

Common Mistakes

Using Extension When Composition Is Better​

Avoid:

A extends B
extends C
extends D
extends E

Deep hierarchies become difficult to maintain.

Forgetting Declaration Merging​

Interfaces merge automatically.

Two interfaces with same name:

interface User {}
interface User {}

are combined.

Not overwritten.

Wrong Index Signature Type​

Wrong

interface Scores {
  [key: string]: number;
}
{
  math: "90";
}

Error.

Value must be:

number;

Interview Questions

Q1​

Can Interfaces describe functions?

Answer​

Yes.

interface Add {
  (a: number, b: number): number;
}

Q2​

What is an Index Signature?

Answer​

A way to describe dynamic property names.

Example:

[key: string]: number

Q3​

Can an Interface extend another Interface?

Answer​

Yes.

interface Admin extends User {}

Q4​

Can an Interface extend multiple Interfaces?

Answer​

Yes.

interface A extends B, C {}

Q5​

What is Declaration Merging?

Answer​

Interfaces with the same name are automatically combined by TypeScript.

Q6​

Do Type Aliases support Declaration Merging?

Answer​

No.

Only interfaces support declaration merging.

Cheat Sheet

interface Add {
  (a: number, b: number): number;
}
interface Scores {
  [key: string]: number;
}
interface Admin extends User {}
interface Admin extends User, Employee {}
interface User {
  name: string;
}

interface User {
  age: number;
}

↓

{
  name: string;
  age: number;
}

Key Takeaways

  • Interfaces can describe functions, not just objects.
  • Index Signatures support dynamic property names.
  • Interface Extension promotes reusability and avoids duplication.
  • Interfaces can inherit from multiple interfaces.
  • Declaration Merging is a unique feature of interfaces.
  • Interfaces are heavily used in API contracts and library typings.
  • Index Signatures are common when working with dictionaries and maps.
  • Extension helps create scalable type hierarchies.
  • Declaration Merging is one of the major differences between interfaces and type aliases.
  • Advanced interface concepts are widely used in large TypeScript codebases.

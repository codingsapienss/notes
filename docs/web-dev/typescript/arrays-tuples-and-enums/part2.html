Tuples

Arrays are useful when all elements have the same type.

Example:

const numbers: number[] = [10, 20, 30];

Every element is:

number;

But sometimes we need:

Fixed Number of Elements
Fixed Order
Different Types at Different Positions

This is where:

Tuples

become useful.

Tuples are one of the most frequently asked TypeScript interview topics.

What is a Tuple?​

A Tuple is a special type of array where:

Length is Fixed
Order is Fixed
Types are Fixed

Example​

const user: [string, number] = ["Prashant", 25];

Meaning:

Index 0 → string

Index 1 → number

Why Not Use a Normal Array?​

Consider:

const user: (string | number)[] = ["Prashant", 25];

TypeScript only knows:

Array contains strings and numbers

It does NOT know:

Position 0 → string

Position 1 → number

A Tuple provides this guarantee.

Tuple Syntax​

Basic Syntax​

[type1, type2];

Example

const user: [string, number] = ["Prashant", 25];

Valid Tuple​

const user: [string, number] = ["Prashant", 25];

Explanation

Index 0 → string

Index 1 → number

Matches perfectly.

Invalid Tuple​

Wrong Order

const user: [string, number] = [25, "Prashant"];

Error.

Because:

Index 0 expected string

Received number

Another Invalid Example​

Missing Element

const user: [string, number] = ["Prashant"];

Error.

Because:

Tuple expects 2 elements.

Fixed Length Guarantee​

Example

const point: [number, number] = [10, 20];

Valid.

Attempt

const point: [number, number] = [10, 20, 30];

Error.

Why?

Because:

Tuple length is fixed.

Accessing Tuple Values​

Example

const user: [string, number] = ["Prashant", 25];

Access

console.log(user[0]);

Output

Prashant

Access

console.log(user[1]);

Output

25

TypeScript Knows Exact Types​

Example

const user: [string, number] = ["Prashant", 25];

TypeScript knows:

user[0];

is:

string;

Therefore:

user[0].toUpperCase();

Valid.

TypeScript also knows:

user[1];

is:

number;

Therefore:

user[1].toFixed(2);

Valid.

Real World Example​

Coordinates

Example

const location: [number, number] = [28.6139, 77.209];

Meaning

Latitude
Longitude

Without Tuple

number[]

TypeScript cannot guarantee:

Exactly two values.

Another Real World Example​

API Result

const response: [boolean, string] = [true, "Success"];

Meaning

Index 0 → Status

Index 1 → Message

Tuple with More Elements​

Example

const employee: [number, string, boolean] = [101, "Prashant", true];

Meaning

ID
Name
Active Status

Readonly Tuples​

Sometimes tuple values should never change.

Example

const point: readonly [number, number] = [10, 20];

Attempt

point[0] = 50;

Error.

Readonly protects tuple values.

Alternative Readonly Syntax​

Using Utility Type

Readonly<[number, number]>;

Example

const point: Readonly<[number, number]> = [10, 20];

Equivalent to:

readonly[(number, number)];

Optional Tuple Elements​

Sometimes some positions are optional.

Syntax​

[
  type1,
  type2?
]

Example

const user: [string, number?] = ["Prashant"];

Valid.

Also Valid

["Prashant", 25];

Why Use Optional Tuple Elements?​

Example:

Name required

Age optional

Tuple

[
  string,
  number?
]

models this perfectly.

Rest Elements in Tuples​

A tuple can contain:

Fixed Values
+
Variable Values

Example

type Scores = [string, ...number[]];

Meaning:

First value → string

Remaining values → numbers

Valid

const result: Scores = ["Prashant", 90, 95, 100];

Named Tuples​

Introduced for better readability.

Example

type User = [name: string, age: number];

Usage

const user: User = ["Prashant", 25];

Named tuples do NOT affect runtime.

They improve:

Readability
Documentation
Developer Experience

Tuple vs Array​

One of the most common interview questions.

Array​

const values: string[] = ["A", "B", "C"];

Properties:

Variable Length

Same Type

Tuple​

const user: [string, number] = ["Prashant", 25];

Properties:

Fixed Length

Fixed Order

Different Types Allowed

Comparison Table​

FeatureArrayTuple
LengthVariableFixed
Order ImportantNoYes
Different TypesUsually NoYes
Position Based TypesNoYes
Real World UsageListsStructured Data

Tuple Destructuring​

Very common.

Example

const user: [string, number] = ["Prashant", 25];

Destructure

const [name, age] = user;

Result

name → "Prashant"

age → 25

Function Returning Tuples​

Common pattern.

Example

function getUser(): [string, number] {
  return ["Prashant", 25];
}

Usage

const [name, age] = getUser();

Common Mistakes​

Using Array Instead of Tuple​

Wrong

const point: number[] = [10, 20];

TypeScript cannot guarantee:

Exactly two values.

Use:

[number, number];

Wrong Order​

Wrong

[25, "Prashant"];

when tuple expects:

[string, number];

Confusing Tuple with Object​

Tuple

["Prashant", 25];

Object

{
  name: "Prashant",
  age: 25
}

Objects are usually more readable.

When Should You Use Tuples?​

Good Use Cases:

Coordinates

RGB Colors

Database Results

API Status + Message

Function Returns

Structured Fixed Data

Avoid When:

Data has many fields

Meaning is unclear

Object would be more readable

Instead use:

{
  name: string;
  age: number;
}

Interview Questions​

Q1​

What is a Tuple?

Answer​

A fixed-length array
where each position has a predefined type.

Q2​

Difference between Array and Tuple?

Answer​

Arrays:
Variable length

Tuples:
Fixed length and fixed types.

Q3​

Can Tuples have optional values?

Answer​

Yes.

[string, number?]

Q4​

Can Tuples have rest elements?

Answer​

Yes.

[string, ...number[]]

Q5​

Can Tuples be readonly?

Answer​

Yes.

readonly[(number, number)];

Cheat Sheet​

[string, number];
[number, number];
readonly[(number, number)];
[string, number?]
[
  string,
  ...number[]
]
[
  name: string,
  age: number
]

Key Takeaways​

  • Tuples are fixed-length arrays with fixed types at specific positions.
  • Tuples provide stronger guarantees than regular arrays.
  • TypeScript knows the exact type of each tuple element.
  • Tuples are useful for coordinates, API responses and function return values.
  • Tuples support readonly modifiers.
  • Tuples support optional elements.
  • Tuples support rest elements.
  • Named tuples improve readability.
  • Arrays are best for collections of similar data.
  • Tuples are best for structured data with a fixed shape.

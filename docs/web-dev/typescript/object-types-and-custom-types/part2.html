Type Aliases

In previous chapter, we learned how to define Object Types directly using inline type definitions.

While that works for small examples, it quickly becomes difficult to maintain in real-world applications.

Type Aliases solve this problem by allowing us to:

  • Create reusable custom types
  • Improve readability
  • Reduce duplication
  • Build complex type systems

Type Aliases are one of the most heavily used TypeScript features.

Why Do We Need Type Aliases?​

Consider the following function:

function createUser(user: {
  id: number;
  name: string;
  email: string;
  isAdmin: boolean;
}) {
  console.log(user.name);
}

Looks okay initially.

Now imagine:

function updateUser(user: {
  id: number;
  name: string;
  email: string;
  isAdmin: boolean;
}) {}

function deleteUser(user: {
  id: number;
  name: string;
  email: string;
  isAdmin: boolean;
}) {}

We are repeating the same structure everywhere.

Problems:

Code Duplication
Hard Maintenance
Poor Readability
Higher Chance of Errors

What is a Type Alias?​

A Type Alias allows us to:

Give a name to a type.

Syntax​

type TypeName = TypeDefinition;

Example​

type User = {
  id: number;
  name: string;
  email: string;
};

Now:

const user: User = {
  id: 1,
  name: "Prashant",
  email: "abc@gmail.com",
};

Mental Model​

Think of:

type User = ...

as:

A custom type definition

similar to:

typedef

in C/C++.

Reusing Type Aliases​

Without Alias:

function createUser(user: { id: number; name: string }) {}

With Alias:

type User = {
  id: number;
  name: string;
};

function createUser(user: User) {}

Cleaner.

More maintainable.

More readable.

Type Aliases with Variables​

Example​

type User = {
  name: string;
  age: number;
};
const user: User = {
  name: "Prashant",
  age: 25,
};

Output

console.log(user.name);
Prashant

Type Checking Still Happens​

Valid

const user: User = {
  name: "Prashant",
  age: 25,
};

Invalid

const user: User = {
  name: "Prashant",
  age: "25",
};

Error:

Type 'string'
is not assignable to type 'number'

Nested Type Aliases​

One alias can use another alias.

Example​

type Address = {
  city: string;
  pincode: number;
};
type User = {
  name: string;
  address: Address;
};

Usage

const user: User = {
  name: "Prashant",
  address: {
    city: "Delhi",
    pincode: 201001,
  },
};

Why Nest Types?​

Real applications contain relationships.

Example:

User
 └── Address

Order
 └── Product

Company
 └── Employee

Nested aliases make these models manageable.

Deeply Nested Example​

type Contact = {
  phone: string;
};

type Address = {
  city: string;
  contact: Contact;
};

type User = {
  name: string;
  address: Address;
};

Usage

const user: User = {
  name: "Prashant",
  address: {
    city: "Delhi",
    contact: {
      phone: "9999999999",
    },
  },
};

Type Aliases with Functions​

Aliases can be used in parameters.

Example​

type User = {
  name: string;
  age: number;
};
function printUser(user: User) {
  console.log(user.name);
}

Call

printUser({
  name: "Prashant",
  age: 25,
});

Output

Prashant

Type Aliases with Return Types​

Example

type User = {
  name: string;
  age: number;
};
function createUser(): User {
  return {
    name: "Prashant",
    age: 25,
  };
}

Return Type:

User

Type Aliases with Arrays​

Array of Numbers​

type Scores = number[];

Usage

const marks: Scores = [90, 85, 95];

Array of Strings​

type UserNames = string[];
const users: UserNames = ["Prashant", "John", "Alex"];

Array of Objects​

Very common.

type User = {
  id: number;
  name: string;
};
type Users = User[];

Usage

const users: Users = [
  {
    id: 1,
    name: "Prashant",
  },
  {
    id: 2,
    name: "John",
  },
];

Type Aliases with Functions (Function Types)​

A Type Alias can describe a function itself.

Example​

type AddFunction = (a: number, b: number) => number;

Usage

const add: AddFunction = (a, b) => a + b;

Call

console.log(add(10, 20));

Output

30

Why Use Function Aliases?​

Useful when:

Many functions share
the same signature.

Example

type Calculator = (a: number, b: number) => number;
const add: Calculator = (a, b) => a + b;

const multiply: Calculator = (a, b) => a * b;

Type Aliases with Union Types​

One of the biggest advantages.

Example​

type Status = "success" | "error" | "loading";

Usage

let state: Status;

state = "success";

Valid.

state = "pending";

Error.

Why Is This Useful?​

Restricts values.

Without Alias

let state: "success" | "error" | "loading";

Repeated everywhere.

With Alias

type Status = "success" | "error" | "loading";

Reusable.

Type Aliases with Intersection Types​

Aliases can combine multiple types.

Example​

type Person = {
  name: string;
};
type Employee = {
  employeeId: number;
};
type Staff = Person & Employee;

Result

{
  name: string;
  employeeId: number;
}

Usage

const staff: Staff = {
  name: "Prashant",
  employeeId: 101,
};

Type Alias vs Inline Object Type​

Inline​

function printUser(user: { name: string; age: number }) {}

Alias​

type User = {
  name: string;
  age: number;
};

function printUser(user: User) {}

Alias version is:

Cleaner
Reusable
Maintainable

Can Type Aliases Be Reopened?​

Important Interview Question.

Example

type User = {
  name: string;
};

Trying:

type User = {
  age: number;
};

Error.

Duplicate Identifier

Unlike Interfaces:

Type Aliases Cannot Merge

Naming Conventions​

Common conventions:

PascalCase​

type User = {};
type Product = {};
type ApiResponse = {};

Avoid:

type user = {};

PascalCase is standard.

Real World Example​

API Response

Without Alias

function saveUser(user: { id: number; name: string; email: string }) {}

With Alias

type User = {
  id: number;
  name: string;
  email: string;
};

function saveUser(user: User) {}

Much easier to maintain.

Common Mistakes​

Repeating Types Everywhere​

Bad

function a(user:{...}) {}

function b(user:{...}) {}

Use aliases instead.

Creating Duplicate Aliases​

Wrong

type User = {};
type User = {};

Error.

Using Poor Names​

Bad

type A = {};
type B = {};

Use meaningful names.

Interview Questions​

Q1​

What is a Type Alias?

Answer​

A named reusable type definition.

Q2​

Why use Type Aliases?

Answer​

To improve readability,
reuse and maintainability.

Q3​

Can a Type Alias represent a function?

Answer​

Yes.

type Add = (a: number, b: number) => number;

Q4​

Can a Type Alias represent an array?

Answer​

Yes.

type Users = User[];

Q5​

Can Type Aliases be merged?

Answer​

No.

Unlike interfaces, Type Aliases cannot merge.


---

# Cheat Sheet

```ts
type User = {
  id: number;
  name: string;
};
const user: User = {
  id: 1,
  name: "Prashant",
};
type Users = User[];
type Status = "success" | "error" | "loading";
type Calculator = (a: number, b: number) => number;
type Staff = Person & Employee;

Key Takeaways

  • Type Aliases create reusable named types.
  • They reduce duplication and improve readability.
  • Type Aliases can represent objects, arrays, functions, unions and intersections.
  • Nested aliases help model complex real-world data.
  • Function signatures can be stored inside aliases.
  • Type Aliases are heavily used in production TypeScript applications.
  • Type Aliases cannot be reopened or merged.
  • Use PascalCase naming conventions.
  • Prefer aliases over repeating inline object types.
  • Type Aliases form the foundation for scalable TypeScript codebases.

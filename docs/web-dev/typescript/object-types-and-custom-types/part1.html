Object Types

Objects are one of the most important concepts in TypeScript.

Almost everything in real-world TypeScript applications revolves around objects:

  • API Responses
  • Database Records
  • React Props
  • Configuration Objects
  • User Models
  • Product Models

Understanding Object Types properly is essential before learning:

  • Type Aliases
  • Interfaces
  • Classes
  • Generics

What is an Object?​

An object is a collection of:

Key → Value Pairs

Example:

const user = {
  name: "Prashant",
  age: 25,
};

Why Do We Need Object Types?​

Consider:

const user = {
  name: "Prashant",
  age: 25,
};

TypeScript automatically infers:

{
  name: string;
  age: number;
}

But sometimes we want to explicitly define the structure.

Inline Object Types​

Syntax​

let variable: {
  property: type;
};

Example​

let user: {
  name: string;
  age: number;
};

Valid:

user = {
  name: "Prashant",
  age: 25,
};

Invalid:

user = {
  name: "Prashant",
};

Error:

Property 'age' is missing

Multiple Properties​

let product: {
  id: number;
  name: string;
  price: number;
};

Valid:

product = {
  id: 1,
  name: "Laptop",
  price: 50000,
};

Property Types Can Differ​

let employee: {
  id: number;
  name: string;
  isActive: boolean;
};

Example:

employee = {
  id: 101,
  name: "John",
  isActive: true,
};

Object Type Checking​

TypeScript checks:

Property Names
Property Types
Required Properties

Example

let user: {
  name: string;
  age: number;
};

Invalid Type

user = {
  name: "Prashant",
  age: "25",
};

Error:

Type 'string'
is not assignable to type 'number'

Nested Objects​

Objects can contain other objects.

Example​

let user: {
  name: string;
  address: {
    city: string;
    pincode: number;
  };
};

Assignment

user = {
  name: "Prashant",
  address: {
    city: "Delhi",
    pincode: 201001,
  },
};

Deeply Nested Objects​

let company: {
  name: string;
  owner: {
    name: string;
    contact: {
      phone: string;
    };
  };
};

Example

company = {
  name: "Tech Corp",
  owner: {
    name: "John",
    contact: {
      phone: "9999999999",
    },
  },
};

Object Types in Functions​

Very common in real projects.

Parameter Object​

function printUser(user: { name: string; age: number }) {
  console.log(user.name);
}

Call

printUser({
  name: "Prashant",
  age: 25,
});

Output:

Prashant

Function Returning Objects​

Example

function createUser(): {
  name: string;
  age: number;
} {
  return {
    name: "Prashant",
    age: 25,
  };
}

Returned Value

{
  name: "Prashant",
  age: 25
}

Problem with Inline Object Types​

Imagine:

function createUser(user: {
  name: string;
  age: number;
  city: string;
  isAdmin: boolean;
}) {}

Repeated everywhere.

This becomes:

Hard to Read
Hard to Maintain
Hard to Reuse

Solution:

Type Aliases
Interfaces

covered in upcoming chapters.

Object Type Inference​

TypeScript can infer object types automatically.

Example

const user = {
  name: "Prashant",
  age: 25,
};

Inferred Type

{
  name: string;
  age: number;
}

Accessing Properties​

Example

const user = {
  name: "Prashant",
  age: 25,
};

Access

console.log(user.name);

Output:

Prashant

Invalid Property Access​

console.log(user.salary);

Error:

Property 'salary'
does not exist

This is one of the biggest benefits of TypeScript.

Duck Typing​

TypeScript follows:

Structural Typing

also known as:

Duck Typing

What Does It Mean?​

TypeScript cares about:

Structure

not:

Object Name

Example

function printUser(user: { name: string }) {
  console.log(user.name);
}

Object

const person = {
  name: "Prashant",
  age: 25,
  city: "Delhi",
};

Valid

printUser(person);

Output:

Prashant

Why?

Because:

Required structure exists.

Structural Typing Example​

Function

function display(item: { id: number }) {}

Object

const product = {
  id: 1,
  name: "Laptop",
  price: 50000,
};

Valid

display(product);

TypeScript only checks:

Does it contain id:number?

Yes.

Therefore valid.

Excess Property Checking​

One of the most common interview questions.

Consider:

function createUser(user: { name: string }) {}

Direct Object Literal

createUser({
  name: "Prashant",
  age: 25,
});

Error.

Object literal may only specify known properties.

Why Error Occurs?​

When passing:

Direct Object Literal

TypeScript performs:

Excess Property Checking

It verifies:

Extra properties not allowed.

Example​

Expected

{
  name: string;
}

Received

{
  name: string;
  age: number;
}

Result:

Compilation Error

Interesting Behavior​

Object Variable

const user = {
  name: "Prashant",
  age: 25,
};

Pass Variable

createUser(user);

Valid.

Why?

Because:

Structural Typing

takes over.

The object contains:

name:string

which satisfies the requirement.

Required vs Extra Properties​

Expected

{
  name: string;
}

Provided

{
  name: string;
}

Valid.

Provided

{
  name: string;
  age: number;
}

Depends on context.

Direct Literal:

Error

Variable:

Valid

Real World Example​

API Response

const user = {
  id: 1,
  name: "Prashant",
  email: "abc@gmail.com",
};

Function

function printUser(user: { name: string }) {
  console.log(user.name);
}

Call

printUser(user);

Valid.

Output

Prashant

Common Mistakes​

Missing Required Property​

let user: {
  name: string;
  age: number;
};

user = {
  name: "Prashant",
};

Error.

Wrong Property Type​

user = {
  name: "Prashant",
  age: "25",
};

Error.

Invalid Property Access​

console.log(user.salary);

Error.

Interview Questions​

Q1​

What is an Object Type?

Answer​

A type that describes the structure
of an object.

Q2​

What is Structural Typing?

Answer​

TypeScript checks object structure
instead of object names.

Q3​

What is Duck Typing?

Answer​

If an object has the required properties,
it is considered compatible.

Q4​

What is Excess Property Checking?

Answer​

Extra property validation performed
on object literals.

Q5​

Difference between:

createUser({
  name: "John",
  age: 20,
});

and

const user = {
  name: "John",
  age: 20,
};

createUser(user);

Answer​

Direct object literals trigger Excess Property Checking.

Variables use Structural Typing.

Cheat Sheet​

let user: {
  name: string;
  age: number;
};
{
  name: "Prashant",
  age: 25
}
function printUser(user: { name: string }) {}
function createUser(): {
  name: string;
  age: number;
} {}
const user = {
  name: "Prashant",
  age: 25,
};

Key Takeaways​

  • Object Types define the structure of objects.
  • TypeScript validates property names and property types.
  • Objects can be nested indefinitely.
  • Object Types can be used in variables, parameters and return values.
  • TypeScript uses Structural Typing (Duck Typing).
  • Compatibility depends on structure, not object names.
  • Direct object literals trigger Excess Property Checking.
  • Object variables are checked using Structural Typing.
  • Inline object types become difficult to maintain in large applications.
  • Type Aliases and Interfaces solve this problem and are the next logical step.

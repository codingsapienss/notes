Optional Properties & Readonly Properties

In real-world applications, not every property is always available and not every property should be allowed to change.

Consider:

  • A user's phone number may not exist yet.
  • A product may not have a discount.
  • A database ID should never change after creation.
  • Configuration values should remain immutable.

TypeScript provides two powerful features for such scenarios:

Optional Properties (?)
Readonly Properties (readonly)

These are heavily used in:

  • APIs
  • React Applications
  • Configuration Objects
  • Domain Models
  • Database Entities

Optional Properties​

What are Optional Properties?​

Normally, every property in an object type is:

Required

Example:

type User = {
  name: string;
  age: number;
};

Valid

const user: User = {
  name: "Prashant",
  age: 25,
};

Invalid

const user: User = {
  name: "Prashant",
};

Error:

Property 'age' is missing

Making Properties Optional​

Use:

?

after the property name.

Syntax​

type User = {
  property?: Type;
};

Example​

type User = {
  name: string;
  age?: number;
};

Now:

const user: User = {
  name: "Prashant",
};

Valid.

Also Valid

const user: User = {
  name: "Prashant",
  age: 25,
};

How Optional Properties Work Internally​

This is a very important concept.

When you write:

type User = {
  age?: number;
};

TypeScript internally treats it as:

type User = {
  age: number | undefined;
};

Meaning:

The property may exist
or
The property may be undefined

Example​

type User = {
  age?: number;
};

const user: User = {};

Access

console.log(user.age);

Output:

undefined

Why Optional Properties Exist​

Imagine a profile form.

User may provide:

Name

but may skip:

Phone Number

Model

type UserProfile = {
  name: string;
  phone?: string;
};

Valid

{
  name: "Prashant";
}

Valid

{
  name: "Prashant",
  phone: "9999999999"
}

Optional Properties in Functions​

Very common.

Example

type User = {
  name: string;
  age?: number;
};
function printUser(user: User) {
  console.log(user.name);
}

Call

printUser({
  name: "Prashant",
});

Valid.

Accessing Optional Properties Safely​

Problem

type User = {
  age?: number;
};

const user: User = {};

Attempt

console.log(user.age.toFixed(2));

Error.

Why?

Because:

age may be undefined

Safe Check​

if (user.age !== undefined) {
  console.log(user.age.toFixed(2));
}

Valid.

Optional Chaining​

Modern TypeScript provides:

?.

Example

console.log(user.age?.toFixed(2));

Behavior

If age exists → execute

If age is undefined → return undefined

Much cleaner.

Multiple Optional Properties​

Example

type User = {
  name: string;
  email?: string;
  phone?: string;
  address?: string;
};

Valid

{
  name: "Prashant";
}

Valid

{
  name: "Prashant",
  phone: "9999999999"
}

Real World Example​

API Response

type User = {
  id: number;
  name: string;
  profileImage?: string;
};

Some users may have:

Profile Picture

Others may not.

Optional properties model this perfectly.

Readonly Properties​

What are Readonly Properties?​

Readonly properties can:

Be Read

but cannot:

Be Modified

after creation.

Syntax​

readonly property: Type;

Example​

type User = {
  readonly id: number;
  name: string;
};

Create Object

const user: User = {
  id: 1,
  name: "Prashant",
};

Read

console.log(user.id);

Valid.

Modify

user.id = 5;

Error.

Cannot assign to readonly property

Why Use Readonly?​

Some values should never change.

Examples:

Database IDs
Order IDs
Product IDs
User IDs
Configuration Values

Example​

type Product = {
  readonly productId: number;
  name: string;
};

Valid

const product = {
  productId: 101,
  name: "Laptop",
};

Invalid

product.productId = 999;

Readonly Does NOT Affect Other Properties​

Example

type User = {
  readonly id: number;
  name: string;
};

Valid

user.name = "John";

Invalid

user.id = 5;

Only readonly properties are protected.

Readonly Arrays​

Very common interview topic.

Normal Array

const numbers: number[] = [1, 2, 3];

Allowed

numbers.push(4);

Allowed

numbers.pop();

Readonly Array​

const numbers: readonly number[] = [1, 2, 3];

Attempt

numbers.push(4);

Error.

Attempt

numbers.pop();

Error.

Attempt

numbers[0] = 100;

Error.

Alternative Syntax​

Equivalent

readonly number[]

and

ReadonlyArray<number>;

Both create immutable arrays.

Example​

const scores: ReadonlyArray<number> = [90, 95, 100];

Invalid

scores.push(80);

Readonly vs const​

One of the most common interview questions.

Many beginners think:

const

and

readonly;

are identical.

They are not.

const​

Protects:

Variable Reassignment

Example

const user = {
  name: "Prashant",
};

Invalid

user = {};

But:

user.name = "John";

Valid.

readonly​

Protects:

Property Modification

Example

type User = {
  readonly name: string;
};

Invalid

user.name = "John";

Comparison​

Featureconstreadonly
Protect Variable✅❌
Protect Property❌✅
Used On Variables✅❌
Used On Properties❌✅

Combining Optional and Readonly​

Example

type User = {
  readonly id: number;
  name: string;
  email?: string;
};

Valid

const user: User = {
  id: 1,
  name: "Prashant",
};

Invalid

user.id = 2;

Valid

user.name = "John";

Valid

user.email = "abc@gmail.com";

Real World Example​

Database Entity

type User = {
  readonly id: number;
  name: string;
  email?: string;
};

Meaning:

id

Never changes.

email

May not exist.

Perfect real-world modeling.

Common Mistakes​

Forgetting Undefined Check​

Wrong

user.age.toFixed(2);

Correct

user.age?.toFixed(2);

or

if(user.age !== undefined)

Thinking Readonly Exists at Runtime​

Important.

TypeScript:

readonly;

works only at:

Compile Time

Generated JavaScript:

No readonly protection exists.

Confusing const with readonly​

Remember:

const → variable

readonly → property

Interview Questions​

Q1​

What does ? mean in TypeScript?

Answer​

Optional Property

Q2​

How does TypeScript internally treat:

age?: number

Answer​

age: number | undefined;

Q3​

What does readonly do?

Answer​

Prevents property reassignment.

Q4​

Difference between const and readonly?

Answer​

const protects variables.

readonly protects properties.

Q5​

How do you create a readonly array?

Answer​

readonly number[]

or

ReadonlyArray<number>;

Cheat Sheet​

age?: number;

↓

age: number | undefined;
readonly id: number;
user.age?.toFixed(2);
readonly number[]
ReadonlyArray<number>;

Key Takeaways​

  • Optional properties use the ? operator.
  • Optional properties may be missing or undefined.
  • Internally, optional properties behave like a union with undefined.
  • Always safely access optional properties.
  • Readonly properties cannot be reassigned after creation.
  • Readonly is useful for IDs and immutable data.
  • Readonly arrays prevent modification operations.
  • const and readonly solve different problems.
  • Both optional and readonly properties are extremely common in production TypeScript code.
  • These concepts form the foundation for Interfaces, Classes and API Modeling.

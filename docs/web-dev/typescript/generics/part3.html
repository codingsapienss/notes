Generic Classes

Now we'll learn one of the most powerful applications of Generics:

Generic Classes

Generic Classes allow us to create:

  • Reusable Classes
  • Type-Safe Containers
  • Generic Data Structures
  • Repositories
  • Service Layers
  • Cache Systems

without sacrificing type safety.

Why Do We Need Generic Classes?​

Suppose we want a class that stores data.

Without Generics

class StringStorage {
  private data: string[] = [];

  add(item: string) {
    this.data.push(item);
  }

  getAll() {
    return this.data;
  }
}

Usage

const users = new StringStorage();

users.add("Prashant");

Works.

Now we need:

Number Storage
Boolean Storage
User Storage
Product Storage

Without Generics

class NumberStorage {}

class BooleanStorage {}

class UserStorage {}

class ProductStorage {}

Problem:

Code Duplication

Generic Class Solution​

Instead of creating multiple classes:

StringStorage;

NumberStorage;

UserStorage;

Create:

Storage<T>;

One class.

Many types.

What is a Generic Class?​

A Generic Class is:

A Class That Accepts
Type Parameters

Syntax

class ClassName<T> {}

Here:

T;

represents:

A Placeholder Type

First Generic Class​

class Storage<T> {
  private data: T[] = [];

  add(item: T) {
    this.data.push(item);
  }

  getAll() {
    return this.data;
  }
}

Notice:

T[]

The class can now store:

Strings
Numbers
Objects
Custom Types

Using Generic Classes​

String Storage

const users = new Storage<string>();

Add Data

users.add("Prashant");

users.add("John");

Result

["Prashant", "John"];

TypeScript knows:

Storage contains strings

Number Storage​

const scores = new Storage<number>();

Usage

scores.add(100);

scores.add(95);

Result

[100, 95];

Type-safe.

Type Safety in Action​

String Storage

const users = new Storage<string>();

Valid

users.add("Prashant");

Invalid

users.add(100);

Error

Argument of type number
is not assignable to string

Because:

T = string;

Generic Class with Objects​

User Type

type User = {
  id: number;

  name: string;
};

Storage

const userStorage = new Storage<User>();

Usage

userStorage.add({
  id: 1,
  name: "Prashant",
});

Valid.

TypeScript knows:

Storage contains User objects

Generic Class Internally​

Consider:

const users = new Storage<string>();

TypeScript treats it as:

Storage<string>;

Internally:

class Storage {
  private data: string[];
}

Conceptually.

Multiple Generic Parameters​

Classes can have multiple type parameters.

Syntax

class Pair<T, U> {}

Example

class Pair<T, U> {
  constructor(
    public first: T,
    public second: U,
  ) {}
}

Usage

const pair = new Pair<string, number>("Age", 25);

Result

{
  first: string;
  second: number;
}

Generic Key-Value Store​

Example

class KeyValue<K, V> {
  constructor(
    public key: K,
    public value: V,
  ) {}
}

Usage

const item = new KeyValue<string, number>("age", 25);

Result

{
  key: "age",
  value: 25
}

Constructor with Generics​

Generics can be used inside constructors.

Example

class Box<T> {
  constructor(public value: T) {}
}

Usage

const box = new Box<string>("Hello");

Output

box.value;

Type:

string;

Generic Class Methods​

Methods automatically gain access to class generics.

Example

class Box<T> {
  constructor(public value: T) {}

  getValue(): T {
    return this.value;
  }
}

Usage

const box = new Box<number>(100);

console.log(box.getValue());

Output

100

Return Type:

number;

Generic Repository Pattern​

One of the most common enterprise patterns.

Entity

type User = {
  id: number;

  name: string;
};

Repository

class Repository<T> {
  private items: T[] = [];

  add(item: T) {
    this.items.push(item);
  }

  getAll() {
    return this.items;
  }
}

Usage

const users = new Repository<User>();

Add User

users.add({
  id: 1,
  name: "Prashant",
});

Retrieve

users.getAll();

Type:

User[]

Very common in:

Backend Systems
ORMs
Repositories
Services

Generic Cache Example​

class Cache<T> {
  private data: Record<string, T> = {};

  set(key: string, value: T) {
    this.data[key] = value;
  }

  get(key: string) {
    return this.data[key];
  }
}

Usage

const userCache = new Cache<string>();

userCache.set("username", "Prashant");

Retrieve

userCache.get("username");

Type:

string;

Generic Class + Generic Method​

A class can have:

Class Generics

and

Method Generics

simultaneously.

Example

class Storage<T> {
  private data: T[] = [];

  add(item: T) {
    this.data.push(item);
  }

  convert<U>(value: U): U {
    return value;
  }
}

Usage

const store = new Storage<string>();

Class Generic

T = string;

Method Generic

U = number;

Independent.

Generic Class vs Generic Interface​

Generic Interface

interface Box<T> {
  value: T;
}

Describes:

Structure

Generic Class

class Box<T> {
  constructor(public value: T) {}
}

Provides:

Implementation

Real World Example​

API Service

Response Type

type User = {
  id: number;

  name: string;
};

Service

class ApiService<T> {
  constructor(private data: T) {}

  getData(): T {
    return this.data;
  }
}

Usage

const service = new ApiService<User>({
  id: 1,
  name: "Prashant",
});

Retrieve

service.getData();

Type:

User;

Common Mistakes​

Using any Instead of Generics​

Bad

class Storage {
  data: any[];
}

Problem:

Type Safety Lost

Prefer

class Storage<T>

Creating Duplicate Classes​

Bad

StringStorage;

NumberStorage;

UserStorage;

Use:

Storage<T>;

instead.

Forgetting Generic Type​

Wrong

const store = new Storage();

May result in:

unknown;

or weak inference.

Prefer:

new Storage<string>();

Cheat Sheet​

class Storage<T> {}
new Storage<string>();
new Storage<number>();
class Pair<T,U>
class Repository<T>

getValue(): T

Key Takeaways​

  • Generic Classes allow classes to work with multiple data types.
  • They provide reusability while maintaining type safety.
  • T acts as a type placeholder.
  • Generic Classes eliminate duplicate class implementations.
  • Multiple generic parameters are supported.
  • Constructors, properties and methods can all use generic types.
  • Generic Classes are widely used in repositories, services, caches and frameworks.
  • Generic Classes form the foundation of many enterprise TypeScript architectures.
  • Prefer Generic Classes over any whenever type information should be preserved.
  • Understanding Generic Classes is essential before learning Generic Constraints and Advanced Generics.

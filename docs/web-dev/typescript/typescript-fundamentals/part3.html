Functions

Functions are one of the most important concepts in TypeScript.

TypeScript enhances JavaScript functions by adding:

  • Parameter Types
  • Return Types
  • Optional Parameters
  • Default Parameters
  • Function Signatures
  • Better Type Safety

Understanding functions properly is essential before learning:

  • Interfaces
  • Classes
  • Generics
  • React Components
  • API Development

What is a Function?​

A function is a reusable block of code designed to perform a specific task.

Example​

function greet() {
  console.log("Hello World");
}

Calling the function:

greet();

Output:

Hello World

Why Do We Need Functions?​

Without functions:

console.log("Welcome");
console.log("Welcome");
console.log("Welcome");

Repeated code.

With functions:

function welcome() {
  console.log("Welcome");
}

welcome();
welcome();
welcome();

Reusable and cleaner.

Function Syntax​

function functionName(parameters): ReturnType {
  // code
}

Example​

function add(a: number, b: number): number {
  return a + b;
}

Anatomy of a Function​

function add(a: number, b: number): number {
  return a + b;
}

Parameter Types​

TypeScript allows explicit parameter typing.

Example​

function greet(name: string) {
  console.log(name);
}

Valid:

greet("Prashant");

Invalid:

greet(100);

Error:

Argument of type 'number'
is not assignable to parameter of type 'string'

Multiple Parameters​

function add(a: number, b: number) {
  return a + b;
}

Valid:

add(10, 20);

Output:

30

Invalid:

add("10", 20);

Compilation Error.

Return Types​

Functions may return values.

Example​

function add(a: number, b: number): number {
  return a + b;
}

Return Type:

number;

Why Specify Return Types?​

TypeScript can infer return types.

However, explicit return types:

  • Improve readability
  • Improve maintainability
  • Catch mistakes earlier

Example​

function multiply(a: number, b: number): number {
  return a * b;
}

Type Inference for Returns​

function multiply(a: number, b: number) {
  return a * b;
}

TypeScript automatically infers:

number;

Void Functions​

Some functions return nothing.

Example​

function printMessage(): void {
  console.log("Hello");
}

Return Type:

void

Meaning:

No value returned

Example​

function greet(): void {
  console.log("Welcome");
}

What Happens if We Return Something?​

function greet(): void {
  return "hello";
}

Error:

Type 'string'
is not assignable to type 'void'

Never Type​

One of the most misunderstood types.

What is never?​

Represents:

A function that never successfully finishes.

Example 1​

Infinite Loop

function runForever(): never {
  while (true) {}
}

Function never exits.

Example 2​

Always Throws Error

function throwError(): never {
  throw new Error("Something went wrong");
}

Execution stops immediately.

Difference Between void and never​

Featurevoidnever
Returns ValueNoNo
Function FinishesYesNever
Execution Ends NormallyYesNo

void Example​

function print(): void {
  console.log("Hello");
}

Returns:

Nothing

but function finishes.

never Example​

function crash(): never {
  throw new Error();
}

Function never completes.

Optional Parameters​

Sometimes parameters are not required.

Syntax​

parameter?: Type

Example​

function greet(name?: string) {
  console.log(name);
}

Valid

greet();

Valid

greet("Prashant");

What is the Type Internally?​

name?: string

becomes:

string | undefined;

Internally.

Example​

function greet(name?: string) {
  if (name) {
    console.log(name);
  }
}

Multiple Optional Parameters​

function createUser(name: string, age?: number) {}

Valid

createUser("Prashant");

Valid

createUser("Prashant", 25);

Important Rule​

Optional parameters should come after required parameters.

Correct

function test(name: string, age?: number) {}

Incorrect

function test(age?: number, name: string) {}

Compilation Error.

Default Parameters​

Default values automatically apply when no value is provided.

Example​

function greet(name: string = "Guest") {
  console.log(name);
}

Call

greet();

Output:

Guest

Call

greet("Prashant");

Output:

Prashant

Optional vs Default Parameters​

Optional

function greet(name?: string) {}

Value:

undefined;

if not supplied.

Default

function greet(name = "Guest") {}

Value:

"Guest";

if not supplied.

Function Expressions​

Functions can be stored inside variables.

Example​

const add = function (a: number, b: number): number {
  return a + b;
};

Usage

add(10, 20);

Output:

30

Arrow Functions​

Modern JavaScript and TypeScript heavily use Arrow Functions.

Traditional Function​

function add(a: number, b: number): number {
  return a + b;
}

Arrow Version​

const add = (a: number, b: number): number => {
  return a + b;
};

Short Arrow Syntax​

const add = (a: number, b: number): number => a + b;

Example​

const square = (value: number): number => value * value;

Output

square(5);
25

Contextual Typing​

TypeScript can infer parameter types based on context.

Example​

const names = ["Prashant", "John", "Alex"];

names.forEach((name) => {
  console.log(name);
});

TypeScript automatically infers:

name: string;

No explicit annotation required.

Function Type Annotation​

Functions themselves have types.

Example

let add: (a: number, b: number) => number;

Assign Function

add = (a, b) => a + b;

Valid.

Callback Functions​

Functions passed as arguments.

Example

function processUser(callback: () => void) {
  callback();
}

Usage

processUser(() => {
  console.log("Processing");
});

Output

Processing

Rest Parameters​

Accept multiple values.

Example​

function sum(...numbers: number[]): number {
  return numbers.reduce((a, b) => a + b, 0);
}

Call

sum(1, 2, 3, 4);

Output:

10

Common Mistakes​

Missing Return​

function add(a: number, b: number): number {}

Error:

Function lacks ending return statement.

Wrong Parameter Type​

function greet(name: string) {}

greet(100);

Error.

Returning Wrong Type​

function getAge(): number {
  return "25";
}

Error.

Interview Questions​

Q1​

Difference between void and never?

Answer​

void:
Function finishes normally.

never:
Function never completes.

Q2​

What is an optional parameter?

Answer​

name?: string

Parameter may be omitted.

Q3​

What is a default parameter?

Answer​

A parameter with a predefined value.

Q4​

What are arrow functions?

Answer​

A shorter syntax for writing functions.

Q5​

Can TypeScript infer return types?

Answer​

Yes.

TypeScript can automatically infer return types.

Cheat Sheet​

function add(a: number, b: number): number {
  return a + b;
}
function print(): void {}
function crash(): never {
  throw new Error();
}
function greet(name?: string) {}
function greet(name = "Guest") {}
const add = (a: number, b: number): number => a + b;
function sum(...nums: number[]) {}

Key Takeaways​

  • Functions are reusable blocks of code.
  • TypeScript adds type safety to function parameters and return values.
  • Return types can be explicit or inferred.
  • void means no value is returned.
  • never means execution never successfully completes.
  • Optional parameters use ?.
  • Default parameters provide fallback values.
  • Arrow Functions are widely used in modern TypeScript.
  • Contextual Typing allows TypeScript to infer parameter types automatically.
  • Functions themselves can have types.

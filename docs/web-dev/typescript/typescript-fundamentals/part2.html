Primitive Types, Type Annotations & Type Inference

Everything in TypeScript starts with types.

Before learning:

  • Objects
  • Interfaces
  • Classes
  • Generics
  • Advanced Type Manipulation

you must understand:

Primitive Types

because every complex type is ultimately built from primitive types.

What are Primitive Types?​

Primitive Types are the most basic data types available in TypeScript.

They represent:

Single Values

Examples:

string;
number;
boolean;
null;
undefined;
symbol;
bigint;

Primitive Types in TypeScript​

Type Annotation​

Type Annotation means:

Explicitly telling TypeScript
what type a variable should store.

Syntax​

let variableName: Type = value;

Example​

let username: string = "Prashant";

Here:

Variable = username

Type = string

Value = "Prashant"

Why Use Type Annotations?​

Without TypeScript:

let age = 25;

age = "hello";

Valid JavaScript.

With TypeScript:

let age: number = 25;

age = "hello";

Error:

Type 'string' is not assignable to type 'number'

String Type​

Represents text values.

Example​

let firstName: string = "Prashant";

Valid Values​

let city: string = "Delhi";

let company: string = "OpenAI";

Invalid​

let city: string = 100;

Error:

Type 'number' is not assignable to type 'string'

String Methods​

TypeScript automatically understands available methods.

let name: string = "prashant";

console.log(name.toUpperCase());

Output:

PRASHANT

IntelliSense Benefit​

TypeScript knows:

name is a string

therefore suggests:

toUpperCase();
toLowerCase();
trim();
includes();
replace();

Number Type​

Represents:

Integers
Floats
Negative Numbers
Infinity
NaN

Unlike some languages:

number;

handles all numeric values.

Example​

let age: number = 25;

Decimal​

let price: number = 99.99;

Negative​

let balance: number = -500;

Scientific Notation​

let population: number = 1.5e9;

Special Number Values​

Infinity​

let value: number = Infinity;

NaN​

let result: number = NaN;

Example:

console.log(0 / 0);

Output:

NaN

Boolean Type​

Represents:

true
false

only.

Example​

let isLoggedIn: boolean = true;
let isAdmin: boolean = false;

Common Usage​

if (isLoggedIn) {
  console.log("Welcome");
}

Undefined Type​

Represents:

Value not assigned yet

Example​

let username: undefined = undefined;

More commonly:

let username: string | undefined;

covered later in Union Types.

Null Type​

Represents:

Intentional absence of value

Example​

let selectedUser: null = null;

let user = null;

BigInt Type​

Used for very large integers.

Problem with number​

JavaScript numbers become unsafe after:

9007199254740991

Example

console.log(9007199254740992);
console.log(9007199254740993);

Unexpected results may occur.

BigInt Solution​

let largeNumber: bigint = 9007199254740993n;

Notice:

n suffix

Example​

let population: bigint = 9999999999999999999n;

BigInt Rules​

Valid

10n + 20n;

Invalid

10n + 20;

Error.

Must convert first.

BigInt(20);

Symbol Type​

One of the least used primitive types.

Used to create:

Unique Identifiers

Example​

const id1 = Symbol();
const id2 = Symbol();

Comparison

console.log(id1 === id2);

Output:

false

Even though both were created the same way.

Why Symbols Exist?​

Avoid key collisions.

Example

const id = Symbol("id");

const user = {
  [id]: 123,
};

Type Inference​

One of TypeScript's most powerful features.

What is Type Inference?​

TypeScript can automatically determine the type.

Instead of:

let username: string = "Prashant";

you can write:

let username = "Prashant";

TypeScript infers:

string

automatically.

Example​

let age = 25;

TypeScript infers:

number;

Now:

age = "hello";

Error.

Inference Examples​

String​

let city = "Delhi";

Inference:

string;

Number​

let score = 100;

Inference:

number;

Boolean​

let isActive = true;

Inference:

boolean;

When Should You Use Explicit Types?​

Use explicit annotations when:

Function Parameters​

function greet(name: string) {}

API Models​

type User = {
  id: number;
  name: string;
};

Public Interfaces​

interface Product {
  id: number;
}

When Can You Rely on Inference?​

Local Variables​

let count = 10;

Simple Values​

const city = "Delhi";

let vs const Inference​

Important Interview Topic.

let​

let city = "Delhi";

Type:

string;

Any string allowed later.

city = "Mumbai";

Valid.

const​

const city = "Delhi";

Type becomes:

"Delhi";

Literal Type.

Cannot reassign.

city = "Mumbai";

Error.

Type Widening​

Example​

let status = "active";

TypeScript widens:

string;

because value may change later.

Literal Type Inference​

const status = "active";

Type:

"active";

not:

string;

Common Mistakes​

Mixing Types​

let age: number = 25;

age = "twenty five";

Error.

Using BigInt with Number​

Wrong

10n + 20;

Correct

10n + BigInt(20);

Confusing null and undefined​

Wrong assumption:

Both are identical.

They are different values.

Interview Questions​

Q1​

What are Primitive Types?

Answer​

The basic built-in data types such as:
string, number, boolean, null,
undefined, bigint and symbol.

Q2​

What is Type Annotation?

Answer​

Explicitly specifying the type of a variable.

Q3​

What is Type Inference?

Answer​

TypeScript automatically determining a type.

Q4​

Difference between null and undefined?

Answer​

null → intentional absence of value

undefined → value not assigned

Q5​

Why was bigint introduced?

Answer​

To safely represent integers larger than
Number.MAX_SAFE_INTEGER.

Cheat Sheet​

let name: string = "Prashant";
let age: number = 25;
let isAdmin: boolean = false;
let value: null = null;
let value: undefined = undefined;
let bigNumber: bigint = 123n;
const id = Symbol();
let city = "Delhi"; // inferred string

Key Takeaways​

  • Primitive Types are the foundation of TypeScript's type system.
  • Type Annotation explicitly defines a type.
  • Type Inference automatically determines a type.
  • string, number, and boolean are the most commonly used primitive types.
  • null and undefined represent different concepts.
  • bigint is used for very large integers.
  • symbol provides unique identifiers.
  • TypeScript can infer types in many situations.
  • Prefer inference for simple local variables.
  • Use explicit annotations for public APIs, function parameters, and shared types.

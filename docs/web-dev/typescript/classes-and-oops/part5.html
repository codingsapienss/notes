Interfaces with Classes

In previous chapters, we learned:

  • Classes
  • Access Modifiers
  • Getters & Setters
  • Inheritance

However, inheritance is not always the best way to define object behavior.

Sometimes we want to say:

Any class that wants to be a Vehicle
must implement these methods.

without providing any implementation.

This is where:

interface;

and

implements;

become extremely useful.

This chapter covers:

  • Interfaces with Classes
  • implements Keyword
  • Multiple Interfaces
  • Polymorphism

What is an Interface?​

An Interface defines:

A Contract

that a class must follow.

Example:

interface Animal {
  makeSound(): void;
}

Meaning:

Any class implementing Animal
must provide makeSound().

Notice:

interface Animal {
  makeSound(): void;
}

contains:

What should exist

NOT

How it should work

Why Do We Need Interfaces?​

Suppose we have:

Dog
Cat
Lion

All should have:

makeSound();

But:

Implementation differs

Dog:

Woof

Cat:

Meow

Lion:

Roar

An Interface helps define:

Common Structure

without forcing implementation.

implements Keyword​

A class uses:

implements;

to follow an interface contract.

Basic Example​

Interface

interface Animal {
  makeSound(): void;
}

Class

class Dog implements Animal {
  makeSound() {
    console.log("Woof!");
  }
}

Usage

const dog = new Dog();

dog.makeSound();

Output

Woof!

What Happens Internally?​

Interface

interface Animal {
  makeSound(): void;
}

tells TypeScript:

Any implementing class
must define makeSound()

If Dog forgets:

class Dog implements Animal {}

Error

Class 'Dog'
incorrectly implements interface 'Animal'

Because:

Contract Broken

Interface with Properties​

Interfaces can define properties.

Example

interface User {
  id: number;

  name: string;
}

Class

class Employee implements User {
  constructor(
    public id: number,
    public name: string,
  ) {}
}

Usage

const emp = new Employee(1, "Prashant");

Output

Employee {
 id: 1,
 name: "Prashant"
}

Interface with Methods​

Example

interface Shape {
  area(): number;
}

Class

class Rectangle implements Shape {
  constructor(
    public width: number,
    public height: number,
  ) {}

  area() {
    return this.width * this.height;
  }
}

Usage

const rect = new Rectangle(10, 5);

console.log(rect.area());

Output

50

Interface with Multiple Members​

Example

interface Vehicle {
  brand: string;

  start(): void;

  stop(): void;
}

Implementation

class Car implements Vehicle {
  constructor(public brand: string) {}

  start() {
    console.log("Car Started");
  }

  stop() {
    console.log("Car Stopped");
  }
}

Usage

const car = new Car("BMW");

car.start();

Output

Car Started

Class Can Implement Multiple Interfaces​

Unlike inheritance:

extends

which supports:

Single Inheritance

Interfaces support:

Multiple Contracts

Example​

Interface 1

interface Flyable {
  fly(): void;
}

Interface 2

interface Swimmable {
  swim(): void;
}

Class

class Duck implements Flyable, Swimmable {
  fly() {
    console.log("Flying");
  }

  swim() {
    console.log("Swimming");
  }
}

Usage

const duck = new Duck();

duck.fly();

duck.swim();

Output

Flying
Swimming

Why Multiple Interfaces Matter?​

Real World Example:

User

might be:

Serializable
Loggable
Cacheable

all at the same time.

Example

class User
implements
  Serializable,
  Loggable,
  Cacheable

A class can satisfy multiple behaviors.

extends vs implements​

One of the most important interview topics.

extends​

Used for:

Inheritance

Example

class Dog
extends Animal

Meaning:

Dog IS-A Animal

Dog inherits:

Properties
Methods
Implementation

implements​

Used for:

Contracts

Example

class Dog
implements Animal

Meaning:

Dog follows Animal rules

Dog inherits:

Nothing

Must implement everything itself.

Comparison Table​

PurposeInheritanceContract
Reuse Code✅❌
Inherit Methods✅❌

Interface + Inheritance Together​

Very common.

Example

class Animal {
  eat() {
    console.log("Eating");
  }
}

Interface

interface Flyable {
  fly(): void;
}

Class

class Bird extends Animal implements Flyable {
  fly() {
    console.log("Flying");
  }
}

Usage

const bird = new Bird();

bird.eat();

bird.fly();

Output

Eating
Flying

Bird gets:

eat()

from:

Animal

and satisfies:

fly()

from:

Flyable

What is Polymorphism?​

One of the four pillars of OOP.

Word Meaning:

Poly = Many

Morph = Forms

Meaning:

Same Interface
Different Implementations

Example​

Interface

interface Animal {
  makeSound(): void;
}

Classes

class Dog implements Animal {
  makeSound() {
    console.log("Woof");
  }
}
class Cat implements Animal {
  makeSound() {
    console.log("Meow");
  }
}
class Lion implements Animal {
  makeSound() {
    console.log("Roar");
  }
}

All classes:

Look the same externally

But behave differently.

Polymorphic Function​

function playSound(animal: Animal) {
  animal.makeSound();
}

Usage

playSound(new Dog());

Output

Woof

Usage

playSound(new Cat());

Output

Meow

Usage

playSound(new Lion());

Output

Roar

Same function.

Different behavior.

This is:

Runtime Polymorphism

Why Polymorphism Matters?​

Without polymorphism:

if (type === "dog") {
}

if (type === "cat") {
}

if (type === "lion") {
}

Huge code duplication.

With polymorphism:

animal.makeSound();

Clean.

Flexible.

Extensible.

Real World Example​

Payment Systems

Interface

interface Payment {
  pay(amount: number): void;
}

Implementation

class CreditCard implements Payment {
  pay(amount: number) {
    console.log(
      `Paid ${amount}
       via Credit Card`,
    );
  }
}
class PayPal implements Payment {
  pay(amount: number) {
    console.log(
      `Paid ${amount}
       via PayPal`,
    );
  }
}

Function

function processPayment(payment: Payment) {
  payment.pay(100);
}

Usage

processPayment(new CreditCard());

Output

Paid 100 via Credit Card

Usage

processPayment(new PayPal());

Output

Paid 100 via PayPal

Common Mistakes​

Confusing extends and implements​

Wrong Thinking:

implements inherits code

No.

implements only enforces rules

Forgetting Required Members​

Wrong

class Dog implements Animal {}

Must implement:

makeSound();

Using Interfaces for Code Reuse​

Interfaces provide:

Structure

not

Implementation

Use:

extends

for code reuse.

Interview Questions​

Q1​

What is an Interface?

Answer​

A contract that defines
the structure a class
must follow.

Q2​

What does implements do?

Answer​

Makes a class satisfy
an interface contract.

Q3​

Difference between extends and implements?

Answer​

extends:
Inheritance

implements:
Contract enforcement

Q4​

Can a class implement multiple interfaces?

Answer​

Yes

Q5​

What is Polymorphism?

Answer​

Same interface
Different implementations

Cheat Sheet​

  makeSound(): void;
}

  fly(): void;
}
implements;

extends

Key Takeaways​

  • Interfaces define contracts, not implementations.
  • Classes use implements to satisfy interface requirements.
  • Interfaces can contain properties and methods.
  • A class can implement multiple interfaces.
  • implements does not provide code reuse.
  • extends provides inheritance and code reuse.
  • Polymorphism allows the same interface to have multiple implementations.
  • Interfaces are heavily used in enterprise TypeScript applications.
  • Polymorphism reduces conditional logic and improves extensibility.
  • Interfaces and Classes together form the foundation of scalable OOP design.

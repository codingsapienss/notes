OOP Design Concepts

In previous chapters, we learned:

  • Classes
  • Access Modifiers
  • Getters & Setters
  • Inheritance
  • Interfaces
  • Abstract Classes

These are language features.

However, writing good software requires understanding:

How to Design Systems Properly

This chapter covers the most important Object-Oriented Design concepts:

  • Encapsulation
  • Abstraction
  • Inheritance
  • Polymorphism
  • Composition over Inheritance

These concepts are frequently discussed in:

  • Interviews
  • System Design
  • Enterprise Applications
  • Framework Design

The Four Pillars of OOP​

Object-Oriented Programming is built upon:

1. Encapsulation
2. Abstraction
3. Inheritance
4. Polymorphism

1. Encapsulation​

What is Encapsulation?​

Encapsulation means:

Bundling Data
+
Methods That Operate On Data
Inside A Single Unit

That unit is usually:

A Class

Real World Example​

Without Encapsulation:

let balance = 1000;

balance = -50000;

Problem:

Anyone can modify balance
Invalid state possible

With Encapsulation:

class BankAccount {
  private balance = 0;

  deposit(amount: number) {
    this.balance += amount;
  }

  getBalance() {
    return this.balance;
  }
}

Usage:

const account = new BankAccount();

account.deposit(1000);

console.log(account.getBalance());

Output:

1000

Now:

account.balance;

is inaccessible.

Benefits​

Data Protection
Controlled Access
Reduced Bugs
Better Maintainability

2. Abstraction​

What is Abstraction?​

Abstraction means:

Show What
Hide How

Users should see:

What a system does

not

How it works internally

Real World Example​

Using a Car:

Steering
Brake
Accelerator

You don't need to know:

Fuel Injection
Engine Timing
Combustion Process

Those details are hidden.

Example​

Interface:

interface Payment {
  pay(amount: number): void;
}

Implementation:

class CreditCard implements Payment {
  pay(amount: number) {
    console.log(`Paid ${amount}`);
  }
}

Usage:

const payment = new CreditCard();

payment.pay(100);

User only sees:

pay();

Not:

API Calls
Encryption
Database Updates

Benefits​

Reduced Complexity
Cleaner APIs
Loose Coupling
Better Flexibility

3. Inheritance​

What is Inheritance?​

Inheritance allows:

One Class
to acquire
properties and methods
from another class.

Example:

class Animal {
  eat() {
    console.log("Eating");
  }
}
class Dog extends Animal {}

Usage:

const dog = new Dog();

dog.eat();

Output:

Eating

Dog inherited:

eat()

from:

Animal

Benefits​

Code Reuse
Reduced Duplication
Hierarchical Relationships

Drawbacks​

Overusing inheritance creates:

Deep Hierarchies
Tight Coupling
Complex Systems

Example:

Animal
  ↓
FlyingAnimal
  ↓
FlyingSwimmingAnimal
  ↓
FlyingSwimmingTalkingAnimal

This becomes difficult to maintain.

4. Polymorphism​

What is Polymorphism?​

Word Meaning:

Poly  = Many

Morph = Forms

Meaning:

Same Interface
Different Behaviors

Example​

Interface:

interface Animal {
  makeSound(): void;
}

Implementation:

class Dog implements Animal {
  makeSound() {
    console.log("Woof");
  }
}
class Cat implements Animal {
  makeSound() {
    console.log("Meow");
  }
}

Function:

function playSound(animal: Animal) {
  animal.makeSound();
}

Usage:

playSound(new Dog());

Output:

Woof

Usage:

playSound(new Cat());

Output:

Meow

Same function.

Different behavior.

This is:

Polymorphism

Benefits​

Flexible Systems
Reusable Logic
Less Conditional Code
Easy Extensibility

The Biggest Modern OOP Principle​

Composition Over Inheritance​

What is Composition?​

Composition means:

Building Objects
Using Smaller Objects

instead of:

Large Inheritance Trees

Traditional Inheritance Approach​

Example:

class Animal {
  eat() {}
}
class FlyingAnimal extends Animal {
  fly() {}
}
class FlyingSwimmingAnimal extends FlyingAnimal {
  swim() {}
}

Problem:

Deep Class Hierarchies
Tight Coupling
Hard To Change

Composition Approach​

Create behaviors separately.

class FlyBehavior {
  fly() {
    console.log("Flying");
  }
}
class SwimBehavior {
  swim() {
    console.log("Swimming");
  }
}
class Duck {
  constructor(
    public flyBehavior: FlyBehavior,

    public swimBehavior: SwimBehavior,
  ) {}
}

Usage:

const duck = new Duck(new FlyBehavior(), new SwimBehavior());

duck.flyBehavior.fly();

duck.swimBehavior.swim();

Output:

Flying
Swimming

Why Composition is Better?​

Instead of:

IS-A Relationship

Inheritance focuses on:

Dog IS-A Animal

Composition focuses on:

HAS-A Relationship

Example:

Duck HAS-A FlyBehavior
Duck HAS-A SwimBehavior

This is usually more flexible.

Real World Example​

React Components

React does NOT use:

Deep Inheritance Trees

Instead:

Component Composition

Example:

<Card>
  <Header />
  <Body />
  <Footer />
</Card>

This is:

Composition

Another Example​

Car System

Bad Design

class Vehicle {}

class Car extends Vehicle {}

class SportsCar extends Car {}

class RacingSportsCar extends SportsCar {}

Good Design

class Engine {}

class GPS {}

class Turbo {}
class Car {
  constructor(
    public engine: Engine,
    public gps: GPS,
    public turbo: Turbo,
  ) {}
}

More flexible.

When To Use What?​

Use Encapsulation When​

Protecting Internal State

Examples:

Bank Accounts
User Passwords
Tokens

Use Abstraction When​

Complex Logic
Must Be Hidden

Examples:

Payment Systems
Database Access
API Clients

Use Inheritance When​

True IS-A Relationship Exists

Examples:

Dog → Animal
Car → Vehicle

Use Polymorphism When​

Same Action
Different Implementations

Examples:

Payment Methods
Animals
Notifications

Use Composition When​

Behavior Changes Frequently

Examples:

React Components
Game Objects
Large Applications

Modern Industry Recommendation​

Historically:

Inheritance First

Modern Software Design:

Composition First

Reason:

Less Coupling
Better Reusability
More Flexible Systems

Interview Questions​

Q1​

What are the four pillars of OOP?

Answer​

Encapsulation
Abstraction
Inheritance
Polymorphism

Q2​

What is Encapsulation?

Answer​

Bundling data and methods
while protecting internal state.

Q3​

What is Abstraction?

Answer​

Showing important details
while hiding implementation.

Q4​

What is Polymorphism?

Answer​

Same interface,
different implementations.

Q5​

What is Composition?

Answer​

Building objects using
other objects instead of
deep inheritance.

Q6​

Why is Composition preferred over Inheritance?

Answer​

More flexible,
less coupled,
easier to maintain.

Cheat Sheet​

Encapsulation
=
Protect Data
Abstraction
=

Inheritance
=

Composition

Key Takeaways​

  • Encapsulation protects internal state.
  • Abstraction hides implementation details.
  • Inheritance enables code reuse through IS-A relationships.
  • Polymorphism enables multiple implementations behind a common interface.
  • Composition builds objects from smaller reusable pieces.
  • Modern TypeScript applications prefer Composition over deep inheritance.
  • Good software design focuses on flexibility and maintainability rather than excessive class hierarchies.
  • Understanding these principles is more important than memorizing syntax.
  • These concepts appear frequently in interviews and real-world system design discussions.

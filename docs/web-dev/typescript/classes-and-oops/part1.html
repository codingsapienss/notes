Classes Fundamentals

Classes are one of the core Object-Oriented Programming (OOP) features in TypeScript.

They allow us to model real-world entities using:

  • Properties (Data)
  • Methods (Behavior)
  • Constructors (Initialization)

Classes help organize code and are heavily used in:

  • Backend Development
  • Enterprise Applications
  • Design Patterns
  • Angular
  • NestJS
  • OOP-based Architectures

What is a Class?​

A Class is a blueprint for creating objects.

Think of it as:

Class  → Blueprint

Object → Actual Instance

Real World Example​

Blueprint of House
        ↓
Actual House

Similarly:

Class User
       ↓
User Object

Why Do We Need Classes?​

Without Classes:

const user1 = {
  name: "Prashant",
  age: 25,
  greet() {
    console.log("Hello");
  },
};

const user2 = {
  name: "John",
  age: 30,
  greet() {
    console.log("Hello");
  },
};

Problem:

Code Duplication
Hard Maintenance
Poor Scalability

With Classes:

class User {
  name: string;
  age: number;

  greet() {
    console.log("Hello");
  }
}

Create multiple users easily.

Class Syntax​

Basic Structure​

class ClassName {
  properties;

  methods;
}

Example​

class User {
  name: string;
  age: number;
}

Here:

Class Name → User

Properties:
- name
- age

Creating Objects​

Classes themselves do not create objects.

We create objects using:

new

keyword.

Example​

class User {
  name: string;
  age: number;
}

Create Object

const user = new User();

Memory:

User Class
      ↓
new User()
      ↓
Object Created

What is an Object?​

An Object is:

An instance of a class.

Example

class User {}
const user1 = new User();
const user2 = new User();

Memory

user1 → User Object

user2 → User Object

Both are separate objects.

Adding Properties​

Example​

class User {
  name: string;
  age: number;
}

Create Object

const user = new User();

user.name = "Prashant";
user.age = 25;

Output

console.log(user);
User {
  name: 'Prashant',
  age: 25
}

Constructor​

One of the most important concepts.

What is a Constructor?​

A Constructor is a special method that runs:

Automatically

when an object is created.

Why Do We Need Constructors?​

Without Constructor:

const user = new User();

user.name = "Prashant";
user.age = 25;

Multiple assignments needed.

With Constructor:

const user = new User("Prashant", 25);

Cleaner.

Constructor Syntax​

class User {
  constructor() {}
}

Example

class User {
  constructor() {
    console.log("User Created");
  }
}

Create Object

const user = new User();

Output

User Created

Constructor automatically executes.

Constructor with Parameters​

Example

class User {
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }
}

Create Object

const user = new User("Prashant", 25);

Output

console.log(user);
User {
  name: 'Prashant',
  age: 25
}

Understanding this​

One of the most important OOP concepts.

Consider:

class User {
  name: string;

  constructor(name: string) {
    this.name = name;
  }
}

Question:

What is:

this.name;

?

Meaning of this​

Current Object

Example

const user1 = new User("Prashant");

const user2 = new User("John");

During Object Creation

For user1:

this.name = "Prashant";

For user2:

this.name = "John";

So:

this refers to the object
currently being created or used.

Memory Visualization​

Object Creation

const user = new User("Prashant", 25);

Memory

user
 │
 ▼

{
  name: "Prashant",
  age: 25
}

Constructor executes:

this.name = name;
this.age = age;

Result:

Object Initialized

Methods​

Classes can contain functions.

Inside classes they are called:

Methods

Example

class User {
  name: string;

  constructor(name: string) {
    this.name = name;
  }

  greet() {
    console.log(`Hello ${this.name}`);
  }
}

Create Object

const user = new User("Prashant");

user.greet();

Output

Hello Prashant

Multiple Methods​

Example

class BankAccount {
  balance: number;

  constructor(balance: number) {
    this.balance = balance;
  }

  deposit(amount: number) {
    this.balance += amount;
  }

  withdraw(amount: number) {
    this.balance -= amount;
  }

  showBalance() {
    console.log(this.balance);
  }
}

Usage

const account = new BankAccount(1000);

account.deposit(500);

account.withdraw(200);

account.showBalance();

Output

1300

Class as a Custom Type​

A class automatically creates a type.

Example

class User {
  name: string;

  constructor(name: string) {
    this.name = name;
  }
}

Valid

const user: User = new User("Prashant");

Notice:

User;

is acting as:

Both Class and Type

Property Initialization Error​

Common TypeScript Error.

Example

class User {
  name: string;
}

Error

Property 'name'
has no initializer
and is not definitely assigned.

Why?

Because TypeScript expects:

Every property must be initialized.

Fix 1 — Constructor Initialization​

class User {
  name: string;

  constructor(name: string) {
    this.name = name;
  }
}

Fix 2 — Default Value​

class User {
  name: string = "";
}

Fix 3 — Definite Assignment Assertion​

class User {
  name!: string;
}

Meaning:

Trust me TypeScript,
I will initialize it later.

Use carefully.

Constructor Shorthand​

Very common in production code.

Instead of:

class User {
  name: string;
  age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }
}

Use:

class User {
  constructor(
    public name: string,
    public age: number,
  ) {}
}

TypeScript automatically creates:

name;
age;

properties.

Equivalent behavior.

Class Expression​

Classes can be stored in variables.

Example

const User = class {
  name: string = "Prashant";
};

Create Object

const user = new User();

Less common than normal classes.

Real World Example​

User Model

class User {
  constructor(
    public id: number,
    public name: string,
    public email: string,
  ) {}

  display() {
    console.log(`${this.name} (${this.email})`);
  }
}

Usage

const user = new User(1, "Prashant", "abc@gmail.com");

user.display();

Output

Prashant (abc@gmail.com)

Common Mistakes​

Forgetting new​

Wrong

const user = User();

Error.

Correct

const user = new User();

Using Property Before Initialization​

Wrong

class User {
  name: string;
}

Initialize properly.

Forgetting this​

Wrong

constructor(name: string) {
  name = name;
}

Correct

constructor(name: string) {
  this.name = name;
}

Interview Questions​

Q1​

What is a Class?

Answer​

A blueprint used to create objects.

Q2​

What is an Object?

Answer​

An instance of a class.

Q3​

What is a Constructor?

Answer​

A special method that executes
automatically when an object is created.

Q4​

What does this refer to?

Answer​

The current object instance.

Q5​

Why use Classes?

Answer​

To model real-world entities,
improve code organization,
and support OOP principles.

Cheat Sheet​

class User {}
const user = new User();
constructor() {}
this.name = name;
class User {
  constructor(public name: string) {}
}
user.greet();

Key Takeaways​

  • A class is a blueprint for creating objects.
  • Objects are instances of classes.
  • The new keyword creates objects.
  • Constructors run automatically during object creation.
  • this refers to the current object instance.
  • Methods define object behavior.
  • Classes act as both runtime constructs and TypeScript types.
  • Properties should be initialized properly.
  • Constructor shorthand is heavily used in production TypeScript.
  • Classes form the foundation for inheritance, abstraction, and advanced OOP concepts.

Inheritance

One of the biggest advantages of Object-Oriented Programming is:

Code Reusability

Instead of rewriting the same properties and methods again and again, we can create:

Parent Classes

and allow other classes to inherit their functionality.

This concept is called:

Inheritance

In TypeScript, inheritance is implemented using:

extends

keyword.

What is Inheritance?​

Inheritance allows one class to:

Acquire Properties and Methods
from another class.

Real World Example​

Animal
 │
 ├── Dog
 ├── Cat
 └── Lion

All animals may have:

name
age
eat()
sleep()

Instead of duplicating code:

class Dog {}
class Cat {}
class Lion {}

we create:

Animal;

once and reuse it.

Parent Class vs Child Class​

Parent Class​

Also called:

Base Class
Super Class

Example

class Animal {}

Child Class​

Also called:

Derived Class
Subclass

Example

class Dog extends Animal {}

Relationship:

Animal
  ↑
 Dog

extends Keyword​

Inheritance is created using:

extends

Example

class Animal {
  eat() {
    console.log("Eating...");
  }
}

class Dog extends Animal {}

Create Object

const dog = new Dog();

dog.eat();

Output

Eating...

Notice:

Dog never defined eat()

It inherited it from:

Animal

Memory Visualization​

Animal
 │
 └── eat()
Dog
 │
 └── inherits eat()

Object

const dog = new Dog();

Can access:

dog.eat();

because:

Dog IS-A Animal

Adding Child-Specific Methods​

Child classes can have their own methods.

Example

class Animal {
  eat() {
    console.log("Eating...");
  }
}

class Dog extends Animal {
  bark() {
    console.log("Woof!");
  }
}

Usage

const dog = new Dog();

dog.eat();

dog.bark();

Output

Eating...
Woof!

Inheritance with Properties​

Example

class Animal {
  name: string;

  constructor(name: string) {
    this.name = name;
  }
}

Child Class

class Dog extends Animal {}

Usage

const dog = new Dog("Tommy");

console.log(dog.name);

Output

Tommy

Property inherited successfully.

Constructors in Inheritance​

Things become more interesting when constructors exist.

Parent

class Animal {
  name: string;

  constructor(name: string) {
    this.name = name;
  }
}

Child

class Dog extends Animal {}

Usage

const dog = new Dog("Tommy");

Works.

Because:

Dog automatically uses
Animal's constructor.

Child Constructor​

Suppose Dog needs:

name
breed

Example

class Animal {
  constructor(public name: string) {}
}
class Dog extends Animal {
  constructor(
    name: string,
    public breed: string,
  ) {}
}

Error.

Constructors for derived classes
must contain a super call.

What is super()?​

When a child class defines its own constructor:

Parent Constructor
must execute first.

This is done using:

super();

Correct Example​

class Animal {
  constructor(public name: string) {}
}
class Dog extends Animal {
  constructor(
    name: string,
    public breed: string,
  ) {
    super(name);
  }
}

Usage

const dog = new Dog("Tommy", "Labrador");

Output

console.log(dog.name);
console.log(dog.breed);
Tommy
Labrador

Why Must super() Come First?​

Consider:

class Dog extends Animal {
  constructor(name: string) {
    console.log("Dog");
    super(name);
  }
}

Error.

Because:

this cannot be used
before super()

Flow

Create Parent Object
        ↓
Initialize Parent
        ↓
Initialize Child

Parent must exist first.

Method Overriding​

One of the most important OOP concepts.

What is Method Overriding?​

A child class can:

Replace

the implementation of a parent method.

Parent

class Animal {
  makeSound() {
    console.log("Some Sound");
  }
}

Child

class Dog extends Animal {
  makeSound() {
    console.log("Woof!");
  }
}

Usage

const dog = new Dog();

dog.makeSound();

Output

Woof!

Parent method overridden.

Overriding Example​

Parent

class Employee {
  work() {
    console.log("Working...");
  }
}

Child

class Developer extends Employee {
  work() {
    console.log("Writing Code...");
  }
}

Usage

const dev = new Developer();

dev.work();

Output

Writing Code...

Calling Parent Method using super​

Sometimes we want:

Parent Logic
+
Child Logic

Example

class Animal {
  makeSound() {
    console.log("Animal Sound");
  }
}
class Dog extends Animal {
  makeSound() {
    super.makeSound();

    console.log("Woof!");
  }
}

Usage

const dog = new Dog();

dog.makeSound();

Output

Animal Sound
Woof!

super Keyword​

The keyword:

super

refers to:

Parent Class

Used for:

super();

Parent Constructor

super.method();

Parent Method

Access Modifiers and Inheritance​

Parent

class Animal {
  public a = 1;

  protected b = 2;

  private c = 3;
}

Child

class Dog extends Animal {
  test() {
    console.log(this.a);

    console.log(this.b);

    console.log(this.c);
  }
}

Result

a → Valid

b → Valid

c → Error

Because:

private

is not inherited.

Multi-Level Inheritance​

TypeScript supports:

Inheritance Chain

Example

class Animal {}

class Dog extends Animal {}

class Labrador extends Dog {}

Hierarchy

Animal
   ↑
 Dog
   ↑
Labrador

Labrador inherits:

Animal
Dog

features.

instanceof Operator​

Used to check inheritance relationships.

Example

class Animal {}

class Dog extends Animal {}

Usage

const dog = new Dog();
console.log(dog instanceof Dog);

Output

true
console.log(dog instanceof Animal);

Output

true

Because:

Dog IS-A Animal

Real World Example​

class Employee {
  constructor(public name: string) {}

  work() {
    console.log(`${this.name} is working`);
  }
}
class Developer extends Employee {
  constructor(
    name: string,
    public language: string,
  ) {
    super(name);
  }

  work() {
    console.log(
      `${this.name}
       writes ${this.language}`,
    );
  }
}

Usage

const dev = new Developer("Prashant", "TypeScript");

dev.work();

Output

Prashant writes TypeScript

Common Mistakes​

Forgetting super()​

Wrong

class Dog extends Animal {
  constructor() {}
}

Error.

Always call:

super(...)

Using this Before super()​

Wrong

constructor() {

  this.name = "Tom";
  super();
}

Error.

Correct

constructor() {

  super();

  this.name = "Tom";
}

Expecting private Members to Inherit​

Wrong

this.password;

inside child class.

private members:

Are Not Accessible

in child classes.

Use:

protected;

instead.

Interview Questions​

Q1​

What is Inheritance?

Answer​

A mechanism where one class
acquires properties and methods
from another class.

Q2​

Which keyword is used for inheritance?

Answer​

extends

Q3​

What is Method Overriding?

Answer​

Providing a new implementation
of a parent method in a child class.

Q4​

What is super()?

Answer​

Calls the parent constructor.

Q5​

Can private members be inherited?

Answer​

No.

Only public and protected
members are accessible.

Q6​

What does instanceof do?

Answer​

Checks whether an object
belongs to a class hierarchy.

Cheat Sheet​

class Dog extends Animal {}
super(name);
super.method();
instanceof
override method()

(Conceptually overriding)

protected member

accessible in child classes.

Key Takeaways​

  • Inheritance allows code reuse between classes.
  • extends is used to create inheritance relationships.
  • Child classes automatically inherit parent properties and methods.
  • super() calls the parent constructor.
  • super.method() calls a parent method.
  • Method Overriding allows child classes to customize behavior.
  • private members are not accessible in child classes.
  • protected members are accessible in child classes.
  • Multi-level inheritance is supported.
  • instanceof helps identify inheritance relationships at runtime.
  • Inheritance is one of the core pillars of Object-Oriented Programming.

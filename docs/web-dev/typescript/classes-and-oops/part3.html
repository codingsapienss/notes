Getters, Setters & Static Members

In Part 6A and 6B, we learned:

  • Classes
  • Constructors
  • Methods
  • Access Modifiers

However, directly exposing properties is not always ideal.

Sometimes we need:

  • Validation before updating data
  • Computed properties
  • Controlled access to internal state
  • Shared class-level data

TypeScript provides:

  • Getters
  • Setters
  • Static Properties
  • Static Methods

to solve these problems.

Why Do We Need Getters and Setters?​

Consider:

class User {
  public age: number = 0;
}

Usage:

const user = new User();

user.age = -100;

Problem:

Invalid State

Age should never be negative.

We need:

Controlled Read Access
Controlled Write Access

This is where:

Getters
Setters

become useful.

What is a Getter?​

A Getter is a special method used to:

Read a value like a property

while executing custom logic behind the scenes.

Getter Syntax​

get propertyName() {
  return value;
}

Basic Getter Example​

class User {
  private _name: string;

  constructor(name: string) {
    this._name = name;
  }

  get name() {
    return this._name;
  }
}

Usage

const user = new User("Prashant");

console.log(user.name);

Output

Prashant

Notice:

user.name;

NOT

user.name();

Getters behave like:

Normal Properties

How Getters Work Internally​

When you write:

user.name;

TypeScript automatically calls:

get name()

Flow

user.name
     ↓
get name()
     ↓
return _name

Why Use Getters?​

Useful when:

Reading Data Requires Logic

Example

class User {
  constructor(
    public firstName: string,
    public lastName: string,
  ) {}

  get fullName() {
    return `${this.firstName} ${this.lastName}`;
  }
}

Usage

const user = new User("Prashant", "Sharma");

console.log(user.fullName);

Output

Prashant Sharma

Notice:

fullName

is never stored in memory.

It is:

Computed Dynamically

Computed Property Example​

class Rectangle {
  constructor(
    public width: number,
    public height: number,
  ) {}

  get area() {
    return this.width * this.height;
  }
}

Usage

const rect = new Rectangle(10, 5);

console.log(rect.area);

Output

50

No separate:

area;

property exists.

What is a Setter?​

A Setter is a special method used to:

Control Property Updates

before data is stored.

Setter Syntax​

set propertyName(value) {
}

Basic Setter Example​

class User {
  private _age = 0;

  set age(value: number) {
    this._age = value;
  }
}

Usage

const user = new User();

user.age = 25;

Setter automatically runs.

Validation Using Setters​

One of the most common use cases.

Example

class User {
  private _age = 0;

  set age(value: number) {
    if (value < 0) {
      throw new Error("Age cannot be negative");
    }

    this._age = value;
  }

  get age() {
    return this._age;
  }
}

Valid

user.age = 25;

Invalid

user.age = -10;

Throws:

Error:
Age cannot be negative

Getter and Setter Together​

Very common.

Example

class User {
  private _name = "";

  get name() {
    return this._name;
  }

  set name(value: string) {
    if (value.length < 3) {
      throw new Error("Name too short");
    }

    this._name = value;
  }
}

Usage

const user = new User();

user.name = "Prashant";

console.log(user.name);

Output

Prashant

Why Use Private Backing Variables?​

Notice:

private _name

instead of:

name;

Why?

Because:

set name()

and

get name()

already use:

name;

as the public interface.

Convention:

_name;
_age;
_balance;
_salary;

represent:

Internal Storage

Real World Example​

Bank Account

class BankAccount {
  private _balance = 0;

  get balance() {
    return this._balance;
  }

  set balance(value: number) {
    if (value < 0) {
      throw new Error("Balance cannot be negative");
    }

    this._balance = value;
  }
}

Usage

const account = new BankAccount();

account.balance = 5000;

console.log(account.balance);

Output

5000

When Should You Use Getters and Setters?​

Use them when:

Validation Needed
Computed Values
Controlled Access
Data Transformation

Avoid them when:

Simple Property Access

is sufficient.

What are Static Members?​

Until now:

const user1 = new User();
const user2 = new User();

Every object gets its own properties.

Sometimes we need:

One Shared Value
Across Entire Class

This is where:

static;

comes in.

Static Properties​

A Static Property belongs to:

The Class

NOT:

The Object

Example​

class User {
  static totalUsers = 0;

  constructor() {
    User.totalUsers++;
  }
}

Create Objects

new User();
new User();
new User();

Output

console.log(User.totalUsers);
3

Memory Visualization​

Normal Property

user.name;

Stored inside:

Each Object

Static Property

User.totalUsers;

Stored inside:

Class Itself

Static Property Access​

Correct

User.totalUsers;

Wrong

const user = new User();

user.totalUsers;

Error.

Because:

Static Members
Belong To Class

Static Methods​

Just like static properties.

They belong to:

Class

not objects.

Example​

class MathUtils {
  static add(a: number, b: number) {
    return a + b;
  }
}

Usage

console.log(MathUtils.add(10, 20));

Output

30

No Object Needed

MathUtils.add();

works directly.

Why Use Static Methods?​

Useful for:

Helper Functions
Utility Methods
Shared Logic
Factory Methods

Examples

Math.max()
Number.isNaN()
Array.isArray()

All are static methods.

Static Method Example​

class Temperature {
  static celsiusToFahrenheit(celsius: number) {
    return (celsius * 9) / 5 + 32;
  }
}

Usage

console.log(Temperature.celsiusToFahrenheit(30));

Output

86

Static vs Instance Members​

Instance Property

class User {
  name: string = "";
}

Access

const user = new User();

user.name;

Static Property

class User {
  static totalUsers = 0;
}

Access

User.totalUsers;

Can Static Methods Access Instance Properties?​

Consider:

class User {
  name = "Prashant";

  static printName() {
    console.log(this.name);
  }
}

Problem:

Error

Why?

Because:

Static Methods
Do Not Belong To Objects

They belong to:

Class

What Does this Mean in Static Methods?​

Instance Method

greet() {
  console.log(this.name);
}

Here:

this

refers to:

Current Object

Static Method

static print() {}

Here:

this

refers to:

The Class Itself

Example

class User {
  static totalUsers = 5;

  static printCount() {
    console.log(this.totalUsers);
  }
}

Valid.

Output

5

Real World Example​

class User {
  static totalUsers = 0;

  constructor(public name: string) {
    User.totalUsers++;
  }

  static getTotalUsers() {
    return User.totalUsers;
  }
}

Usage

new User("A");
new User("B");
new User("C");

console.log(User.getTotalUsers());

Output

3

Common Mistakes​

Accessing Static Members Through Objects​

Wrong

const user = new User();

user.totalUsers;

Correct

User.totalUsers;

Accessing Instance Properties in Static Methods​

Wrong

static print() {
  console.log(this.name);
}

Because:

No Object Exists

Infinite Recursion in Getter​

Wrong

get name() {
  return this.name;
}

Calls itself forever.

Correct

get name() {
  return this._name;
}

A special method used to read

Answer​

A special method used to control
property updates.

Why use Setters?

Answer​

Validation
Controlled Updates
Business Rules

What is a Static Property?

Answer​

A property that belongs
to the class itself.

Answer​

Cheat Sheet​

get name() {
  return this._name;
}
set name(value) {
  this._name = value;
}
static totalUsers = 0;
static add(a,b) {
  return a+b;
}
User.totalUsers;
MathUtils.add(10, 20);

Key Takeaways​

  • Getters allow controlled reading of data.
  • Setters allow controlled updates and validation.
  • Getters and Setters behave like normal properties.
  • Private backing variables (_name, _age) are commonly used.
  • Static members belong to the class, not objects.
  • Static properties store shared data.
  • Static methods provide utility and helper functionality.
  • this inside static methods refers to the class itself.
  • Static members are accessed using the class name.
  • Getters, Setters and Static members are heavily used in production TypeScript applications.

Interface Fundamentals

Interfaces are one of the most important features in TypeScript.

They allow us to define:

The Structure of an Object

Interfaces help us create:

  • Reusable Type Definitions
  • Better Code Organization
  • Strong Type Safety
  • Self-Documenting Code

Almost every large TypeScript codebase uses interfaces extensively.

What is an Interface?​

An Interface is a blueprint that describes the shape of an object.

Example:

interface User {
  name: string;
  age: number;
}

This means:

Any object of type User
must contain:

name → string
age → number

Why Do We Need Interfaces?​

Without interfaces:

function createUser(user: { name: string; age: number; email: string }) {}

Imagine writing this everywhere.

Hard to Read
Hard to Maintain
Repeated Code

Instead:

interface User {
  name: string;
  age: number;
  email: string;
}
function createUser(user: User) {}

Much cleaner.

Interface Syntax​

Basic Syntax

interface InterfaceName {
  property: type;
}

Example

interface User {
  name: string;
  age: number;
}

Usage

const user: User = {
  name: "Prashant",
  age: 25,
};

Type Checking​

TypeScript validates:

Property Names
Property Types
Required Properties

Valid

const user: User = {
  name: "Prashant",
  age: 25,
};

Invalid

const user: User = {
  name: "Prashant",
};

Error:

Property 'age' is missing

Invalid

const user: User = {
  name: "Prashant",
  age: "25",
};

Error.

Interface vs Object Type​

You already learned:

type User = {
  name: string;
  age: number;
};

Equivalent Interface:

interface User {
  name: string;
  age: number;
}

Both describe object structures.

Differences will be covered in Part 5C.

Optional Properties​

Sometimes properties may not exist.

Use:

?

Example

interface User {
  name: string;
  age?: number;
}

Valid

const user: User = {
  name: "Prashant",
};

Valid

const user: User = {
  name: "Prashant",
  age: 25,
};

Internally:

age?: number

behaves like:

age: number | undefined;

Real World Example​

User Profile

interface UserProfile {
  name: string;
  phone?: string;
}

Some users may have:

Phone Number

Others may not.

Readonly Properties​

Some properties should never change.

Use:

readonly;

Example

interface User {
  readonly id: number;
  name: string;
}

Create Object

const user: User = {
  id: 1,
  name: "Prashant",
};

Valid

user.name = "John";

Invalid

user.id = 10;

Error:

Cannot assign to readonly property

Why Use Readonly?​

Common examples:

Database IDs
Order IDs
User IDs
Created Timestamps

Values that should never change.

Method Signatures​

Interfaces can describe methods.

Example

interface User {
  name: string;

  greet(): void;
}

Implementation

const user: User = {
  name: "Prashant",

  greet() {
    console.log("Hello");
  },
};

Output

Hello

Method Parameters​

Methods can have parameters.

Example

interface Calculator {
  add(a: number, b: number): number;
}

Implementation

const calc: Calculator = {
  add(a, b) {
    return a + b;
  },
};

Usage

console.log(calc.add(10, 20));

Output

30

Method Return Types​

Interfaces also enforce return types.

Example

interface UserService {
  getName(): string;
}

Valid

const service: UserService = {
  getName() {
    return "Prashant";
  },
};

Invalid

const service: UserService = {
  getName() {
    return 100;
  },
};

Error.

Nested Interfaces​

Interfaces can contain other interfaces.

Example

interface Address {
  city: string;
  pincode: number;
}
interface User {
  name: string;
  address: Address;
}

Usage

const user: User = {
  name: "Prashant",

  address: {
    city: "Delhi",
    pincode: 201001,
  },
};

Deeply Nested Interfaces​

Example

interface Contact {
  phone: string;
}
interface Address {
  city: string;
  contact: Contact;
}
interface User {
  name: string;
  address: Address;
}

Usage

const user: User = {
  name: "Prashant",

  address: {
    city: "Delhi",

    contact: {
      phone: "9999999999",
    },
  },
};

Arrays Inside Interfaces​

Very common.

Example

interface User {
  name: string;
  skills: string[];
}

Usage

const user: User = {
  name: "Prashant",

  skills: ["TypeScript", "React", "Node.js"],
};

Interface Reusability​

One of the biggest advantages.

Example

interface User {
  name: string;
  age: number;
}

Reuse everywhere:

const user1: User;
const user2: User;
function saveUser(user: User) {}
function updateUser(user: User) {}

Single source of truth.

Interfaces in Functions​

Parameter Example

interface User {
  name: string;
  age: number;
}
function printUser(user: User) {
  console.log(user.name);
}

Call

printUser({
  name: "Prashant",
  age: 25,
});

Interfaces as Return Types​

Example

interface User {
  name: string;
  age: number;
}
function getUser(): User {
  return {
    name: "Prashant",
    age: 25,
  };
}

Structural Typing Still Applies​

TypeScript uses:

Structural Typing

Example

interface User {
  name: string;
}

Object

const person = {
  name: "Prashant",
  age: 25,
};

Valid

const user: User = person;

Why?

Because:

Required structure exists.

Real World Example​

API User Model

interface User {
  id: number;
  name: string;
  email: string;
  isAdmin: boolean;
}

Usage

const user: User = await getUser();

TypeScript now knows:

user.id;

↓

number;
user.email;

↓

string;

Strong typing everywhere.

Common Mistakes​

Forgetting Required Properties​

Wrong

const user: User = {
  name: "Prashant",
};

Missing:

age;

Wrong Property Types​

Wrong

const user: User = {
  name: "Prashant",
  age: "25",
};

Must be:

number;

Modifying Readonly Properties​

Wrong

user.id = 100;

Readonly prevents reassignment.

Interview Questions​

Q1​

What is an Interface?

Answer​

A blueprint that describes
the structure of an object.

Q2​

Why use Interfaces?

Answer​

Reusability
Readability
Type Safety
Maintainability

Q3​

How do you create an optional property?

Answer​

age?: number;

Q4​

How do you create a readonly property?

Answer​

readonly id: number;

Q5​

Can Interfaces contain methods?

Answer​

Yes.

greet(): void;

Cheat Sheet​

interface User {
  name: string;
  age: number;
}
age?: number;
readonly id: number;
greet(): void;
address: Address;
skills: string[];
function saveUser(user: User) {}

Key Takeaways​

  • Interfaces define object structures.
  • They improve readability and reusability.
  • Interfaces support optional properties using ?.
  • Interfaces support readonly properties using readonly.
  • Interfaces can define methods and method signatures.
  • Interfaces can be nested.
  • Interfaces work seamlessly with arrays and functions.
  • TypeScript uses structural typing with interfaces.
  • Interfaces are one of the most commonly used features in large TypeScript codebases.
  • Mastering interfaces is essential before learning interface extension and declaration merging.

Intersection Types

In previour lession, we learned about Union Types (|), which allow a value to be one type OR another type.

Intersection Types solve a different problem.

Instead of:

Type A OR Type B

they allow:

Type A AND Type B

This is extremely useful when building:

  • User Models
  • API Responses
  • Database Entities
  • React Props
  • Shared Type Systems

Intersection Types allow us to combine multiple types into a single type.

What is an Intersection Type?​

An Intersection Type combines multiple types into one.

Syntax​

TypeA & TypeB;

Read as:

TypeA AND TypeB

Mental Model​

Suppose:

type Person = {
  name: string;
};

and

type Employee = {
  employeeId: number;
};

Intersection:

type Staff = Person & Employee;

Result:

{
  name: string;
  employeeId: number;
}

The new type contains:

ALL properties
from BOTH types

First Example​

Type 1​

type Person = {
  name: string;
};

Type 2​

type Employee = {
  employeeId: number;
};

Combined​

type Staff = Person & Employee;

Result

{
  name: string;
  employeeId: number;
}

Usage​

const staff: Staff = {
  name: "Prashant",
  employeeId: 101,
};

Valid.

Missing Property Example​

const staff: Staff = {
  name: "Prashant",
};

Error.

Why?

Because:

employeeId

is also required.

Visualization​

Person

{
  name: string;
}

Employee

{
  employeeId: number;
}

Intersection

{
  name: string;
  employeeId: number;
}

Intersection with Multiple Types​

Not limited to two types.

Example

type Person = {
  name: string;
};
type Employee = {
  employeeId: number;
};
type Admin = {
  permissions: string[];
};

Combined

type AdminStaff = Person & Employee & Admin;

Result

{
  name: string;
  employeeId: number;
  permissions: string[];
}

Usage​

const admin: AdminStaff = {
  name: "Prashant",
  employeeId: 101,
  permissions: ["read", "write"],
};

Why Do We Need Intersections?​

Without Intersections:

type AdminUser = {
  name: string;
  employeeId: number;
  permissions: string[];
};

Now imagine:

50 Types
100 Models

Huge duplication.

Intersections allow:

Composition

instead of repetition.

Real World Example​

Suppose every entity has:

type Timestamps = {
  createdAt: Date;
  updatedAt: Date;
};

User:

type User = {
  id: number;
  name: string;
};

Combine

type UserEntity = User & Timestamps;

Result

{
  id: number;
  name: string;
  createdAt: Date;
  updatedAt: Date;
}

Very common in backend applications.

Intersection vs Union​

This is one of the most important concepts.

Union​

type A = {
  name: string;
};

type B = {
  age: number;
};

type Result = A | B;

Allowed

{
  name: "Prashant";
}

Allowed

{
  age: 25;
}

Allowed

{
  name: "Prashant",
  age: 25
}

Because:

A OR B

Intersection​

type Result = A & B;

Required

{
  name: "Prashant",
  age: 25
}

Invalid

{
  name: "Prashant";
}

Invalid

{
  age: 25;
}

Because:

A AND B

Visual Comparison​

Union

A | B;

Means:

Either A
or B
or both

Intersection

A & B;

Means:

Must satisfy A
and B
simultaneously

Intersection with Primitive Types​

Possible but rarely useful.

Example

type Result = string & number;

Question:

Can a value be:

string AND number

at the same time?

No.

Result:

never;

Meaning:

Impossible Type

Example​

type Value = string & number;

Trying:

let x: Value;

No valid value can exist.

Property Conflict Example​

Type 1

type A = {
  id: string;
};

Type 2

type B = {
  id: number;
};

Intersection

type Result = A & B;

Becomes

{
  id: string & number;
}

Which becomes:

never;

Result:

{
  id: never;
}

Impossible to satisfy.

Example​

type A = {
  id: string;
};

type B = {
  id: number;
};

type Result = A & B;

Cannot create:

const value: Result = {
  id: ???
};

No value satisfies both.

Compatible Property Example​

Type A

type A = {
  id: number;
};

Type B

type B = {
  name: string;
};

Intersection

type Result = A & B;

Result

{
  id: number;
  name: string;
}

Perfectly valid.

Intersections with Type Aliases​

Very common.

Example

type BaseUser = {
  id: number;
};
type Profile = {
  name: string;
};
type User = BaseUser & Profile;

Result

{
  id: number;
  name: string;
}

API Response Example​

Common Metadata

type ApiMeta = {
  timestamp: string;
};

Actual Data

type UserData = {
  id: number;
  name: string;
};

Response

type ApiResponse = ApiMeta & UserData;

Result

{
  timestamp: string;
  id: number;
  name: string;
}

Why Intersections Are Powerful​

They encourage:

Composition

instead of:

Duplication

Bad

type User = {
  id: number;
  createdAt: Date;
  updatedAt: Date;
};
type Product = {
  id: number;
  createdAt: Date;
  updatedAt: Date;
};

Repeated code.

Better

type Timestamps = {
  createdAt: Date;
  updatedAt: Date;
};
type User = BaseUser & Timestamps;
type Product = BaseProduct & Timestamps;

Cleaner.

Common Mistakes​

Confusing Union and Intersection​

Wrong Mental Model:

&
means OR

No.

&
means AND

Conflicting Properties​

Bad

type A = {
  id: string;
};

type B = {
  id: number;
};

Produces:

id: never;

Overusing Huge Intersections​

Bad

A & B & C & D & E & F;

Can become difficult to understand.

Keep types meaningful.

Interview Questions​

Q1​

What is an Intersection Type?

Answer​

A type that combines multiple
types into one.

Q2​

What does & mean?

Answer​

AND

between types.

Q3​

Difference between:

A | B;

and

A & B;

Answer​

A | B;

means:

Either A or B
A & B;

means:

Both A and B

Q4​

What happens when two intersected properties have incompatible types?

Answer​

They become:

never;

making the type impossible to satisfy.

Q5​

Why are Intersection Types useful?

Answer​

They reduce duplication by
allowing type composition.

Cheat Sheet​

type Staff = Person & Employee;
type UserEntity = User & Timestamps;
type AdminUser = User & Employee & Admin;
type Result = string & number;

↓

never;

Key Takeaways​

  • Intersection Types combine multiple types into one.
  • The & operator means "AND".
  • The resulting type must satisfy all intersected types.
  • Intersections are commonly used for type composition.
  • They help eliminate duplication.
  • Union Types increase flexibility; Intersection Types increase requirements.
  • Property conflicts can produce never.
  • Intersections work extremely well with Type Aliases.
  • They are heavily used in API models, database entities, and large applications.
  • Understanding intersections is essential before learning Type Narrowing and Discriminated Unions.

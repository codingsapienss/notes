Introduction

Strings look simple:

string name="Prashant";

but internally they involve:

  • arrays
  • pointers
  • memory
  • null termination
  • dynamic allocation
  • object management

Many C++ bugs come from misunderstanding strings and characters.

Character (char)

Intuition​

A character stores exactly one symbol.

Examples:

A
B
a
1
$
#

Definition​

char is a fundamental data type used to store a single character.

Syntax:

char ch='A';

Internal Storage​

Characters are stored internally as integers using ASCII values.

Example:

char ch='A';

Internally:

'A'

↓

65

ASCII Table​

CharacterASCII Value
A65
B66
a97
b98
048
149

Example​

#include<iostream>
using namespace std;

int main()
{
    char ch='A';

    cout<<ch<<endl;

    cout<<(int)ch;
}

Output:

A
65

Code Explanation​

(int)ch

converts character into integer.

Character Array

Definition​

Character array is an array of characters.

Syntax:

char ch[10];

Initialization​

char ch[10]={'a'};

Memory:

Index:

0 1 2 3 4 5 ...

Value:

a \0 \0 \0 \0 \0

Remaining values automatically become:

'\0'

Example​

char ch[5]={'H','e','l','l','o'};

Memory:

H e l l o

Notice:

No null character

This is dangerous.

Correct:

char ch[]="Hello";

Memory:

H e l l o \0

String Literal

Definition​

String literals are enclosed in double quotes.

Example:

"Hello"

Important Hidden Detail​

String literals are:

Read Only

Wrong:

char *ptr="Hello";

ptr[0]='M';

Undefined behavior.

Correct:

char arr[]="Hello";

arr[0]='M';

Output:

Mello

Why?

String literal stored internally:

Read-only memory segment

Character array:

Stack memory

modifiable.

Null Character

Definition​

Null character:

'\0'

ASCII value:

0

Purpose:

Marks end of string.

Why Null Character Exists

Suppose:

char ch[]="Prashant";

Memory:

P r a s h a n t \0

When printing:

cout<<ch;

C++ reads:

P
r
a
s
h
a
n
t

stop at '\0'

Example

char arr[]={'a','\0','b','\0','c'};

cout<<arr;

Output:

a

Reason:

Printing stops at first null.

Input Into Character Array

Method 1​

char name[20];

cin>>name;

Input:

Prashant

Output:

Prashant

Hidden Behavior​

Input:

Prashant Sharma

Output:

Prashant

Why?​

cin stops at:

  • Space
' '
  • Tab
'\t'
  • New line
'\n'

Execution:

Input:

Prashant Sharma

Memory:

P r a s h a n t \0

Remaining:

Sharma

still exists in input buffer.

Safe Input Using getline()

char name[50];

cin.getline(name,50);

Input:

Prashant Sharma

Output:

Prashant Sharma

Buffer Overflow Problem

Wrong:

char name[5];

cin>>name;

Input:

Prashant

Problem:

Memory:

P r a s h a n t

Array:

Only 5 positions

Result:

Buffer overflow

Possible:

  • Segmentation fault
  • Memory corruption
  • Undefined behavior

Output

Character​

char ch='A';

cout<<ch;

Output:

A

Character Array​

char name[]="Prashant";

cout<<name;

Output:

Prashant

Important Hidden Behavior​

Character arrays behave differently.

Example:

int arr[]={1,2,3};

cout<<arr;

Output:

Address

But:

char arr[]="Hello";

cout<<arr;

Output:

Hello

Reason:

cout has special handling for character arrays.

Strings (string)

Definition​

C++ string is a class from STL.

Header:

#include<string>

Syntax:

string s="Hello";

Important Correction​

Wrong:

String is 1D character array

Correct:

C++ string is an STL class which internally manages dynamic character arrays.

Internal Working

string s="Hello";

Internally:

String Object

↓

Pointer

↓

Heap Memory

↓

H e l l o \0

String object stores:

  • pointer
  • size
  • capacity

Size vs Capacity

Example:

string s="Hello";

Possible:

Size=5

Capacity=15

Dynamic Length Handling

Character array:

char arr[20];

Fixed size.

String:

string s;

Can grow automatically.

Example:

s+="Hello";
s+="World";

How Length Is Determined In Character Array

For character arrays:

strlen()

works by scanning:

until '\0'

Example:

char ch[]="Hello";

Memory:

H e l l o \0

Length:

5

Internal Working​

Count++

until

'\0'

Complexity:

O(N)

Why String Doesn't Need Size

Character array:

char arr[20];

Needs memory reservation.

String:

string s;

internally allocates dynamically.

Array Of Characters vs Array Of Strings

Array of Characters​

char arr[]="Hello";

Memory:

H e l l o \0

Array of Strings​

string arr[3]=
{
"Hello",
"Hi",
"Bye"
};

Memory:

arr[0]

↓

Hello

arr[1]

↓

Hi

arr[2]

↓

Bye

Important STL String Functions

length()​

string s="Hello";

cout<<s.length();

Output:

5

size()​

cout<<s.size();

empty()​

cout<<s.empty();

push_back()​

s.push_back('A');

pop_back()​

s.pop_back();

substr()​

string s="Prashant";

cout<<s.substr(2,4);

Output:

asha

find()​

cout<<s.find("sha");

Output:

3

erase()​

s.erase(2,3);

insert()​

s.insert(2,"ABC");

reverse()​

reverse(
s.begin(),
s.end()
);

Complexity Table

OperationComplexity
AccessO(1)
Push BackO(1) amortized
LengthO(1)
Insert MiddleO(N)
Erase MiddleO(N)

Common Bugs

Bug 1​

Wrong:

char arr[5]="Hello";

Problem:

Needs:

6 positions

because:

H e l l o \0

Correct:

char arr[6]="Hello";

Bug 2​

Wrong:

char *ptr="Hello";

ptr[0]='M';

Bug 3​

Wrong:

cin>>name;

Input:

Prashant Sharma

Output:

Prashant

Fix:

getline(cin,name);

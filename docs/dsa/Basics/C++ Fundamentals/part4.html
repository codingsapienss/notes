Input / Output Handling

Programs become useful only when they can interact with users.

Input allows a program to receive data.

Output allows a program to display results.

What is Input and Output?​

User Input
     ↓
 Program
     ↓
Output Result

Example:

Enter Age: 25

Your Age is 25

Output in C++​

Output is performed using:

cout

where:

cout

stands for:

Character Output

Basic Output​

#include <iostream>
using namespace std;

int main()
{
    cout << "Hello World";

    return 0;
}
Output​
Hello World

Printing Multiple Values​

int age = 25;

cout << "Age: " << age;
Output​
Age: 25

The Insertion Operator (<<)​

Used to send data to output stream.

cout << "Hello";

Read as:

Insert "Hello" into cout

Multiple Outputs​

cout << "Age: " << 25 << " Years";
Output​
Age: 25 Years

New Line​

Using endl​

cout << "Hello" << endl;
cout << "World";
Output​
Hello
World

Using \n​

cout << "Hello\n";
cout << "World";
Output​
Hello
World

endl vs \n​

Featureendl\n
New LineYesYes
Flushes BufferYesNo
FasterNoYes

Recommendation​

Prefer:

"\n"

for better performance.

Input in C++​

Input is performed using:

cin

where:

Character Input

Basic Input​

int age;

cin >> age;
Input​
25
Output​
cout << age;
25

Extraction Operator (>>)​

Used to extract data from input stream.

cin >> age;

Read as:

Take data from cin and store in age

Multiple Inputs​

int a, b;

cin >> a >> b;
Input​
10 20
Values​
a = 10
b = 20

Example Program​

#include <iostream>
using namespace std;

int main()
{
    int age;

    cout << "Enter Age: ";

    cin >> age;

    cout << "Age: " << age;

    return 0;
}

Problem with cin​

Consider:

string name;

cin >> name;

Input:

Prashant Sharma

Stored Value:

Prashant

Only first word is captured.

Why?​

cin >> stops reading when it encounters:

Space (' ')
Tab (\t)
Newline (\n)

getline()​

Used to read an entire line.

Syntax​

getline(cin, name);

Example​

string name;

getline(cin, name);

cout << name;

Input:

Prashant Sharma

Output:

Prashant Sharma

cin vs getline()​

Featurecin >>getline()
Reads Single WordYesYes
Reads SpacesNoYes
Stops at SpaceYesNo
Stops at EnterYesYes

Common Issue with getline()​

Example:

int age;

cin >> age;

string name;

getline(cin, name);

Input:

25
Prashant

Output:

name = ""

Why Does This Happen?​

After:

cin >> age;

the Enter key remains inside input buffer.

Memory:

25\n
   ↑
Remaining

getline() immediately reads that leftover newline.

Solution: cin.ignore()​

int age;

cin >> age;

cin.ignore();

string name;

getline(cin, name);

Now:

Input:

25
Prashant Sharma

Output:

Prashant Sharma

Input Buffer​

Whenever user enters data:

Keyboard
   ↓
Input Buffer
   ↓
cin

Example

Input:

10 20 30

Buffer:

10 20 30

After:

cin >> a;

Buffer:

20 30

After:

cin >> b;

Buffer:

30

Input Failure​

Example:

int age;

cin >> age;

Input:

abc

Result:

Input Failure

Checking Input Failure​

int age;

cin >> age;

if(cin.fail())
{
    cout << "Invalid Input";
}

Clearing Input Error​

cin.clear();

Used to clear error flags.

Example

cin.clear();
cin.ignore(1000, '\n');

Output Formatting​

fixed​

double pi = 3.1415926535;

cout << fixed << pi;

Output:

3.141593

setprecision()​

Requires:

#include <iomanip>

Example

cout << fixed << setprecision(2);

cout << 3.14159;

Output:

3.14

Example​

#include <iostream>
#include <iomanip>

using namespace std;

int main()
{
    double price = 99.9999;

    cout << fixed
         << setprecision(2)
         << price;
}

Output:

100.00

Common Beginner Mistakes​

Forgetting &​

Wrong:

cin << age;

Correct:

cin >> age;

Using cin for Full Names​

Wrong:

cin >> name;

Input:

Prashant Sharma

Output:

Prashant

Use:

getline()

instead.

Mixing cin and getline​

Wrong:

cin >> age;

getline(cin, name);

Correct:

cin >> age;

cin.ignore();

getline(cin, name);

Interview Questions​

Q1. Difference between cin and cout?​

Answer​
cin

takes input.

cout

produces output.

Q2. Difference between endl and \n?​

Answer​

Both create a new line.

endl also flushes the output buffer.

Q3. Why does cin >> stop at spaces?​

Answer​

Because space, tab and newline are treated as delimiters.

Q4. When should we use getline()?​

Answer​

When input may contain spaces.

Q5. Why is cin.ignore() used?​

Answer​

To remove leftover newline characters from the input buffer.

Cheat Sheet​

cout << "Hello";

cout << "Hello\n";

cout << "Hello" << endl;
int age;

cin >> age;
string name;

getline(cin, name);
cin.ignore();
if(cin.fail())
{
    cout << "Invalid Input";
}
cout << fixed << setprecision(2);

Key Takeaways​

  • cout is used for output.
  • cin is used for input.
  • << is the insertion operator.
  • >> is the extraction operator.
  • \n and endl create new lines.
  • cin >> stops at space, tab, and newline.
  • getline() reads entire lines including spaces.
  • cin.ignore() helps when mixing cin and getline().
  • Input is processed through an input buffer.
  • fixed and setprecision() are used for formatted output.

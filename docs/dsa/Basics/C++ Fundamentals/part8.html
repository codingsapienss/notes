Introduction​

Strings in C++ look simple:

string name="Prashant";

but internally they involve:

  • arrays
  • pointers
  • memory management
  • null terminators
  • heap allocation
  • object management
  • compiler behavior

Many serious bugs in C and C++ happen because developers do not properly understand:

  • character arrays
  • string literals
  • C-style strings
  • writable vs read-only memory
  • null termination

This chapter explains everything deeply.

The 4 Most Important Concepts​

Most beginners confuse these:

'A'
"Hello"
char arr[]="Hello";
string s="Hello";

These are NOT the same.

Quick Overview​

ConceptExampleMeaning
Character Literal'A'Single character
String Literal"Hello"Read-only null-terminated string
Character Arraychar arr[10]Array storing characters
C-Style Stringchar arr[]="Hello"Null-terminated character array
C++ Stringstring s="Hello"Dynamic string object

Character (char)​

Definition​

char is a primitive data type used to store a single character.

Syntax​

char ch='A';

Important Rule​

Characters use:

''

single quotes.

Internal Storage​

Characters are internally stored using ASCII values.

Example:

char ch='A';

Internally:

'A'

↓

65

Memory:

Address    Value

1000       65

ASCII Table​

CharacterASCII Value
A65
B66
a97
b98
048
149

Example​

#include<iostream>
using namespace std;

int main()
{
    char ch='A';

    cout<<ch<<endl;

    cout<<(int)ch;
}

Expected Output:

A
65

Code Explanation​

(int)ch

typecasts character into integer.

Character Literal vs String Literal​

Character Literal​

'A'

Meaning:

Single character

Size:

1 byte

String Literal​

"Hello"

Meaning:

Sequence of characters ending with '\0'

Internally:

H e l l o \0

Character Array​

Definition​

Array capable of storing characters.

Syntax:

char arr[10];

Important Distinction​

This:

char arr[5]={'H','e','l','l','o'};

is NOT a proper C-style string.

It is ONLY:

Character array

Reason:

No '\0'

Memory Layout​

char arr[5]={'H','e','l','l','o'};

Memory:

Index:

0 1 2 3 4

Value:

H e l l o

Notice:

No null terminator exists

Dangerous Consequence​

cout<<arr;

Possible output:

Hello▒╬@...

or even crash.

Why Does This Happen?​

cout assumes:

Character arrays are null terminated

So printing continues until it accidentally finds:

'\0'

somewhere later in memory.

This is:

Undefined Behavior

Correct Manual Initialization​

char arr[6]={'H','e','l','l','o','\0'};

Memory:

Index:

0 1 2 3 4 5

Value:

H e l l o \0

Expected Output​

cout<<arr;

Output:

Hello

Best Initialization Method​

char arr[]="Hello";

Compiler automatically adds:

'\0'

Internally:

H e l l o \0

Actual size becomes:

6

because:

5 characters + '\0'

Important Compilation Rule​

Correct:

char arr[6]="Hello";

Required memory:

H e l l o \0

Total:

6 bytes

Wrong Example​

char arr[5]="Hello";

Why Wrong?​

Needs:

6 positions

because:

Hello + '\0'

Expected Result​

Usually:

Compilation Error

or compiler warning.

Correct Mental Model​

Without:

'\0'

this:

char arr[]={'H','e','l','l','o'};

is NOT a proper string.

It is ONLY:

array of characters

Null Character ('\0')​

Definition​

Null character:

'\0'

ASCII value:

0

Purpose:

Marks end of C-style string

Why Null Character Exists​

Suppose:

char arr[]="Prashant";

Memory:

P r a s h a n t \0

When printing:

cout<<arr;

Execution:

P
r
a
s
h
a
n
t

'\0' found

STOP

Important Example​

char arr[]={'a','\0','b','\0','c'};

cout<<arr;

Expected Output:

a

Why?​

Printing stops at first:

'\0'

String Literal​

Definition​

Text enclosed in:

""

Example:

"Hello"

Internal Representation​

H e l l o \0

Compiler automatically appends:

'\0'

MOST IMPORTANT CONCEPT​

String literals are:

READ ONLY

Dangerous Example​

char *ptr="Hello";

ptr[0]='M';

Why Dangerous?​

String literal stored in:

Read-only memory segment

Attempting modification causes:

  • Undefined behavior
  • Segmentation fault
  • Crash

Internal Memory Layout​

ptr
 ↓
"H e l l o \0"

Pointer points to read-only memory.

Correct Writable Version​

char arr[]="Hello";

arr[0]='M';

cout<<arr;

Expected Output:

Mello

Why Does This Work?​

Because:

char arr[]="Hello";

creates COPY of literal into writable memory.

Usually:

Stack Memory

Character Array vs String Literal​

FeatureCharacter ArrayString Literal
Examplechar arr[]="Hello""Hello"
WritableYesNo
MemoryStackRead-only segment
Can modifyYesNo
Has \0YesYes

C-Style String​

Definition​

A C-style string is:

Character array ending with '\0'

Important Rule​

This:

char arr[]="Hello";

IS a proper C-style string.

This:

char arr[]={'H','e','l','l','o'};

is NOT.

How Length Is Determined​

C-style strings do NOT store length.

Functions like:

strlen()

scan memory until:

'\0'

Example​

#include<iostream>
#include<cstring>

using namespace std;

int main()
{
    char arr[]="Hello";

    cout<<strlen(arr);
}

Expected Output:

5

Internal Working​

Memory:

H e l l o \0

Execution:

H → count 1
e → count 2
l → count 3
l → count 4
o → count 5

'\0' found

STOP

Complexity​

O(N)

because full scan required.

Dangerous Example​

char arr[5]={'H','e','l','l','o'};

cout<<strlen(arr);

Problem​

No null terminator exists.

strlen() keeps scanning memory.

Undefined behavior.

Input Into Character Array​

Method 1 — cin​

char name[20];

cin>>name;

Input:

Prashant

Expected Output:

Prashant

Hidden Behavior​

Input:

Prashant Sharma

Expected Output:

Prashant

Why?​

cin stops at:

  • Space ' '
  • Tab '\t'
  • Newline '\n'

Internal Buffer Behavior​

Input:

Prashant Sharma

Stored:

P r a s h a n t \0

Remaining buffer:

Sharma

still exists in input stream.

Safe Full-Line Input​

char name[50];

cin.getline(name,50);

Input:

Prashant Sharma

Expected Output:

Prashant Sharma

Buffer Overflow Problem​

Dangerous Example​

char name[5];

cin>>name;

Input:

Prashant

Problem​

Memory allocated:

Only 5 bytes

Input requires:

9 bytes

including:

'\0'

Possible Results​

  • Memory corruption
  • Crash
  • Segmentation fault
  • Undefined behavior

Output Behavior​

Character Output​

char ch='A';

cout<<ch;

Expected Output:

A

Character Array Output​

char arr[]="Hello";

cout<<arr;

Expected Output:

Hello

Important Hidden Behavior​

This works because:

cout

has special overload for:

char*

Compare With Integer Array​

int arr[]={1,2,3};

cout<<arr;

Expected Output:

Memory Address

Why?​

Only character pointers treated specially as strings.

C++ String (std::string)​

Definition​

C++ string is an STL class.

Header:

#include<string>

Syntax:

string s="Hello";

Important Correction​

Wrong:

String is 1D character array

Correct:

std::string is a class that internally manages dynamic character arrays.

Internal Structure​

string s="Hello";

Internally object stores:

  • Pointer
  • Size
  • Capacity

Memory:

String Object

↓

Pointer

↓

Heap Memory

↓

H e l l o \0

Dynamic Resizing​

Character array:

char arr[20];

Fixed size.

String:

string s;

Dynamic size.

Can grow automatically.

Example​

string s="Hello";

s+=" World";

cout<<s;

Expected Output:

Hello World

Size vs Capacity​

string s="Hello";

Possible:

Size = 5

Capacity = 15

Why Extra Capacity?​

To reduce repeated reallocations.

Same concept as vector.

Character Array vs C++ String​

FeatureCharacter ArrayC++ String
SizeFixedDynamic
Memory ManagementManualAutomatic
Null Terminator NeededYesInternally managed
SaferNoYes
Rich FunctionsNoYes
Easy ManipulationDifficultEasy

Array of Characters vs Array of Strings​

Array of Characters​

char arr[]="Hello";

Represents:

Single string

Memory:

H e l l o \0

Array of Strings​

string arr[3]=
{
"Hello",
"Hi",
"Bye"
};

Contains:

3 separate string objects

Memory Model​

arr[0] → Hello

arr[1] → Hi

arr[2] → Bye

Important STL String Functions​

length()​

string s="Hello";

cout<<s.length();

Expected Output:

5

size()​

cout<<s.size();

Expected Output:

5

empty()​

string s="";

cout<<s.empty();

Expected Output:

1

push_back()​

string s="Hell";

s.push_back('o');

cout<<s;

Expected Output:

Hello

pop_back()​

string s="Hello";

s.pop_back();

cout<<s;

Expected Output:

Hell

substr()​

string s="Prashant";

cout<<s.substr(2,4);

Expected Output:

asha

find()​

string s="Prashant";

cout<<s.find("sha");

Expected Output:

3

erase()​

string s="Prashant";

s.erase(2,3);

cout<<s;

Expected Output:

Prant

insert()​

string s="Prant";

s.insert(2,"ASH");

cout<<s;

Expected Output:

PrASHant

reverse()​

#include<algorithm>

reverse(
s.begin(),
s.end()
);

Example​

string s="Hello";

reverse(
s.begin(),
s.end()
);

cout<<s;

Expected Output:

olleH

Important C String Functions​

Header:

#include<cstring>

strlen()​

char arr[]="Hello";

cout<<strlen(arr);

Expected Output:

5

strcpy()​

char src[]="Hello";

char dest[10];

strcpy(dest,src);

cout<<dest;

Expected Output:

Hello

strcat()​

char a[20]="Hello ";

char b[]="World";

strcat(a,b);

cout<<a;

Expected Output:

Hello World

strcmp()​

cout<<strcmp("ABC","ABC");

Expected Output:

0

strcmp Return Meaning​

ResultMeaning
0Equal
<0First smaller
>0First greater

sizeof vs strlen​

Example​

char arr[]="Hello";

cout<<sizeof(arr)<<endl;

cout<<strlen(arr);

Expected Output:

6
5

Why Different?​

Memory:

H e l l o \0

sizeof()​

Counts:

Entire memory

including:

'\0'

strlen()​

Counts:

Characters before '\0'

SituationRecommended

Key Takeaways​

  • Character arrays and strings are NOT identical
  • '\0' is critical in C-style strings
  • Missing null terminator causes undefined behavior
  • String literals are read-only
  • Character arrays are writable
  • std::string is safest and preferred in modern C++
  • strlen() depends on '\0'
  • sizeof() and strlen() are different
  • cout treats character arrays specially
  • Buffer overflow is dangerous

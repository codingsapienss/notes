Control Flow & Decision Making

By default, a program executes statements from top to bottom.

However, real-world programs often need to make decisions:

  • Is the user eligible to vote?
  • Is the password correct?
  • Is a number positive or negative?

Control flow statements allow programs to choose different execution paths based on conditions.

What is Control Flow?​

Control Flow determines:

Which statements execute
When they execute
How many times they execute

Default Flow​

cout << "A";
cout << "B";
cout << "C";
Output​
ABC

Execution:

A
↓
B
↓
C

Conditional Statements​

Conditional statements allow execution of code only when a condition is true.

if Statement​

Syntax​

if(condition)
{
    // code
}

Example​

int age = 20;

if(age >= 18)
{
    cout << "Eligible to Vote";
}
Output​
Eligible to Vote

How It Works​

Condition Checked
       ↓
     True
       ↓
 Execute Block

if Statement (False Condition)​

int age = 15;

if(age >= 18)
{
    cout << "Eligible";
}
Output​
No Output

Because:

Condition = False

if-else Statement​

Used when two possible paths exist.

Syntax​

if(condition)
{
    // true block
}
else
{
    // false block
}

Example​

int age = 15;

if(age >= 18)
{
    cout << "Eligible";
}
else
{
    cout << "Not Eligible";
}
Output​
Not Eligible

Flow Diagram​

Condition
    │
 ┌──┴──┐
 │     │
True False
 │     │
 A     B

else if Ladder​

Used when multiple conditions must be checked.

Syntax​

if(condition1)
{
}
else if(condition2)
{
}
else if(condition3)
{
}
else
{
}

Example​

int marks = 75;

if(marks >= 90)
{
    cout << "Grade A";
}
else if(marks >= 80)
{
    cout << "Grade B";
}
else if(marks >= 70)
{
    cout << "Grade C";
}
else
{
    cout << "Fail";
}
Output​
Grade C

Important Point​

As soon as one condition becomes true:

Remaining conditions are skipped.

Nested if​

An if statement inside another if.

Example​

int age = 20;
bool hasLicense = true;

if(age >= 18)
{
    if(hasLicense)
    {
        cout << "Can Drive";
    }
}
Output​
Can Drive

Flow​

Age Check
    ↓
License Check
    ↓
Result

switch Statement​

Used when comparing one variable against multiple fixed values.

Syntax​

switch(variable)
{
    case value1:
        break;

    case value2:
        break;

    default:
}

Example​

int day = 2;

switch(day)
{
    case 1:
        cout << "Monday";
        break;

    case 2:
        cout << "Tuesday";
        break;

    case 3:
        cout << "Wednesday";
        break;

    default:
        cout << "Invalid";
}
Output​
Tuesday

Why break is Important?​

Without:

break;

execution continues into the next case.

Example​

int day = 1;

switch(day)
{
    case 1:
        cout << "Monday";

    case 2:
        cout << "Tuesday";

    case 3:
        cout << "Wednesday";
}
Output​
MondayTuesdayWednesday

This behavior is called:

Fallthrough

default Case​

Executed when no case matches.

Example​

int day = 10;

switch(day)
{
    case 1:
        cout << "Monday";
        break;

    default:
        cout << "Invalid Day";
}
Output​
Invalid Day

if-else vs switch​

Featureif-elseswitch
Range CheckingYesNo
Multiple ConditionsYesLimited
ReadabilityGoodBetter for fixed values
Supports ExpressionsYesNo

break Statement​

Used to immediately exit a loop or switch.

Example​

for(int i=1;i<=10;i++)
{
    if(i==5)
    {
        break;
    }

    cout << i << " ";
}
Output​
1 2 3 4

continue Statement​

Used to skip the current iteration and move to the next one.

Example​

for(int i=1;i<=5;i++)
{
    if(i==3)
    {
        continue;
    }

    cout << i << " ";
}
Output​
1 2 4 5

Ternary Operator​

Short form of if-else.

Syntax​

condition ? value1 : value2;

Example​

int age = 20;

string result =
(age >= 18)
? "Eligible"
: "Not Eligible";

cout << result;
Output​
Eligible

Common Beginner Mistakes​

Using = Instead of ==​

Wrong

if(age = 18)

Correct

if(age == 18)

Forgetting break in switch​

Wrong

case 1:
    cout << "Monday";

May cause fallthrough.

Writing Multiple if Instead of else if​

Wrong

if(marks >= 90)
{
}

if(marks >= 80)
{
}

Both conditions may execute.

Better

if(...)
{
}
else if(...)
{
}

Interview Questions​

Q1. Difference between if and switch?​

Answer​

if can evaluate ranges and complex expressions.

switch works with fixed values.

Q2. What is fallthrough?​

Answer​

Execution of subsequent switch cases when break is omitted.

Q3. What does break do?​

Answer​

Immediately exits the current loop or switch.

Q4. What does continue do?​

Answer​

Skips the current iteration and moves to the next one.

Q5. When should switch be preferred?​

Answer​

When comparing one variable against multiple fixed values.

Key Takeaways​

  • Control flow determines program execution paths.
  • if executes code when a condition is true.
  • if-else provides two possible execution paths.
  • else if is used for multiple conditions.
  • Nested if statements allow hierarchical decisions.
  • switch is useful for fixed-value comparisons.
  • break prevents fallthrough in switch statements.
  • continue skips the current iteration.
  • The ternary operator provides a compact form of if-else.
  • Choosing the right decision-making statement improves readability and maintainability.

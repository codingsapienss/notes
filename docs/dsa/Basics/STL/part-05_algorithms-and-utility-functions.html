Introduction to STL Algorithms

Intuition​

Suppose you have:

int arr[]={5,2,7,1,3};

Without STL:

- Write sorting algorithm
- Write searching algorithm
- Write max logic
- Write permutation logic

Problems:

  • Time consuming
  • More bugs
  • Reinventing existing solutions

STL algorithms solve this.

Why Algorithms Exist

Algorithms provide:

  • Optimized implementations
  • Less code
  • Better readability
  • Reduced bugs
  • High performance

sort()

Definition​

Used for sorting containers.

Header:

#include<algorithm>

Sorting Arrays

Syntax​

sort(arr,arr+n);

Example​

#include<iostream>
#include<algorithm>

using namespace std;

int main()
{
    int arr[]={5,2,8,1,9};

    int n=5;

    sort(arr,arr+n);

    for(int x:arr)
    {
        cout<<x<<" ";
    }
}

Output:

1 2 5 8 9

Code Explanation​

sort(arr,arr+n);

Start:

arr

points to:

First element

End:

arr+n

points:

One position after last element

Range:

[start,end)

Sorting Vector

Syntax:

sort(v.begin(),v.end());

Example

vector<int> v={4,1,8,2};

sort(v.begin(),v.end());

for(auto x:v)
{
    cout<<x<<" ";
}

Output:

1 2 4 8

Sorting Particular Portion

Syntax:

sort(start,end);

Example

int arr[]={7,4,2,9,1};

sort(arr+1,arr+4);

Before:

7 4 2 9 1

Sorted portion:

4 2 9

Output:

7 2 4 9 1

Sorting Descending Order

Syntax​

sort(arr,arr+n,greater<int>());

Example

int arr[]={5,2,8,1};

sort(arr,arr+4,greater<int>());

for(int x:arr)
{
    cout<<x<<" ";
}

Output:

8 5 2 1

Internal Working of sort()

STL sort internally uses:

IntroSort

IntroSort combines:

Quick Sort
        +
Heap Sort
        +
Insertion Sort

Why Not Only Quick Sort?

Quick sort worst case:

O(N²)

IntroSort avoids that.

Complexity

CaseComplexity
BestO(NlogN)
AverageO(NlogN)
WorstO(NlogN)

Custom Comparator

Why Custom Sorting?​

Default sorting:

Ascending

Sometimes requirements differ.

Example:

Sort by second value

If second equal:

Sort first in descending

Example

Input:

pair<int,int> arr[]=
{
    {1,2},
    {2,1},
    {4,1}
};

Requirement:

Sort by second

If same:

Sort first descending

Comparator Function

bool comp(
pair<int,int> p1,
pair<int,int> p2)
{

    if(p1.second<p2.second)
        return true;

    if(p1.second>p2.second)
        return false;

    return p1.first>p2.first;

}

Use Comparator

sort(arr,arr+3,comp);

Output:

4 1

2 1

1 2

How Comparator Works

Rule:

return true

means:

Keep first element before second

Wrong Comparator

Wrong:

bool comp(int a,int b)
{
    return a>=b;
}

Reason:

Comparator should satisfy:

Strict Weak Ordering

Use:

return a>b;

instead.

__builtin_popcount()

Definition​

Returns:

Number of set bits

Set bit:

Bit = 1

Example:

int num=7;

cout<<__builtin_popcount(num);

Binary:

7

111

Output:

3

Internal Working

Binary:

13

1101

Count:

1+1+0+1

Output:

3

__builtin_popcountll()

Used for:

long long

Example:

long long num=123456789123;

cout<<__builtin_popcountll(num);

Why Separate Function?

Normal:

__builtin_popcount()

works on:

int

Range:

32 bits

Long long:

64 bits

next_permutation()

Definition​

Creates next lexicographically larger permutation.

Syntax​

next_permutation(
s.begin(),
s.end()
);

Example

string s="123";

next_permutation(
s.begin(),
s.end()
);

cout<<s;

Output:

132

Internal Working

Input:

123

Permutations:

123
132
213
231
312
321

Next after:

123

is:

132

Print All Permutations

Important rule:

Always sort first.

Example

#include<iostream>
#include<algorithm>

using namespace std;

int main()
{
    string s="321";

    sort(s.begin(),s.end());

    do
    {
        cout<<s<<endl;

    }
    while(
    next_permutation(
    s.begin(),
    s.end()
    ));

}

Output:

123
132
213
231
312
321

Why Sort First?

Wrong:

string s="321";

Output becomes incomplete.

Because:

next_permutation()

moves forward only.

max_element()

Definition​

Returns iterator pointing to largest element.

Syntax:

max_element(start,end);

Example

int arr[]={4,7,2,9};

int mx=
*max_element(
arr,
arr+4
);

cout<<mx;

Output:

9

Internal Working

Process:

Current max=4

7 >4

Current max=7

2<7

9>7

Current max=9

min_element()

Returns iterator of minimum element.

Example

int arr[]={4,7,2,9};

int mn=
*min_element(
arr,
arr+4
);

cout<<mn;

Output:

2

Time Complexity

FunctionComplexity
sortO(NlogN)
max_elementO(N)
min_elementO(N)
next_permutationO(N)
popcountO(Number of Bits)

  • Rankings

next_permutation()​

Used in:

popcount()​

Used in:

max_element()​

Used in:

  • Analytics
  • Statistics

Common Bugs

Bug 1​

Wrong:

sort(arr,arr+n-1);

Problem:

Last element excluded.

Correct:

sort(arr,arr+n);

Bug 2​

Wrong:

max_element(arr,arr+n);

Problem:

Returns iterator.

Correct:

*max_element(arr,arr+n)

Bug 3​

Wrong:

__builtin_popcount(longLongValue)

Use:

__builtin_popcountll()

Bug 4​

Wrong:

string s="321";

do
{
    cout<<s;

}
while(next_permutation(
s.begin(),
s.end()
));

Output incomplete.

Fix:

sort(s.begin(),s.end());

Interview Questions

Q1​

What algorithm does STL sort use?

Answer:

IntroSort

Q2​

Why is IntroSort better than QuickSort?

Answer:

Avoids O(N²) worst case.

Q3​

Why does max_element return iterator?

Answer:

Works uniformly with STL containers.

Q4​

Why sort before next_permutation?

Answer:

To generate all permutations.

Q5​

Difference:

__builtin_popcount()

vs

__builtin_popcountll()

Answer:

FunctionData Type
popcountint
popcountlllong long

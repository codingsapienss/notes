Arrays

Intuition​

Suppose you have marks of 5 students:

90
85
78
92
88

One approach:

int a=90;
int b=85;
int c=78;
int d=92;
int e=88;

Problems:

  • Too many variables
  • Hard to maintain
  • Difficult to process with loops
  • Difficult to scale

Arrays solve this problem.

Arrays allow us to store multiple values under a single variable name.

Real-world analogy:

Apartment Building

Room 0 → Person A
Room 1 → Person B
Room 2 → Person C
Room 3 → Person D

Similarly:

arr[0] → 10
arr[1] → 20
arr[2] → 30
arr[3] → 40

Definition​

Array is a collection of elements of the same data type stored in contiguous memory locations.

Why Arrays Exist​

Arrays solve:

  • Storage organization
  • Fast access
  • Efficient memory use
  • Easy iteration

Internal Working​

Array stores elements continuously in memory.

Example:

int arr[4]={10,20,30,40};

Memory representation:

Address      Value

1000         10
1004         20
1008         30
1012         40

Since:

sizeof(int)=4 bytes

Address calculation:

Address of element

= Base Address + (Index × Size of datatype)

Accessing:

arr[2]

Compiler converts internally:

*(arr+2)

Calculation:

1000 + (2×4)

1008

Value:

30

Why Array Access Is Fast​

Array indexing:

arr[index]

requires direct address calculation.

No traversal needed.

Time complexity:

OperationComplexity
AccessO(1)

Syntax

datatype arrayName[size];

Example:

int arr[5];

Array Initialization

Method 1​

int arr[5]={1,2,3,4,5};

Method 2​

Compiler automatically calculates size.

int arr[]={1,2,3,4};

Method 3​

Initialize all values with zero.

int arr[5]={0};

Output:

0 0 0 0 0

Method 4​

Partial initialization

int arr[5]={1,2};

Output:

1 2 0 0 0

Remaining values become:

0

Basic Example

#include<iostream>
using namespace std;

int main()
{
    int arr[5]={10,20,30,40,50};

    cout<<arr[0]<<endl;
    cout<<arr[2]<<endl;
    cout<<arr[4];

    return 0;
}

Code Explanation​

int arr[5]

Creates an array of size 5.

arr[0]

Accesses first element.

arr[2]

Accesses third element.

Execution:

arr[0] → 10

arr[2] → 30

arr[4] → 50

Array Properties

PropertyValue
Memory LayoutContiguous
SizeFixed
Random AccessYes
Dynamic ResizeNo
Insert MiddleO(N)
Delete MiddleO(N)
AccessO(1)

Array Iteration Methods

Method 1 — Traditional Loop​

#include<iostream>
using namespace std;

int main()
{
    int arr[]={1,2,3,4,5};

    for(int i=0;i<5;i++)
    {
        cout<<arr[i]<<" ";
    }
}

Output:

1 2 3 4 5

Method 2 — Pointer Iteration​

#include<iostream>
using namespace std;

int main()
{
    int arr[]={10,20,30};

    for(int i=0;i<3;i++)
    {
        cout<<*(arr+i)<<" ";
    }
}

Explanation:

arr+i

moves pointer.

*(arr+i)

dereferences pointer.

Method 3 — Range-based Loop​

#include<iostream>
using namespace std;

int main()
{
    int arr[]={5,6,7};

    for(auto x:arr)
    {
        cout<<x<<" ";
    }
}

Output:

5 6 7

Pair

Intuition​

Suppose:

Student ID → Marks

101 → 90
102 → 85
103 → 76

Using separate arrays:

int id[3];
int marks[3];

This becomes difficult.

Pair combines two related values.

Definition​

Pair stores two values together.

Header:

#include<utility>

Syntax​

pair<int,int> p;

Assign Values​

pair<int,int> p;

p={1,2};

Access:

cout<<p.first;

cout<<p.second;

Basic Example​

#include<iostream>
#include<utility>

using namespace std;

int main()
{
    pair<int,int> p={10,20};

    cout<<p.first<<endl;

    cout<<p.second;
}

Output:

10
20

Nested Pair​

pair<int,pair<int,int>> p;

p={1,{2,3}};

cout<<p.second.second;

Output:

3

Array of Pairs​

pair<int,int> arr[]=
{
    {1,2},
    {3,4},
    {5,6}
};

cout<<arr[1].second;

Output:

4

Vector

Intuition​

Arrays have a limitation:

Fixed size

Problem:

int arr[100];

Case 1:

Need only:

5 elements

Memory wasted.

Case 2:

Need:

150 elements

Not enough memory.

Vector solves this problem.

Important Correction

Wrong:

Vector internally uses linked list

Correct:

Vector internally uses dynamic contiguous memory

Vector internally behaves like a resizable array.

Internal Working

Initially:

vector<int> v;

Memory:

Size = 0
Capacity = 0

Insert:

v.push_back(10);

Memory:

10

Size:

1

Capacity:

1

Insert:

v.push_back(20);

Memory:

10 20

Size:

2

Capacity:

2

Insert:

v.push_back(30);

Vector becomes full.

New memory allocation:

Old memory:

10 20

↓

New memory:

10 20 30 _

Capacity grows.

Usually:

1

↓

2

↓

4

↓

8

↓

16

Size vs Capacity

SizeCapacity
Actual elementsAllocated memory

Example:

vector<int> v;

v.push_back(1);
v.push_back(2);
v.push_back(3);

cout<<v.size();
cout<<v.capacity();

Possible output:

3
4

Vector Initialization

Empty Vector​

vector<int> v;

Fixed Size​

vector<int> v(5);

Output:

0 0 0 0 0

Fixed Size With Value​

vector<int> v(5,100);

Output:

100 100 100 100 100

Copy Vector​

vector<int> v1={1,2,3};

vector<int> v2(v1);

Element Access

cout<<v[0];

cout<<v.at(1);

cout<<v.back();

Push Back vs Emplace Back

push_back()​

Creates object first.

Then inserts.

v.push_back(10);

emplace_back()​

Constructs object directly inside memory.

v.emplace_back(10);

Example​

vector<pair<int,int>> v;

v.push_back({1,2});

v.emplace_back(3,4);

Should We Always Use emplace_back() ?​

No.

For:

int
char
double

Difference is negligible.

More useful for:

  • classes
  • objects
  • complex data structures

Iterators

Definition​

Iterator behaves like a pointer.

Used for traversing containers.

Syntax​

vector<int>::iterator it=v.begin();

Using auto:

auto it=v.begin();

begin()​

Points to first element.

10 20 30 40
↑
begin()

end()​

Does NOT point to last element.

Points after last element.

10 20 30 40
            ↑
          end()

rbegin()​

10 20 30 40
            ↑
        rbegin()

rend()​

10 20 30 40
↑
rend()

Reverse Iterator Increment​

auto it=v.rbegin();

it++;

Movement:

40 → 30 → 20 → 10

Iterator Loop

for(auto it=v.begin();it!=v.end();it++)
{
    cout<<*it<<" ";
}

Reverse Loop

for(auto it=v.rbegin();it!=v.rend();it++)
{
    cout<<*it<<" ";
}

auto

Why use auto​

Without auto:

vector<pair<int,int>>::iterator it=v.begin();

Large declaration.

Using auto:

auto it=v.begin();

Compiler detects type automatically.

const

Prevents modification.

Example:

const int x=10;

Wrong:

x=20;

Compiler error.

Using const reference:

for(const auto &x:v)
{
    cout<<x;
}

Benefits:

  • No copying
  • No modification
  • Better performance

Range-based Loop

Syntax:

for(auto x:v)
{

}

Hidden Behavior​

Wrong:

for(auto x:v)
{
    x+=5;
}

No change occurs.

Reason:

x

is copied.

Correct:

for(auto &x:v)
{
    x+=5;
}

Complexity Summary

OperationArrayVector
AccessO(1)O(1)
Insert EndN/AO(1) amortized
Delete EndN/AO(1)

Delete MiddleO(N)O(N)

Common Mistakes

Mistake 1​

auto it=v.end();

cout<<*it;

Problem:

end()

points after last element.

Undefined behavior.

Mistake 2​

for(auto x:v)
{
    x++;
}

Only copy modified.

Mistake 3​

int arr[5];

cout<<arr[0];

Contains garbage value.

Interview Questions

Q1​

Why vector indexing is O(1)?

Answer:

Because vector stores elements in contiguous memory.

Q2​

Difference between size and capacity?

Answer:

Size:

Actual elements

Capacity:

Allocated memory

Q3​

Why insertion in middle of vector is expensive?

Answer:

Elements must shift.

Complexity:

O(N)

Cheat Sheet

vector<int> v;

v.push_back(10);

v.emplace_back(20);

v.begin();

v.end();

v.rbegin();

v.rend();

v.back();

v.size();

pair<int,int> p={1,2};

p.first;

p.second;

Key Takeaways

  • Arrays use contiguous memory
  • Vector uses dynamic contiguous memory
  • Vector is not linked list
  • Size and capacity are different
  • end() points after last element
  • auto reduces verbosity
  • const prevents modifications
  • Range loops may create copies

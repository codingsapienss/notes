Why Do Pointers Exist?​

Before learning more syntax, we need to answer:

Why were pointers invented?

Problem Without Pointers​

Suppose:

void increment(int value)
{
    value++;
}

Program​

#include<iostream>
using namespace std;

void increment(int value)
{
    value++;
}

int main()
{
    int number = 10;

    increment(number);

    cout << number;
}

Expected Output:

10

Why?​

Because:

increment(number);

creates a copy.

Memory:

number

Address: 1000
Value: 10

Function receives:

value

Address: 2000
Value: 10

Different memory locations.

Visual​

number
 10

↓

COPY

value
 10

Result​

Changing:

value++

changes only:

value

not:

number

How Can We Modify Original Variable?​

We must somehow provide:

Original memory address

to the function.

This is where pointers become useful.

Pass By Address​

Example:

void increment(int *ptr)
{
    (*ptr)++;
}

Program​

#include<iostream>
using namespace std;

void increment(int *ptr)
{
    (*ptr)++;
}

int main()
{
    int number = 10;

    increment(&number);

    cout << number;
}

Expected Output:

11

Execution Flow​

Before:

number

Address: 1000
Value: 10

Function call:

increment(&number);

passes:

1000

Inside function:

ptr = 1000

Operation:

(*ptr)++

means:

Go to address 1000

Increment value

Memory After

number

Address: 1000
Value: 11

Why Pointers Are Powerful​

Without pointers:

Only copies can be modified.

With pointers:

Original memory can be modified.

Pointer States​

A pointer can exist in multiple states.

Understanding these states prevents bugs.

Overview​

Pointer StateSafe?
Valid PointerYes
Null PointerYes
Wild PointerNo
Dangling PointerNo

Valid Pointer​

Definition​

Pointer contains address of valid memory.

Example

int value = 100;

int *ptr = &value;

Memory:

ptr

↓

value
100

Null Pointer​

Definition​

Pointer intentionally points nowhere.

Example

int *ptr = nullptr;

Memory

ptr

↓

NULL

Why Null Pointers Exist​

Instead of:

Random garbage address

we explicitly say:

Pointer currently points nowhere.

Example​

#include<iostream>
using namespace std;

int main()
{
    int *ptr = nullptr;

    cout << ptr;
}

Expected Output:

0

or

(nullptr)

depending on compiler.

Best Practice​

Always prefer:

int *ptr = nullptr;

instead of:

int *ptr;

Wild Pointer​

Definition​

Pointer declared but never initialized.

Example

int *ptr;

Memory

ptr

↓

Random Address

Example:

0xABCD1234

Why Dangerous?​

Program believes:

Valid memory exists there

even though it doesn't.

Example​

int *ptr;

cout << ptr;

Possible Output:

Random Address

Even Worse​

int *ptr;

cout << *ptr;

Possible Result:

Crash
Segmentation Fault
Undefined Behavior

Wild Pointer vs Null Pointer​

FeatureWild PointerNull Pointer
InitializedNoYes
SafeNoYes
Known StateNoYes
RecommendedNoYes

Dangling Pointer​

Definition​

Pointer that points to memory that no longer exists.

Example​

int *ptr;

{
    int number = 10;

    ptr = &number;
}

Memory​

Initially:

ptr

↓

number
10

After Block Ends​

{
    int number = 10;
}

Variable destroyed.

But:

ptr

still stores old address.

Visual​

ptr

↓

Memory No Longer Valid

Why Dangerous?​

Pointer thinks:

Object still exists

Reality:

Object already destroyed

Example​

int *ptr;

{
    int number = 10;

    ptr = &number;
}

cout << *ptr;

Result:

Undefined Behavior

Important Note​

We will study dangling pointers in detail later during:

Dynamic Memory Allocation

Value Copy vs Address Copy​

One of the most important interview concepts.

Value Copy​

Example:

int firstNumber = 100;

int secondNumber = firstNumber;

Memory​

firstNumber

Address: 1000
Value: 100


secondNumber

Address: 2000
Value: 100

Modify Second Variable​

secondNumber = 500;

Result:

firstNumber = 100

secondNumber = 500

Why?​

Because:

Value copied

not memory.

Address Copy​

Example

int value = 100;

int *pointerOne = &value;

int *pointerTwo = pointerOne;

Memory​

pointerOne

↓

value
100

pointerTwo

↓

value
100

Visual​

pointerOne ──┐
             │
             ▼

           value
            100

             ▲
             │

pointerTwo ──┘

Modify Through Second Pointer​

*pointerTwo = 999;

Result

value = 999

Example​

#include<iostream>
using namespace std;

int main()
{
    int value = 100;

    int *pointerOne = &value;

    int *pointerTwo = pointerOne;

    *pointerTwo = 999;

    cout << value;
}

Expected Output:

999

Comparison Table​

FeatureValue CopyAddress Copy
Copies ValueYesNo
Copies AddressNoYes
Shared MemoryNoYes
Modification ReflectsNoYes

Why Pass By Address Is Useful​

Consider swapping two numbers.

Without pointers:

void swapValues(int a,int b)
{
    int temp = a;
    a = b;
    b = temp;
}

Program​

#include<iostream>
using namespace std;

void swapValues(int a,int b)
{
    int temp = a;
    a = b;
    b = temp;
}

int main()
{
    int x = 10;
    int y = 20;

    swapValues(x,y);

    cout << x << " " << y;
}

Expected Output:

10 20

Why Failed?​

Function received:

Copies

Using Pointers​

void swapValues(
int *first,
int *second
)
{
    int temp = *first;

    *first = *second;

    *second = temp;
}

Program​

#include<iostream>
using namespace std;

void swapValues(
int *first,
int *second
)
{
    int temp = *first;

    *first = *second;

    *second = temp;
}

int main()
{
    int x = 10;
    int y = 20;

    swapValues(&x,&y);

    cout << x << " " << y;
}

Expected Output:

20 10

Why Successful?​

Function modifies:

Original memory locations

instead of copies.

Special Behavior of Character Pointers​

This is a very common interview trap.

Example​

char grade = 'A';

char *ptr = &grade;

cout << ptr;

Beginner Expectation​

Address

Actual Behavior​

Undefined / unexpected output.

Possibly:

A▒▒▒▒...

Why?​

Because:

cout

treats:

char*

as:

C-style string

Example​

char name[] = "Prashant";

char *ptr = name;

cout << ptr;

Expected Output:

Prashant

Why?​

cout assumes:

char*

points to a string.

How To Print Actual Address?​

Use:

cout << static_cast<void*>(ptr);

Example​

char name[] = "Hello";

char *ptr = name;

cout << static_cast<void*>(ptr);

Expected Output:

0x7fffdd3ef8c4

Pointer Safety Rules​

Rule 1​

Always initialize pointers.

Good:

int *ptr = nullptr;

Bad:

int *ptr;

Rule 2​

Never dereference null pointers.

Wrong:

*ptr

when:

ptr == nullptr

Rule 3​

Never dereference wild pointers.

Wrong:

int *ptr;

*ptr = 10;

Rule 4​

Do not use dangling pointers.

Rule 5​

When unsure:

ptr = nullptr;

Common Bugs​

Bug 1​

int *ptr = nullptr;

cout << *ptr;

Reason:

Null Pointer Dereference

Bug 2​

int *ptr;

*ptr = 100;

Reason:

Wild Pointer

Bug 3​

char ch='A';

char *ptr=&ch;

cout << ptr;

Reason:

cout treats char* specially

Bug 4​

int *ptr;

{
    int x = 10;

    ptr = &x;
}

cout << *ptr;

Reason:

Dangling Pointer

Interview Questions​

Q1​

Why do pointers exist?

Answer​

To work with original memory addresses and allow direct memory manipulation.

Q2​

Difference between null pointer and wild pointer?

Answer​

Null PointerWild Pointer
InitializedUninitialized
Safe StateUnsafe State
Known ValueGarbage Value

Q3​

What is a dangling pointer?

Answer​

Pointer pointing to memory that no longer exists.

Q4​

Difference between value copy and address copy?

Answer​

Value copy duplicates data.

Address copy shares access to same memory.

Q5​

Why does:

cout << charPointer;

behave differently?

Answer​

Because cout treats char* as a C-style string.

Cheat Sheet​

int *ptr = nullptr;

int *ptr;

*ptr;

(*ptr)++;

void increment(int *ptr)
{
    (*ptr)++;
}

increment(&value);

int *p1 = &value;

int *p2 = p1;

char *ptr = name;

cout << static_cast<void*>(ptr);

Key Takeaways​

  • Pointers exist to work with original memory.
  • Pass by value creates copies.
  • Pass by address allows modification of original data.
  • Null pointers are safe placeholders.
  • Wild pointers are dangerous.
  • Dangling pointers reference destroyed memory.
  • Value copy and address copy are fundamentally different.
  • Multiple pointers can point to the same memory.
  • char* has special behavior with cout.
  • Always initialize pointers.

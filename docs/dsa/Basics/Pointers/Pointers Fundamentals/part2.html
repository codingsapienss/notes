C++ Pointers Fundamentals

Dereference Operator, Memory Access, Pointer Copying, Call By Value Fundamentals​

In Part 1A we learned:

  • Memory addresses
  • Address-of operator (&)
  • Pointer declaration
  • Pointer initialization
  • Null pointers
  • Pointer sizes

But a major question still remains:

We stored the address.

Now what?

A pointer becomes useful only when we can access the memory it points to.

This chapter teaches:

  • Dereferencing
  • Reading values through pointers
  • Modifying values through pointers
  • Pointer copying
  • Value copying
  • Shared memory concepts
  • Call by Value fundamentals

Revisiting Pointer Basics​

Consider:

int score = 100;

int *scorePtr = &score;

Memory:

score

Address: 1000
Value:   100


scorePtr

Address: 2000
Value:   1000

Visual:

scorePtr
    │
    │ stores address
    ▼

 score
 100

The Big Question​

We know:

scorePtr

contains:

1000

which is the address of:

score

But how do we access:

100

stored at that address?

Dereference Operator (*)​

Definition​

The dereference operator accesses the value stored at the address held by a pointer.

Symbol:

*

Syntax​

*pointerName

Example:

*scorePtr

Meaning:

Go to the address stored inside scorePtr
and give me the value stored there.

Memory Diagram​

int score = 100;

int *scorePtr = &score;

Memory:

score

Address: 1000
Value:   100


scorePtr

Address: 2000
Value:   1000

Evaluation:

*scorePtr

Step 1:

scorePtr contains 1000

Step 2:

Go to address 1000

Step 3:

Read value stored there

Result:

100

First Dereference Example​

#include<iostream>
using namespace std;

int main()
{
    int score = 100;

    int *scorePtr = &score;

    cout << *scorePtr;
}

Expected Output:

100

Code Explanation​

scorePtr

contains:

address of score
*scorePtr

means:

value stored at address inside scorePtr

Pointer vs Dereferenced Pointer​

One of the most important concepts.

Pointer​

scorePtr

stores:

Address

Example:

0x7fffdd3ef8c4

Dereferenced Pointer​

*scorePtr

stores:

Value at that address

Example:

100

Example​

cout << scorePtr << endl;

cout << *scorePtr;

Expected Output:

0x7fffdd3ef8c4
100

Common Interview Question​

What is the difference between:

ptr

and

*ptr

Answer:

ExpressionMeaning
ptrAddress
*ptrValue at address

Relationship Between Variable and Dereferenced Pointer​

Consider:

int marks = 95;

int *marksPtr = &marks;

Memory​

Address: 1000

Value: 95

Access Through Variable​

cout << marks;

Output:

95

Access Through Pointer​

cout << *marksPtr;

Output:

95

Important Observation​

Both produce:

95

Why?

Because both ultimately refer to:

same memory location

Visual Diagram​

marks
  │
  │
  ▼

Address 1000
Value 95

  ▲
  │
  │

*marksPtr

Key Rule​

For a valid pointer:

marks == *marksPtr

in terms of value.

Writing Through Pointer​

Until now we only read values.

Now let's modify them.

Example​

int score = 100;

int *scorePtr = &score;

*scorePtr = 500;

Memory Before​

Address: 1000

Value: 100

Execution​

*scorePtr = 500;

means:

Go to address stored in scorePtr

Store 500 there

Memory After​

Address: 1000

Value: 500

Complete Program​

#include<iostream>
using namespace std;

int main()
{
    int score = 100;

    int *scorePtr = &score;

    *scorePtr = 500;

    cout << score;
}

Expected Output:

500

Why Did score Change?​

Many beginners think:

Pointer stores a copy

Wrong.

Pointer stores:

Address

not value.

Visual Representation​

Before:

score
100

Pointer:

scorePtr

↓

score
100

After:

*scorePtr = 500;

Memory becomes:

score
500

Reading and Writing Together​

#include<iostream>
using namespace std;

int main()
{
    int value = 200;

    int *valuePtr = &value;

    cout << "Before: "
         << value << endl;

    *valuePtr = 700;

    cout << "After: "
         << value;
}

Expected Output:

Before: 200

After: 700

Increment Through Pointer​

Example:

int value = 200;

int *valuePtr = &value;

(*valuePtr)++;

Execution​

Before:

200

Operation:

(*valuePtr)++

means:

Increment value stored
at target memory location.

After:

201

Example​

#include<iostream>
using namespace std;

int main()
{
    int value = 200;

    int *valuePtr = &value;

    (*valuePtr)++;

    cout << value;
}

Expected Output:

201

Important Distinction​

Many students confuse:

(*ptr)++

and

ptr++

These are NOT the same.

Version 1​

(*ptr)++

Changes:

Stored value

Version 2​

ptr++

Changes:

Pointer address

Pointer arithmetic will be covered later.

Copying Variables​

Now let's compare variable copying and pointer copying.

Example​

int firstNumber = 100;

int secondNumber = firstNumber;

Memory​

firstNumber

Address: 1000
Value:   100


secondNumber

Address: 2000
Value:   100

Important Observation​

Two separate variables.

Two separate memory locations.

Modify One​

secondNumber = 500;

Memory:

firstNumber = 100

secondNumber = 500

Example​

#include<iostream>
using namespace std;

int main()
{
    int firstNumber = 100;

    int secondNumber = firstNumber;

    secondNumber = 500;

    cout << firstNumber << endl;

    cout << secondNumber;
}

Expected Output:

100
500

Why?​

Because:

Value was copied

not memory.

Pointer Copying​

Now compare:

int value = 100;

int *pointerOne = &value;

int *pointerTwo = pointerOne;

Memory​

value

Address: 1000
Value:   100


pointerOne

Address: 2000
Value:   1000


pointerTwo

Address: 3000
Value:   1000

Important Observation​

Both pointers store:

Same address

Visual Diagram​

pointerOne ──┐
             │
             ▼

           value
            100

             ▲
             │

pointerTwo ──┘

Example​

#include<iostream>
using namespace std;

int main()
{
    int value = 100;

    int *pointerOne = &value;

    int *pointerTwo = pointerOne;

    cout << *pointerOne << endl;

    cout << *pointerTwo;
}

Expected Output:

100
100

Modify Through Second Pointer​

*pointerTwo = 999;

Memory​

Before:

value = 100

After:

value = 999

Example​

#include<iostream>
using namespace std;

int main()
{
    int value = 100;

    int *pointerOne = &value;

    int *pointerTwo = pointerOne;

    *pointerTwo = 999;

    cout << value << endl;

    cout << *pointerOne << endl;

    cout << *pointerTwo;
}

Expected Output:

999
999
999

Why?​

All three refer to:

same memory location

Multiple Pointers To Same Variable​

Example:

int marks = 80;

int *p1 = &marks;

int *p2 = &marks;

int *p3 = &marks;

Visual​

p1 ──┐
      │
p2 ──┼──► marks
      │      80
p3 ──┘

Modify Through Any Pointer​

*p2 = 150;

Result:

marks = 150

Call By Value Fundamentals​

Consider:

void update(int value)
{
    value = 500;
}

Program​

#include<iostream>
using namespace std;

void update(int value)
{
    value = 500;
}

int main()
{
    int number = 100;

    update(number);

    cout << number;
}

Expected Output:

100

Why Did It Not Change?​

Many beginners expect:

500

What Actually Happens​

When function is called:

update(number);

compiler creates:

COPY

Memory​

Before:

number

Address: 1000
Value: 100

Function receives:

value

Address: 2000
Value: 100

Visual​

number
100

↓

copy

value
100

Modify Copy​

value = 500;

Now:

number = 100

value = 500

After Function Ends​

value destroyed

Only:

number = 100

remains.

Call By Value Summary​

Function receives:

Copy of value

Therefore:

Changes do not affect original variable.

Common Bugs​

Bug 1​

int *ptr = nullptr;

cout << *ptr;

Reason:

Dereferencing null pointer

Bug 2​

int *ptr;

*ptr = 100;

Reason:

Uninitialized pointer

Bug 3​

int *ptr = nullptr;

(*ptr)++;

Reason:

No valid memory exists

Interview Questions​

Q1​

What does:

*ptr

mean?

Answer​

Value stored at the address held by pointer.

Q2​

Difference between:

ptr

and

*ptr

Answer​

ExpressionMeaning
ptrAddress
*ptrValue at address

Q3​

Why does:

*ptr = 500;

change original variable?

Answer​

Because pointer accesses original memory location.

Q4​

Difference between copying variable and copying pointer?

Answer​

Variable copy:

Copies value

Pointer copy:

Copies address

Q5​

Why does call by value not modify original variable?

Answer​

Function receives a separate copy.

Cheat Sheet​

int value = 100;

int *ptr = &value;

cout << ptr;      // address

cout << *ptr;     // value

*ptr = 500;

(*ptr)++;

int *ptr2 = ptr;

*ptr2 = 999;


int a = 100;

int b = a;        // value copy

Key Takeaways​

  • ptr stores an address.
  • *ptr accesses the value at that address.
  • Reading through a pointer is called dereferencing.
  • Writing through a pointer modifies original memory.
  • Variable copying creates independent memory.
  • Pointer copying shares memory access.
  • Multiple pointers can point to the same variable.
  • Call by value creates copies.
  • Changes to copies do not affect originals.
  • Understanding ptr vs *ptr is the foundation of all advanced pointer topics.

Now we answer one of the most important questions:

What happens when we add or subtract numbers from pointers?

Many beginners think:

ptr + 1

means:

Address + 1 byte

This is WRONG.

Understanding pointer arithmetic is one of the most important steps toward mastering:

  • Arrays
  • Dynamic Memory
  • STL Iterators
  • Memory Traversal
  • Low-level Programming

Why Does Pointer Arithmetic Exist?​

Consider:

int marks[5] = {10,20,30,40,50};

Memory:

Address      Value

1000         10
1004         20
1008         30
1012         40
1016         50

Question:

How can we move from one element to the next?

Pointer arithmetic solves this problem.

First Example​

int value = 100;

int *ptr = &value;

Memory:

Address: 1000

Value: 100

Suppose:

ptr = ptr + 1;

What should happen?

Common Beginner Assumption​

Many think:

1000 + 1

↓

1001

Wrong.

Actual Rule​

When adding:

ptr + n

compiler performs:

Current Address

+

(n × sizeof(data_type))

General Formula​

New Address

=

Current Address

+

(Number × Size Of Data Type)

Example:

int value = 100;

int *ptr = &value;

Assume:

ptr = 1000

Example​

#include<iostream>
using namespace std;

int main()
{
    int number = 100;

    int *ptr = &number;

    cout << ptr + 2;
}

Different addresses

Example:

1000

1008

Actual addresses vary.

Why Not Add One Byte?​

Because compiler knows:

ptr

is:

int*

Meaning:

Pointer to integer

Moving by one means:

Move to next integer

not:

Move to next byte

Character Pointer Arithmetic​

Consider:

char grade = 'A';

char *ptr = &grade;

Memory Layout​

Address

1000
1001
1002
1003

because:

sizeof(char) = 1

ptr + 1​

Calculation:

1000 + (1 × 1)

=

1001

Example​

char grade = 'A';

char *ptr = &grade;

cout << ptr << endl;

cout << ptr + 1;

Addresses differ by:

1 byte

Syntax​

ptr++;

Meaning​

ptr = ptr + 1;

Example​

int value = 100;

int *ptr = &value;

ptr++;

Equivalent to:

ptr = ptr + 1;

Visual​

After:

ptr

Syntax​

ptr--;

Equivalent:

ptr = ptr - 1;

Example​

Assume:

ptr = 1008

Operation:

ptr--;

Result:

1004

Example​

ptr - 1

For integer pointer:

1000 - 4

996

Visual​

Very Important Difference​

Many beginners confuse:

(*ptr)++

and

ptr++

These are completely different.

Case 1​

(*ptr)++;

Changes:

Value stored at memory

Example​

int number = 100;

int *ptr = &number;

(*ptr)++;

Memory:

Before:

100

After:

101

Program​

#include<iostream>
using namespace std;

int main()
{
    int number = 100;

    int *ptr = &number;

    (*ptr)++;

    cout << number;
}

Expected Output:

101

Case 2​

ptr++;

Changes:

Pointer address

NOT:

Stored value

Example​

int number = 100;

int *ptr = &number;

ptr++;

Result:

Pointer moves

Value remains unchanged

Side-by-Side Comparison​

ExpressionChanges
(*ptr)++Value
ptr++Address
++(*ptr)Value
++ptrAddress

Result:

Pointer Addition​

Valid:

ptr + 1

ptr + 2

ptr + 5

Example​

int *ptr;

ptr + 5;

Meaning:

Move 5 integers ahead

Pointer Subtraction​

Valid:

ptr - 1

ptr - 3

Example​

int *ptr;

ptr - 2;

Meaning:

Move 2 integers backward

Invalid Operation​

Wrong:

ptr * 2;

Why?​

Pointers support:

Addition

Subtraction

but not:

Multiplication

Division

Invalid Example​

ptr / 2;

Compilation Error.

Why Compiler Disallows It​

Multiplication and division have:

No meaningful memory interpretation

Pointer Difference​

Suppose:

ptr1 = 1008

ptr2 = 1000

Operation​

ptr1 - ptr2

Result:

2

not:

8

Why?​

Compiler calculates:

(Address Difference)

/

sizeof(data_type)

Example​

int arr[5];

int *start = &arr[0];

int *end = &arr[3];

cout << end - start;

Expected Output:

3

Why Useful?​

Used heavily in:

  • Arrays
  • STL Iterators
  • Algorithms

Common Bugs​

Bug 1​

int value = 100;

int *ptr = &value;

ptr++;

cout << *ptr;

Problem:

Pointer now points somewhere else

Undefined behavior.

Bug 2​

int *ptr = nullptr;

ptr++;

cout << *ptr;

Invalid memory access.

Bug 3​

(*ptr)++;

ptr++;

Thinking both do same thing.

They do not.

Interview Questions​

Q1​

What does:

ptr + 1

mean?

Answer​

Move pointer to next object of its data type.

Q2​

Why does:

int*

move 4 bytes?

Answer​

Because:

sizeof(int)=4

Q3​

Difference between:

ptr++

and

(*ptr)++

Answer​

ExpressionChanges
ptr++Address
(*ptr)++Value

Q4​

Can pointers be multiplied?

Answer​

No.

Only addition and subtraction are meaningful.

Q5​

Why does:

end - start

return element count instead of byte count?

Answer​

Compiler divides by:

sizeof(data_type)

Cheat Sheet​

int *ptr;

ptr + 1;

ptr - 1;

ptr++;

ptr--;

(*ptr)++;

++(*ptr);

ptr1 - ptr2;

Key Takeaways​

  • Pointer arithmetic works in units of data types.
  • ptr+1 does NOT mean one byte.
  • Integer pointers usually move 4 bytes.
  • Double pointers usually move 8 bytes.
  • Character pointers move 1 byte.
  • ptr++ changes address.
  • (*ptr)++ changes value.
  • Pointer subtraction returns element distance.
  • Multiplication and division are invalid.
  • Pointer arithmetic is the foundation of arrays and iterators.

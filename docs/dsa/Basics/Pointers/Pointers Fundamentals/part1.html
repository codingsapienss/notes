C++ Pointers Fundamentals

Memory, Addresses, Symbol Table, Pointer Fundamentals​

Before learning pointers, you must first understand how variables actually exist inside memory.

Most beginners memorize:

int num = 5;

without understanding:

  • Where is num stored?
  • What exactly is a memory address?
  • Why do pointers exist?
  • Why can't we simply store addresses in normal variables?
  • What is the compiler actually doing?

This chapter answers those questions from first principles.

In this chapter we will cover:

  2. RAM basics

  4. Stack memory

What Is Memory?​

Intuition​

When a program runs, it needs space to store:

  • numbers
  • strings
  • objects
  • arrays
  • functions
  • temporary values

That storage space is called:

Memory (RAM)

Real World Analogy​

Imagine a huge apartment building.

Building
│
├── Room 100
├── Room 101
├── Room 102
├── Room 103
└── Room 104

Each room:

Has an address
Stores something

Memory works similarly.

Memory
│
├── Address 1000
├── Address 1001
├── Address 1002
├── Address 1003
└── Address 1004

Each location:

Has an address
Stores data

Memory Is Just Bytes​

Computers store information in bytes.

1 Byte = 8 Bits

Memory looks like:

Address      Content

1000         ?
1001         ?
1002         ?
1003         ?
1004         ?

Variables Inside Memory​

Consider:

int age = 25;

Many beginners imagine:

age = 25

But internally:

Address      Value

1000         25
1001
1002
1003

because:

sizeof(int)

typically equals:

4 bytes

Important Observation​

Memory stores:

25

Memory does NOT store:

age

This is extremely important.

Then Where Does Variable Name Exist?​

The variable name:

age

exists primarily for:

Programmer convenience

The CPU does not execute:

age

The CPU executes:

memory addresses

Symbol Table​

Definition​

A symbol table is a compiler-maintained structure that maps:

Variable Name

↓

Memory Location

Example​

Code:

int age = 25;
int salary = 50000;

Conceptually:

Symbol Table

age     → 0x1000

salary  → 0x2000

Why Symbol Table Exists​

Without symbol tables:

cout << age;

would be impossible.

Compiler needs a way to know:

Which memory location belongs to age?

Important Note​

The symbol table is mostly a:

Compile-time concept

At runtime:

CPU works with addresses

not variable names.

Stack Memory​

Definition​

Local variables are generally stored in:

Stack Memory

Example​

int marks = 95;
double cgpa = 8.4;
char grade = 'A';

Possible memory:

Stack

Address        Value

0x1000         95
0x1004         8.4
0x1012         A

Visual Representation​

Stack Memory

┌──────────┐
│ grade    │
├──────────┤
│ cgpa     │
├──────────┤
│ marks    │
└──────────┘

Memory Addresses​

Definition​

Every memory location has a unique address.

Think of it like:

House Number

for memory.

Example​

int number = 100;

Possible memory:

Address         Value

0x7fffdd3ef8c4  100

Why Hexadecimal?​

Memory addresses are usually displayed in:

Hexadecimal

Example:

0x7fffdd3ef8c4

Why Not Decimal?​

Decimal:

140734622234564

Hexadecimal:

0x7fffdd3ef8c4

Much shorter and easier.

Address-of Operator (&)​

Definition​

The address-of operator returns the memory address of a variable.

Symbol:

&

Syntax​

&variableName

Example​

#include<iostream>
using namespace std;

int main()
{
    int number = 100;

    cout << &number;
}

Expected Output​

0x7fffdd3ef8c4

Actual address varies every run.

Code Explanation​

&number

means:

Give me the address where number is stored.

Example With Multiple Variables​

int age = 25;
double salary = 50000.5;
char grade = 'A';

cout << &age << endl;
cout << &salary << endl;
cout << &grade << endl;

Expected Output:

Different addresses

Example:

0x7fff1000
0x7fff1008
0x7fff1016

Why Pointers Exist?​

Suppose:

int number = 100;

Address:

0x7fffdd3ef8c4

Question:

Can we store this address?

Yes.

Naive Approach​

long long address = 0x7fffdd3ef8c4;

Why is this bad?

Because:

Address is not just a number.

It represents:

Location of a specific data type.

Problem With Storing Addresses In Normal Variables​

Consider:

int number = 100;
double price = 19.99;

Addresses:

0x1000
0x2000

If we store both inside:

long long

compiler loses information:

What type exists at that address?

Solution: Pointers​

Pointers store:

Memory addresses

while preserving:

Type information

Definition of Pointer​

A pointer is a variable that stores the address of another variable.

Pointer Declaration​

Syntax​

datatype *pointerName;

Example​

int *ptr;

Read as:

ptr is a pointer to int

Memory Diagram​

int number = 100;

int *ptr = &number;
number

Address: 1000
Value:   100


ptr

Address: 2000
Value:   1000

Visual Representation​

ptr
 │
 │ stores address
 ▼

number
 100

Complete Example​

#include<iostream>
using namespace std;

int main()
{
    int number = 100;

    int *ptr = &number;

    cout << "Address stored in ptr: "
         << ptr << endl;
}

Expected Output:

Address stored in ptr:
0x7fffdd3ef8c4

Understanding The Syntax​

Many students incorrectly think:

int* ptr;

means:

ptr is an integer

Wrong.

Actual meaning:

ptr stores address of an integer.

Different Pointer Types​

Integer Pointer​

int value = 10;

int *ptr = &value;

Double Pointer​

double price = 2.34;

double *ptr = &price;

Character Pointer​

char grade = 'A';

char *ptr = &grade;

Float Pointer​

float marks = 99.5f;

float *ptr = &marks;

Why Different Pointer Types Exist​

Because compiler must know:

What type exists at the target address?

This becomes extremely important later for:

Dereferencing

Pointer Arithmetic

Uninitialized Pointer​

Dangerous Example​

int *ptr;

What Happens?​

Pointer contains:

Garbage address

Example:

0xABCD1234

Why Dangerous?​

Program thinks:

Valid memory exists there

even though it doesn't.

Never Do This​

int *ptr;

cout << ptr;

Output:

Random address

Even Worse​

int *ptr;

cout << *ptr;

Possible result:

Crash
Segmentation Fault
Undefined Behavior

Best Practice​

Always initialize pointers.

Good:

int *ptr = nullptr;

Null Pointer​

Definition​

Null pointer points to:

No valid memory location

Syntax​

int *ptr = nullptr;

Example​

#include<iostream>
using namespace std;

int main()
{
    int *ptr = nullptr;

    cout << ptr;
}

Expected Output:

0

or

(nullptr)

depending on compiler.

Why Null Pointer Is Useful​

Instead of:

Random garbage address

we explicitly say:

This pointer currently points nowhere.

Assign Later​

int value = 5;

int *ptr = nullptr;

ptr = &value;

This is completely valid.

Pointer Size​

A very common interview question.

Example​

int value = 100;

int *intPtr = &value;

double number = 2.34;

double *doublePtr = &number;

Check Sizes​

cout << sizeof(intPtr) << endl;

cout << sizeof(doublePtr);

Expected Output (64-bit system):

8
8

Why Same Size?​

Pointers store:

Addresses

not actual data.

Important Observation​

TypeSize
int4 bytes
double8 bytes
char1 byte
int*8 bytes
double*8 bytes
char*8 bytes

Why?​

Because:

All memory addresses require the same number of bits.

Null Pointer Size​

Example:

int *nullPtr = nullptr;

cout << sizeof(nullPtr);

Expected Output:

8

Why?​

Pointer size depends on:

Architecture

NOT:

Stored value

32-bit vs 64-bit Systems​

32-bit System​

Address size:

4 bytes

Therefore:

sizeof(pointer)

Output:

4

64-bit System​

Address size:

8 bytes

Therefore:

sizeof(pointer)

Output:

8

Summary Table​

ArchitecturePointer Size
32-bit4 Bytes
64-bit8 Bytes

Common Mistakes​

Mistake 1​

int *ptr;

cout << *ptr;

Reason:

Uninitialized pointer

Mistake 2​

int *ptr = nullptr;

cout << *ptr;

Reason:

Null pointer dereference

Mistake 3​

long long address = &number;

Reason:

Loses pointer type information.

Interview Questions​

Q1​

What is a pointer?

Answer​

A pointer is a variable that stores the address of another variable.

Q2​

Why do pointers have data types?

Answer​

Compiler must know:

What type exists at the stored address.

Q3​

Why are addresses displayed in hexadecimal?

Answer​

Hexadecimal is more compact and easier to read than decimal.

Q4​

What is the difference between:

int *ptr;

and

int *ptr = nullptr;
Answer​

First:

Garbage address

Second:

Points nowhere safely

Q5​

Why is size of every pointer same?

Answer​

Pointers store addresses.

Address size depends on architecture, not data type.

Key Takeaways​

  • Variables live in memory.
  • Memory is organized using addresses.
  • Variable names are mostly compiler conveniences.
  • CPU works with addresses, not variable names.
  • Symbol tables map names to memory locations.
  • & returns address of a variable.
  • Pointers store addresses.
  • Pointer types preserve type information.
  • Always initialize pointers.
  • Prefer nullptr over uninitialized pointers.
  • Pointer size depends on architecture, not data type.
  • On modern systems, pointer size is usually 8 bytes.

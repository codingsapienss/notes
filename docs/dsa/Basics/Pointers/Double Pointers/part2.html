Double Pointers and Functions

Up until now, every pointer we have seen had a specific type:

int* ptr;
double* ptr;
char* ptr;

The compiler always knew:

  • What type of data exists at the memory address
  • How many bytes to read during dereferencing
  • How many bytes to move during pointer arithmetic

But what if we want a pointer that can store the address of:

  • an integer
  • a double
  • a character
  • a structure
  • an object

without caring about the actual type?

That is where:

void*

comes into the picture.

Why Do Normal Pointers Need Types?​

Consider:

int number = 10;

int* ptr = &number;

Compiler knows:

Address contains an integer

Therefore:

*ptr

means:

Read 4 bytes

(assuming int = 4 bytes)

Similarly:

double price = 10.5;

double* ptr = &price;

Compiler knows:

Read 8 bytes

Pointer arithmetic:

ptr + 1

also works because compiler knows:

Move 8 bytes

Problem​

Suppose we want a pointer that can point to:

int

today,

double

tomorrow,

and

char

later.

Normal pointers cannot do this.

Example:

int value = 10;

int* ptr = &value;

double price = 4.5;

ptr = &price;

Compilation Error.

Why?​

Because:

int*

can only store:

Address of int

Solution​

Use:

void*

What Is a Void Pointer?​

Definition​

A void pointer is a special pointer that can store the address of any data type.

Syntax​

void* ptr;

Read As​

Pointer to unknown type

or

Generic pointer

Important Clarification​

Many beginners think:

Void pointer stores no type.

This is not entirely accurate.

A better statement:

Compiler does not know the type
stored at the address.

Example​

int number = 10;

void* ptr = &number;

Valid.

Memory:

Address      Value

1000         10

Pointer:

Address      Value

2000         1000

Visual Diagram​

ptr
 │
 ▼

number
  10

Compiler View​

Compiler knows:

Pointer stores an address

Compiler does NOT know:

Address contains an integer

Storing Different Data Types​

One of the biggest advantages of void pointers.

Integer Example​

int number = 10;

void* ptr = &number;

Valid.

Double Example​

double price = 99.5;

void* ptr = &price;

Valid.

Character Example​

char grade = 'A';

void* ptr = &grade;

Valid.

Structure Example​

struct Student
{
    int id;
};

Student s;

void* ptr = &s;

Valid.

Important Observation​

Same pointer type:

void*

can point to:

int
double
char
struct
class
object

Why Was Void Pointer Created?​

Without void pointers:

Every generic function would need multiple versions.

Example:

copyInt()
copyDouble()
copyChar()
copyStudent()

Instead:

copy(void*)

can handle all of them.

Internal Working​

Consider:

int number = 100;

void* ptr = &number;

Memory

1000 → 100

Pointer

2000 → 1000

Question:

Can compiler know what exists at 1000?

Answer:

No

Compiler only knows:

Some data exists there.

Why Dereferencing Fails​

Most important concept of this chapter.

Example​

int number = 100;

void* ptr = &number;

cout << *ptr;

Compilation Error.

Why?​

Compiler asks:

How many bytes should I read?

Possible answers:

1 byte ?
4 bytes ?
8 bytes ?
16 bytes ?

Compiler cannot determine.

Therefore:

*ptr

is illegal.

Error Explanation​

Normal pointer:

int* ptr;

Compiler knows:

Read 4 bytes

Void pointer:

void* ptr;

Compiler knows:

Read how many bytes?

Unknown.

Solution: Type Casting​

Before dereferencing a void pointer, we must tell the compiler:

What data type exists there.

Integer Example​

int number = 100;

void* ptr = &number;

cout << *(int*)ptr;

Output:

100

Execution​

Compiler sees:

(int*)ptr

Now knows:

Read integer

Equivalent

int* temp = (int*)ptr;

cout << *temp;

Double Example​

double price = 99.5;

void* ptr = &price;

cout << *(double*)ptr;

Output:

99.5

Character Example​

char grade = 'A';

void* ptr = &grade;

cout << *(char*)ptr;

Output:

A

Modern C++ Style Cast​

Prefer:

static_cast<int*>(ptr)

instead of:

(int*)ptr

Example

cout << *static_cast<int*>(ptr);

Memory Diagram​

Example:

int value = 500;

Memory:

Address      Value

1000         500

Pointer:

void* ptr = &value;

Memory:

Address      Value

2000         1000

Dereference Flow

static_cast<int*>(ptr)

↓

int*

↓

*ptr

↓

500

Pointer Arithmetic on Void Pointer​

Extremely common interview question.

Example​

void* ptr;

ptr++;

Is This Valid?​

According to standard C++:

No

Why?​

Compiler asks:

Move by how many bytes?

For:

int*

Compiler knows:

4 bytes

For:

double*

Compiler knows:

8 bytes

For:

char*

Compiler knows:

1 byte

For:

void*

Compiler knows:

???

Therefore:

ptr++

is illegal in standard C++.

Comparison Table​

Pointer TypeSize Known?Arithmetic Possible?
int*YesYes
double*YesYes
char*YesYes
void*NoNo

Real-World Applications​

Void pointers are heavily used in:

1. malloc()​

C-style memory allocation:

int* arr = (int*)malloc(10 * sizeof(int));

Return type of:

malloc()

is:

void*

Why?

Because malloc can allocate memory for:

int
double
char
struct
class

2. memcpy()​

memcpy(destination,
       source,
       bytes);

Uses:

void*

internally.

Reason:

Can copy any data type.

3. Generic Libraries​

Low-level libraries frequently use:

void*

for generic interfaces.

4. Operating Systems​

Many OS APIs return:

void*

because actual type depends on usage.

Advantages of Void Pointers​

Generic​

Can point to any data type.

Flexible​

Useful for reusable APIs.

Efficient​

No need for multiple function versions.

Disadvantages of Void Pointers​

No Direct Dereferencing​

Requires casting.

No Type Safety​

Compiler cannot verify correctness.

Easy To Make Mistakes​

Wrong casts can cause bugs.

Dangerous Example​

double price = 99.5;

void* ptr = &price;

cout << *(int*)ptr;

Possible Output:

Garbage Value

or

Undefined Behavior

Why?​

Compiler interprets:

Double memory

as:

Integer memory

Common Bugs​

Bug 1​

void* ptr;

cout << *ptr;

Compilation Error.

Bug 2​

void* ptr = &value;

ptr++;

Not valid in standard C++.

Bug 3​

Wrong cast.

double price = 10.5;

void* ptr = &price;

cout << *(int*)ptr;

Bug 4​

Assuming void pointer stores type information.

It does not.

Comparison: Normal Pointer vs Void Pointer​

Featureint*void*
Stores AddressYesYes
Knows Data TypeYesNo
Direct DereferenceYesNo
Pointer ArithmeticYesNo
Type SafeYesNo
Needs CastingNoYes

Interview Questions​

Q1​

What is a void pointer?

Answer​

A generic pointer capable of storing the address of any data type.

Q2​

Can a void pointer point to an integer?

Answer​

Yes.

int value = 10;

void* ptr = &value;

Q3​

Can you dereference a void pointer directly?

Answer​

No.

Compiler does not know the data type.

Q4​

Why is casting required?

Answer​

Compiler must know:

How many bytes to read.

Q5​

Can pointer arithmetic be performed on void pointers?

Answer​

Not in standard C++.

Q6​

Why does malloc return void*?

Answer​

Because allocated memory can be used for any type.

Cheat Sheet​

void* ptr;

int value = 10;

ptr = &value;

cout << *static_cast<int*>(ptr);

double price = 10.5;

ptr = &price;

cout << *static_cast<double*>(ptr);

// Invalid
// *ptr

// Invalid in standard C++
// ptr++

Key Takeaways​

  • A void pointer is a generic pointer.
  • It can store the address of any data type.
  • Compiler does not know what type exists at the address.
  • Void pointers cannot be dereferenced directly.
  • Casting is required before dereferencing.
  • Pointer arithmetic is not allowed in standard C++.
  • malloc() and memcpy() rely heavily on void pointers.
  • Void pointers trade type safety for flexibility.
  • Wrong casts can lead to undefined behavior.
  • Understanding void pointers is essential for low-level C/C++ programming.

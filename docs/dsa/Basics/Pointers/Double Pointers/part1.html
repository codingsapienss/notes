Double Pointers Fundamentals

Most developers understand:

int value = 10;

int *ptr = &value;

But very few ask:

If a pointer is also a variable,
then does a pointer itself have an address?

The answer is:

YES

And that single idea leads to:

int **ptr;

Double pointers.

This chapter explains double pointers from first principles using memory diagrams, symbol tables, execution traces, and real interview examples.

Recap: Single Pointer​

int value = 5;

int *valuePtr = &value;

Important Observation​

Most beginners stop here.

But look carefully:

value

has an address.

valuePtr

is also a variable.

Therefore:

valuePtr must also have an address.

Address of a Pointer​

Example:

int value = 5;

int *valuePtr = &value;

cout << &value << endl;

cout << &valuePtr << endl;

Why Is This Important?​

Because now we can store:

Address of valuePtr

inside another pointer.

What Is A Double Pointer?​

Definition​

A double pointer is a pointer that stores the address of another pointer.

Syntax​

int **ptr;

Read as:

ptr is a pointer
to a pointer
to an integer

First Example​

int value = 5;

int *valuePtr = &value;

int **doublePtr = &valuePtr;

Symbol Table​

Variable          Value

value             5

valuePtr          Address of value

doublePtr         Address of valuePtr

Memory Layout​

Assume:

value     at 1000
valuePtr  at 2000
doublePtr at 3000

Memory:

2000         1000

3000         2000

Example Program​

#include<iostream>
using namespace std;

int main()
{
    int value = 5;

    int *valuePtr = &value;

    int **doublePtr = &valuePtr;

    cout << value << endl;

    cout << valuePtr << endl;

    cout << *valuePtr << endl;

    cout << doublePtr << endl;

    cout << *doublePtr << endl;

    cout << **doublePtr << endl;
}

Why Do We Need Double Pointers?​

Many developers ask:

Why not just use normal pointers?

Reason 1​

Modify pointers inside functions.

Reason 2​

Dynamic memory management.

Reason 3​

Complex data structures.

Examples:

  • Linked Lists
  • Trees
  • Graphs
  • Dynamic 2D Arrays

Reason 4​

Functions that need to change:

pointer itself

not just:

value pointed to

Understanding Function Behavior​

Consider:

void update(int **p)
{
}

Initial Setup​

int num = 20;

int *numPtr = &num;

int **numPtr2 = &numPtr;

Memory​

num

Address: 1000
Value: 20


numPtr

Address: 2000
Value: 1000


numPtr2

Address: 3000
Value: 2000

Case 1​

p = p + 1​

Function:

void update(int **p)
{
    p = p + 1;
}

What Changes?​

Only:

Local copy

changes.

Before:

p = 3000

After:

p = 3008

(assuming 8-byte pointers)

Outside Function​

numPtr2

remains unchanged.

Why?​

Function receives:

Copy of double pointer

Visual​

Before:

numPtr2 ---> numPtr ---> num

Inside:

p ---> numPtr ---> num

After:

p = p+1;

Only:

p

changes.

Original:

numPtr2

unchanged.

Case 2​

*p = *p + 1​

Function:

void update(int **p)
{
    *p = *p + 1;
}

Evaluation​

*p

is:

numPtr

Operation:

numPtr = numPtr + 1;

What Changes?​

Original pointer:

numPtr

changes.

Memory​

Before:

numPtr = 1000

After:

numPtr = 1004

Effect​

Pointer now points somewhere else.

num Value?​

Still 20

Case 3​

**p = **p + 1​

Function:

void update(int **p)
{
    **p = **p + 1;
}

Evaluation​

**p

is:

num

Operation:

num = num + 1;

Memory​

Before:

num = 20

After:

num = 21

What Changes?​

Only:

Actual value

changes.

Pointers remain same.

Comparison Table​

OperationChanges
p = p+1Local copy of double pointer
*p = *p+1Original pointer
**p = **p+1Original integer value

Triple Pointers​

If pointers have addresses:

Pointer

↓

can be pointed to by

Double Pointer

Then:

Double Pointer

also has an address.

Therefore:

int ***triplePtr;

is possible.

Example​

int value = 5;

int *ptr = &value;

int **doublePtr = &ptr;

int ***triplePtr = &doublePtr;

Diagram​

triplePtr
     │
     ▼

doublePtr
     │
     ▼

ptr
     │
     ▼

value
  5

General Rule​

*

Number of stars:

Number of dereferences required
to reach actual value.

Example​

int ***ptr;

Requires:

***ptr

to reach integer.

Pointer Declaration Style​

You asked:

int *ptr;

vs

int* ptr;

Which is correct?

Answer​

Both are identical.

Example​

int *ptr;
int* ptr;

Compiler treats both same.

Industry Recommendation​

Prefer:

int *ptr;

because:

int *ptr1, *ptr2;

makes pointer nature clearer.

Common Bugs​

Bug 1​

int **ptr;

cout << **ptr;

Uninitialized double pointer.

Bug 2​

int *ptr = nullptr;

int **doublePtr = &ptr;

cout << **doublePtr;

Null pointer dereference.

Bug 3​

Confusing:

*p

and

**p

Interview Questions​

Q1​

What is a double pointer?

Answer​

A pointer that stores the address of another pointer.

Q2​

Why do double pointers exist?

Answer​

To work with pointers indirectly and modify pointers inside functions.

Q3​

Difference between:

p
*p
**p

Answer​

ExpressionMeaning
pAddress of pointer
*pPointer
**pActual value

Q4​

What does:

*p = *p + 1;

change?

Answer​

Original pointer.

Q5​

What does:

**p = **p + 1;

change?

Answer​

Original value.

Cheat Sheet​

int value = 5;

int *ptr = &value;

int **doublePtr = &ptr;

doublePtr;

*doublePtr;

**doublePtr;

p = p + 1;

*p = *p + 1;

**p = **p + 1;

Key Takeaways​

  • Every pointer is itself a variable.
  • Every pointer has its own address.
  • Double pointers store addresses of pointers.
  • p, *p, and **p represent different levels of indirection.
  • Double pointers are essential for modifying pointers inside functions.
  • p = p+1, *p = *p+1, and **p = **p+1 affect completely different memory locations.
  • Triple pointers are possible by extending the same concept.
  • The number of * operators corresponds to the level of dereferencing required.

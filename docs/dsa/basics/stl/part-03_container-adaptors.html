Introduction to Container Adaptors​

What Are Container Adaptors?​

Container adaptors are special containers that modify the behavior of existing containers.

They do not store data independently from scratch.

They provide restricted interfaces.

Examples:

  • Stack
  • Queue
  • Priority Queue

Why Container Adaptors Exist​

Normal containers like vector provide:

  • random access
  • insertion anywhere
  • deletion anywhere

But sometimes we want controlled behavior.

Examples:

Browser History​

Open A
Open B
Open C

Back button:

C
B
A

Requires:

LIFO

Ticket Counter​

Person A
Person B
Person C

First person entering:

Person A

Leaves first.

Requires:

FIFO

Stack​

Intuition​

Stack follows:

LIFO

Last In First Out

Real-world examples:

  • Stack of plates
  • Browser history
  • Undo operation
  • Function calls
  • Backtracking algorithms

Visual Understanding​

Push:

Bottom

10
20
30

Top

Pop:

Bottom

10
20

Top

Removed:

30

Internal Working​

By default:

stack<int>

internally uses:

deque

It can also use:

vector
list

Syntax​

stack<int> st;

Header:

#include<stack>

push()​

Adds element.

st.push(10);

st.push(20);

st.push(30);

Memory:

Bottom

10
20
30

Top

emplace()​

Constructs directly.

st.emplace(40);

top()​

Returns top element.

cout<<st.top();

Output:

40

Element remains in stack.

pop()​

Removes top element.

st.pop();

Important Detail​

Wrong:

cout<<st.pop();

Problem:

pop()

returns nothing.

Correct:

cout<<st.top();

st.pop();

empty()​

Checks whether stack is empty.

if(st.empty())
{
    cout<<"Empty";
}

swap()​

st1.swap(st2);

size()​

cout<<st.size();

Complete Example​

#include<iostream>
#include<stack>

using namespace std;

int main()
{
    stack<int> st;

    st.push(10);

    st.push(20);

    st.push(30);

    cout<<st.top()<<endl;

    st.pop();

    cout<<st.top();
}

Output:

30

20

Code Explanation​

st.push(10)

Adds element.

st.top()

Returns top.

st.pop()

Removes top.

Can We Access First or Middle Elements?​

No.

Wrong:

st[2]

Wrong:

st.begin()

Stack intentionally hides these operations.

Only allowed:

  • top
  • push
  • pop

Queue​

Intuition​

Queue follows:

FIFO

First In First Out

Real-world examples:

  • Ticket queue
  • Printer queue
  • CPU scheduling
  • Task scheduling
  • Message systems

Visual Understanding​

Insertion:

Front

10 20 30

Back

Remove:

Front

20 30

Back

Removed:

Internal Working​

Default implementation:

deque

Syntax​

queue<int> q;

Header:

#include<queue>

push()​

q.push(10);

q.push(20);

q.push(30);

emplace()​

q.emplace(40);

front()​

Returns first element.

cout<<q.front();

Output:

10

back()​

Returns last inserted element.

cout<<q.back();

Output:

40

pop()​

Deletes front element.

q.pop();

Complete Example​

#include<iostream>
#include<queue>

using namespace std;

int main()
{
    queue<int> q;

    q.push(10);

    q.push(20);

    q.push(30);

    cout<<q.front()<<endl;

    q.pop();

    cout<<q.front();
}

Output:

10

20

Priority Queue​

Intuition​

Normal queue:

First come → First serve

Priority queue:

Highest priority first

Example:

Hospital emergency system:

Patient A → Medium

Patient B → Critical

Patient C → Low

Execution:

Patient B

Patient A

Patient C

Internal Working​

Internally uses:

Heap Data Structure

Usually:

Binary Heap

Implemented over:

vector

Max Heap​

Default priority queue:

Largest element remains at top.

Visual Understanding​

Insert:

5
2
8
1
7

Heap:

        8
      /   \
     7     5
    / \
   1   2

Top:

8

Syntax​

priority_queue<int> pq;

Header:

#include<queue>

Operations​

push()​

pq.push(5);

pq.push(2);

pq.push(8);

pq.push(1);

emplace()​

pq.emplace(10);

top()​

cout<<pq.top();

Output:

10

pop()​

pq.pop();

Removes highest priority.

Example​

#include<iostream>
#include<queue>

using namespace std;

int main()
{
    priority_queue<int> pq;

    pq.push(5);

    pq.push(2);

    pq.push(8);

    pq.push(1);

    cout<<pq.top()<<endl;

    pq.pop();

    cout<<pq.top();
}

Output:

8

5

Min Heap​

Normally:

Largest at top

For smallest at top:

Syntax:

priority_queue<
int,
vector<int>,
greater<int>
> pq;

Example​

priority_queue<
int,
vector<int>,
greater<int>
> pq;

pq.push(5);

pq.push(4);

pq.push(2);

cout<<pq.top();

Output:

2

Why push() Takes O(logN)​

Heap height:

logN

Insertion may travel from leaf to root.

Common Bugs​

Bug 1​

Wrong:

cout<<st.pop();

Reason:

pop()

returns nothing.

Bug 2​

Wrong:

cout<<q.front();

q.pop();

cout<<q.front();

Problem:

Queue may become empty.

Fix​

if(!q.empty())
{
    cout<<q.front();
}

Bug 3​

Wrong:

priority_queue<int,
vector<int>,
greater> pq;

Missing datatype.

Correct:

priority_queue<
int,
vector<int>,
greater<int>
> pq;

Best Practices​

  • Use stack for LIFO problems
  • Use queue for FIFO problems
  • Use priority queue when order depends on priority
  • Always check empty()
  • Use emplace() for objects

Interview Questions​

Q1​

Why stack does not allow indexing?

Answer:

Stack intentionally restricts access to preserve LIFO behavior.

Q2​

What is internally used by priority queue?

Answer:

Binary heap.

Q3​

Difference between queue and priority queue?

Answer:

Queue:

FIFO

Priority Queue:

Priority-based

Q4​

Why push in priority queue is O(logN)?

Answer:

Heapify operation may move element through tree height.

Key Takeaways​

  • Stack follows LIFO
  • Queue follows FIFO
  • Priority Queue uses Heap
  • Max Heap is default
  • Min Heap uses greater<int>
  • Stack does not allow indexing
  • Priority Queue insertion takes O(logN)

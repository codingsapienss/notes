Associative Containers​

Intuition​

Till now we studied:

  • Vector
  • List
  • Stack
  • Queue

These containers mainly focus on storing data.

But often we need:

  • Fast searching
  • Unique values
  • Key-value mapping
  • Sorted storage

Associative containers solve these problems.

Main Associative Containers​

ContainerDuplicate AllowedSortedInternal Structure
SetNoYesBalanced Tree
MultisetYesYesBalanced Tree
Unordered SetNoNoHash Table
MapKey uniqueYesBalanced Tree
MultimapKey duplicateYesBalanced Tree
Unordered MapKey uniqueNoHash Table

Set​

Intuition​

Suppose:

Input:

10 20 30 20 40 10

Requirements:

  • Store unique values
  • Keep values sorted

Output:

10 20 30 40

Set solves this automatically.

Definition​

Set stores:

  • Unique elements
  • Sorted elements

Internally uses:

Balanced Binary Search Tree

Usually Red Black Tree

Internal Working​

Insert:

30
10
50
20
40

Tree:

        30
      /    \
    10      50
      \     /
      20   40

Traversal:

10 20 30 40 50

Syntax​

set<int> st;

Header:

#include<set>

Insert​

insert()​

st.insert(10);

st.insert(20);

st.insert(30);

emplace()​

st.emplace(40);

Duplicate Example​

set<int> st;

st.insert(10);

st.insert(20);

st.insert(10);

Stored:

10 20

Duplicate skipped automatically.

Traversal​

for(auto x:st)
{
    cout<<x<<" ";
}

Output:

10 20

find()​

Returns iterator.

Syntax:

auto it=st.find(20);

If exists:

Points to value

If not:

st.end()

Example:

if(it!=st.end())
{
    cout<<"Found";
}

erase()​

Delete element.

st.erase(20);

count()​

Returns:

1 → exists

0 → not exists

Example:

cout<<st.count(20);

lower_bound()​

Returns:

First element >= value

Example:

set<int> st={10,20,30,40};

auto it=st.lower_bound(25);

Result:

30

upper_bound()​

Returns:

First element > value

Example:

set<int> st={10,20,30,40};

auto it=st.upper_bound(30);

Result:

40

Complexity​

OperationComplexity
InsertO(logN)
EraseO(logN)
FindO(logN)
CountO(logN)
Lower BoundO(logN)
Upper BoundO(logN)

Multiset​

Definition​

Multiset stores:

  • Sorted values
  • Duplicate values allowed

Internal structure:

Red Black Tree

Example​

multiset<int> ms;

ms.insert(10);

ms.insert(10);

ms.insert(20);

Stored:

10 10 20

Erase Problem​

ms.erase(10);

Deletes:

ALL 10 values

Delete Single Occurrence​

ms.erase(ms.find(10));

Only first occurrence removed.

Example​

multiset<int> ms;

ms.insert(10);

ms.insert(10);

ms.insert(20);

ms.erase(ms.find(10));

Output:

10 20

Unordered Set​

Definition​

Stores:

  • Unique values
  • No ordering

Internally uses:

Hash Table

Internal Working​

Hash function:

Value → Hash Function → Bucket

Example:

12 → bucket 2

25 → bucket 5

31 → bucket 1

Storage:

Bucket0

Bucket1 → 31

Bucket2 → 12

Bucket5 → 25

Syntax​

unordered_set<int> st;

Header:

#include<unordered_set>

Example​

unordered_set<int> st;

st.insert(20);

st.insert(5);

st.insert(100);

for(auto x:st)
{
    cout<<x<<" ";
}

Output:

Random order

Complexity​

OperationAverageWorst
InsertO(1)O(N)
SearchO(1)O(N)
EraseO(1)O(N)

Why Worst Case O(N)?​

Suppose:

Hash function maps everything:

10 → bucket1
20 → bucket1
30 → bucket1
40 → bucket1

Collision:

bucket1

10 → 20 → 30 → 40

Search becomes:

Linear Search

Complexity:

O(N)

Map​

Intuition​

Suppose:

Roll Number → Marks

101 → 90
102 → 87
103 → 76

Map stores:

key → value

Definition​

Map stores:

  • Unique keys
  • Values can repeat
  • Sorted keys

Internally:

Red Black Tree

Syntax​

map<int,int> mpp;

Different Declarations​

map<int,int> mpp;

map<int,pair<int,int>> mpp;

map<pair<int,int>,int> mpp;

Insert​

Using indexing:

mpp[1]=2;

Stored:

1 → 2

Using insert:

mpp.insert({2,4});

Using emplace:

mpp.emplace(3,5);

Example​

#include<iostream>
#include<map>

using namespace std;

int main()
{
    map<int,int> mpp;

    mpp[1]=100;

    mpp[2]=200;

    for(auto it:mpp)
    {
        cout<<it.first<<" "
            <<it.second<<endl;
    }
}

Output:

1 100
2 200

Important Hidden Behavior​

cout<<mpp[100];

If key doesn't exist:

Map inserts:

100 → 0

This surprises many developers.

Safe Method​

auto it=mpp.find(100);

if(it!=mpp.end())
{
    cout<<it->second;
}

find()​

auto it=mpp.find(2);

Returns iterator.

Complexity​

OperationComplexity
InsertO(logN)
FindO(logN)
DeleteO(logN)

Multimap​

Definition​

Stores:

  • Duplicate keys allowed
  • Sorted keys

Example​

multimap<int,string> mp;

mp.insert({1,"A"});

mp.insert({1,"B"});

Stored:

1 A

1 B

Important Limitation​

Wrong:

mp[1]="ABC";

Not allowed.

Because duplicate keys exist.

Unordered Map​

Definition​

Stores:

  • Unique keys
  • Random order

Internal structure:

Hash Table

Syntax​

unordered_map<int,int> mp;

Example​

unordered_map<int,int> mp;

mp[10]=100;

mp[20]=200;

for(auto x:mp)
{
    cout<<x.first<<" "
        <<x.second<<endl;
}

Complexity​

OperationAverageWorst
InsertO(1)O(N)
SearchO(1)O(N)
DeleteO(1)O(N)

When To Use Which​

SituationContainer
Unique + SortedSet
Duplicate + SortedMultiset
Unique + Fast SearchUnordered Set
Key Value + SortedMap
Duplicate KeysMultimap
Fast Key Value AccessUnordered Map

Real-world Examples​

Set​

Used in:

  • Unique usernames
  • Distinct IDs
  • Duplicate removal

Multiset​

Used in:

  • Frequency systems
  • Duplicate rankings
  • Event counting

Map​

Used in:

  • Student databases
  • Dictionary systems
  • Caching

Unordered Map​

Used in:

  • Frequency counting
  • Fast lookup systems
  • Hashing problems

Common Bugs​

Bug 1​

Wrong:

cout<<mpp[100];

Problem:

Creates new key.

Bug 2​

Wrong:

auto it=st.find(50);

cout<<*it;

Problem:

May equal:

st.end()

Fix:

if(it!=st.end())
{
    cout<<*it;
}

Bug 3​

Wrong:

unordered_set<int> st;

st.lower_bound(10);

Error.

Reason:

Hash table does not maintain order.

Best Practices​

  • Use unordered containers when sorting unnecessary
  • Use set/map when ordering required
  • Avoid using map[] only for searching
  • Prefer find()
  • Use emplace() when possible

Interview Questions​

Q1​

Why set stores sorted values?

Answer:

Because internally it uses balanced BST.

Q2​

Why unordered_map is faster?

Answer:

Uses hash table.

Q3​

Why worst case O(N) in unordered_map?

Answer:

Hash collisions.

Q4​

Difference between map and unordered_map?

Answer:

MapUnordered Map
SortedRandom
O(logN)O(1) average
TreeHash Table

Cheat Sheet​

set<int> st;

st.insert(10);

st.find(10);

st.erase(10);

st.lower_bound(20);


multiset<int> ms;

ms.erase(ms.find(10));


unordered_set<int> us;


map<int,int> mp;

mp[1]=10;

mp.find(1);


multimap<int,int> mm;


unordered_map<int,int> ump;

Key Takeaways​

  • Set stores unique sorted values
  • Multiset allows duplicates
  • Unordered set uses hashing
  • Map stores key-value pairs
  • Multimap allows duplicate keys
  • Unordered map is generally fastest
  • Hash collisions cause O(N)

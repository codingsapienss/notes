Double Pointers Memory Layout

This chapter covers one of the most confusing topics in C++:

const int* ptr;
int* const ptr;
const int* const ptr;

Many developers memorize these declarations without understanding:

  • What is actually constant?
  • The pointer?
  • The value?
  • Both?

Understanding const pointers is extremely important because:

  • They appear heavily in production code
  • They are used extensively in APIs
  • STL uses them internally
  • Modern C++ encourages const correctness
  • They are a favorite interview topic

Why Does Const Exist?​

Consider:

int value = 10;

value = 20;

Perfectly valid.

Sometimes we want:

Value should not be modified.

Example:

const int value = 10;

Now:

value = 20;

Compilation Error.

Why Is This Useful?​

Prevents accidental modification.

Examples:

  • Configuration values
  • Constants
  • API parameters
  • Read-only data

Case 2: Pointer to Const​

Mutable Pointer + Constant Value​

Declaration

const int* ptr;

or

int const* ptr;

Both are identical.

Read It As​

Pointer to Constant Integer

Example

int value = 10;

const int* ptr = &value;

What Is Restricted?​

Value cannot be modified through pointer.

Illegal

*ptr = 20;

Compilation Error.

Why?​

Compiler protects:

Value through pointer

What Is Allowed?​

Pointer can move.

Valid

ptr++;

Valid

ptr = nullptr;

Visual​

Pointer  -> Can Change

Value    -> Cannot Change Through Pointer

Memory Diagram​

 │
 ▼

value
 10

Pointer moves.

Value protected.

Important Interview Trap​

Consider:

int value = 10;

const int* ptr = &value;

Illegal:

*ptr = 50;

But:

value = 50;

is valid.

Why?​

Because:

Object is not constant.

Only:

Pointer view is constant.

Example​

int value = 10;

const int* ptr = &value;

value = 50;

cout << *ptr;

Output:

50

Visual​

value

▲
│

ptr

Pointer cannot modify.

Variable itself can.

Alternative Syntax​

These are identical:

const int* ptr;
int const* ptr;

Compiler treats both same.

Case 3: Const Pointer​

Constant Pointer + Mutable Value​

Declaration

int* const ptr = &value;

Read It As​

Constant Pointer
to Integer

Question:

What becomes constant?

Answer:

Pointer itself.

Example​

int value = 10;

int* const ptr = &value;

What Is Allowed?​

*ptr = 50;

Valid.

Value changes.

What Is NOT Allowed?​

ptr++;

Compilation Error.

ptr = nullptr;

Compilation Error.

Why?​

Pointer address cannot change.

Visual​

Pointer -> Fixed

Value -> Can Change

Memory Diagram​

ptr
 │
 ▼

value
 10

Pointer locked.

Value editable.

Example​

int value = 10;

int* const ptr = &value;

*ptr = 500;

Output:

value = 500

Example​

ptr = nullptr;

Compilation Error.

Why?​

Address stored inside pointer is constant.

Case 4: Const Pointer To Const Data​

Constant Pointer + Constant Value​

Declaration

const int* const ptr = &value;

Read It As​

Constant Pointer
to Constant Integer

Restrictions​

Cannot modify value.

Cannot modify pointer.

Example​

int value = 10;

const int* const ptr = &value;

Illegal

*ptr = 50;

Illegal

ptr++;

Illegal

ptr = nullptr;

Visual​

Pointer -> Fixed

Value -> Protected

Nothing can change through pointer.

Real-World Use Cases​

Read-Only Function Parameters​

Example:

void print(const int* arr)
{
}

Guarantees:

Array won't be modified.

API Design​

Libraries use:

const char*

frequently.

Example:

strlen(const char* str);

Why?

Function should read.

Not modify.

Common Bugs​

Bug 1​

Confusing:

const int* ptr;

with

int* const ptr;

They are completely different.

Bug 2​

Thinking:

const int* ptr

makes object constant.

Wrong.

Only pointer access is restricted.

Bug 3​

Thinking:

int* const ptr

makes value constant.

Wrong.

Only pointer becomes constant.

Bug 4​

Memorizing declarations.

Instead of understanding:

What is protected?

Interview Questions​

Q1​

Difference between:

const int* ptr;

and

int* const ptr;
Answer​
DeclarationConstant
const int* ptrData
int* const ptrPointer

Q2​

Can a pointer to const move?

Answer​

Yes.

ptr++;

valid.

Q3​

Can a const pointer move?

Answer​

No.

Address is fixed.

Q4​

What does:

const int* const ptr;

mean?

Answer​

Both pointer and data are constant.

Q5​

Which declaration is most restrictive?

Answer​
const int* const ptr;

Key Takeaways​

  • Const correctness protects data from accidental modification.
  • A pointer and the value it points to are separate entities.
  • const int* ptr protects data.
  • int* const ptr protects pointer.
  • const int* const ptr protects both.
  • const int* and int const* are identical.
  • Understanding what is constant is more important than memorizing syntax.
  • Const correctness is heavily used in professional C++ codebases.
  • Most pointer-related interview questions on const are variations of these four cases.

Pointers in Functions, Array Decay, Array Parameters, and Passing Arrays to Functions

This chapter explains one of the most important and frequently misunderstood topics in C++:

Functions + Arrays + Pointers

Most developers know how to write:

int arr[5] = {1,2,3,4,5};

but very few understand:

  • What actually gets passed to a function
  • Why arrays behave differently from normal variables
  • Why sizeof(arr) changes inside functions
  • Why modifying an array inside a function affects the original array
  • Why modifying a pointer parameter often does NOT affect the original pointer
  • What "Array Decay" means

Understanding this chapter is critical before learning:

  • Dynamic Memory Allocation
  • STL Iterators
  • Double Pointers
  • C-style String Manipulation
  • Advanced Pointer Concepts

Revisiting Function Calls​

Before understanding arrays, let's revisit normal variables.

Example​

void update(int value)
{
    value = value + 10;
}

int main()
{
    int number = 5;

    update(number);

    cout << number;
}

Expected Output:

5

Why?​

Because:

update(number);

creates a copy.

Memory​

Before function call:

number

Address: 1000
Value: 5

During Function Call​

value

Address: 2000
Value: 5

Visual​

number
 5

↓

COPY

value
 5

Important Rule​

Function parameters receive:

Copies

unless references are used.

Passing Pointers To Functions​

Consider:

void print(int *ptr)
{
    cout << ptr << endl;
    cout << *ptr << endl;
}

Program​

int value = 5;

int *valuePtr = &value;

print(valuePtr);

Memory Before Function Call​

value

Address: 1000
Value: 5


valuePtr

Address: 2000
Value: 1000

What Gets Passed?​

Many beginners think:

The pointer itself

This is partially incorrect.

What actually gets passed is:

A copy of the pointer

Visual​

Before Call

valuePtr

Address: 2000
Value: 1000

Function Parameter​

void print(int *ptr)

receives:

ptr

Address: 3000
Value: 1000

Important Observation​

Two different pointer variables exist:

valuePtr

and

ptr

But both store:

Same address

value
5

↓

Why Does This Matter?​

Because changes to:

ptr

do NOT necessarily affect:

valuePtr

Example​

void update(int *ptr)
{
    ptr = ptr + 1;
}

Program​

int value = 5;

int *valuePtr = &value;

update(valuePtr);

cout << valuePtr;

Expected Output:

Original address unchanged

Why?​

Many developers expect:

Pointer moved

Wrong.

What Actually Happens?​

Function receives:

Copy of pointer

Memory​

Before:

valuePtr

Value: 1000

Function:

ptr

Value: 1000

Operation

ptr = ptr + 1;

changes:

ptr

only.

Result

valuePtr still contains 1000

Visual​

After:

ptr
   ↓

Important Rule​

Changing:

ptr

changes:

Local copy

only.

Example From Your Code​

void update(int *ptr)
{
    ptr = ptr + 1;

    cout << ptr;
}

Main Function​

int value = 5;

int *valuePtr = &value;

update(valuePtr);

cout << valuePtr;

Expected Output:

Different addresses

Inside:

ptr moved

Outside:

valuePtr unchanged

Why Does Updating Value Work?​

Now consider:

void updateValue(int *ptr)
{
    *ptr = *ptr + 10;
}

Program​

int value = 5;

updateValue(&value);

cout << value;

Expected Output:

15

Why Different?​

Because now:

*ptr

is modified.

Memory​

Before:

Address: 1000

Value: 5

Function​

ptr = 1000

Operation​

*ptr = *ptr + 10;

means:

Go to address 1000

Modify value

Memory After​

Address: 1000

Value: 15

Key Insight​

Changing:

ptr

changes local pointer copy.

Changing:

*ptr

changes original memory.

Passing Arrays To Functions​

Now we move to one of the most important concepts.

Example​

int getSum(int arr[], int size)
{
    int sum = 0;

    for(int i=0;i<size;i++)
    {
        sum += arr[i];
    }

    return sum;
}

Function Call​

int numbers[5] =
{
1,2,3,4,5
};

getSum(numbers,5);

Beginner Assumption​

Many developers think:

Entire array copied

Wrong.

What Actually Happens?​

Only:

Address of first element

is passed.

Extremely Important Concept​

This is called:

Array Decay

Array Decay​

Definition​

When an array is passed to a function:

Array automatically converts
to pointer to first element.

Example​

Function:

int getSum(int arr[], int size)

Compiler internally treats it almost as:

int getSum(int *arr, int size)

These Are Equivalent​

void func(int arr[])
void func(int *arr)

Both mean:

Pointer to first element

Memory Diagram​

Original Array

Address      Value

1000         1
1004         2
1008         3
1012         4
1016         5

Passed To Function

arr

↓

1000

Only address passed.

Why Is This Important?​

Because:

No full array copy occurs.

Performance Benefit​

Suppose:

int arr[1000000];

Size:

4 MB

If Entire Array Were Copied​

Every function call would copy:

4 MB

Huge cost.

Actual Behavior​

Only:

8-byte pointer

passed.

Very Fast​

O(1)

parameter passing.

sizeof Surprise​

Consider:

int numbers[10];

cout << sizeof(numbers);

Expected Output:

40

because:

10 × 4

Now Inside Function​

void func(int arr[])
{
    cout << sizeof(arr);
}

Output​

Usually:

8

on a 64-bit system.

Why?​

Because inside function:

arr

is no longer an array.

It became:

int*

Visual​

Before Function

Array

40 bytes

Inside Function

Pointer

8 bytes

Example​

void func(int arr[])
{
    cout << sizeof(arr);
}

int main()
{
    int numbers[10];

    cout << sizeof(numbers) << endl;

    func(numbers);
}

Expected Output:

40
8

Common Interview Question​

Why?

Answer:

Array decays into pointer
when passed to a function.

Modifying Array Inside Function​

Consider:

void update(int arr[])
{
    arr[0] = 999;
}

Program​

int numbers[5] =
{
1,2,3,4,5
};

update(numbers);

cout << numbers[0];

Expected Output:

999

Why?​

Because:

Function received address
of original array.

Memory​

numbers

↓

1 2 3 4 5

Function

arr

↓

1 2 3 4 5

Same memory.

Passing Part Of An Array​

One of the biggest benefits of array decay.

Example​

int numbers[6] =
{
45,
6,
7,
5,
54,
42
};

Function Call​

getSum(numbers+3,3);

What Does numbers+3 Mean?​

Memory:

Index      Value

0          45
1          6
2          7
3          5
4          54
5          42

numbers+3​

Points to:

Index 3

Visual​

45 6 7 5 54 42
       ↑
       |
numbers+3

Function Receives​

5 54 42

Sum​

5 + 54 + 42

=

101

Expected Output​

101

Why Is This Powerful?​

Allows working with:

Subarrays

without creating copies.

Real World Example​

Algorithms often process:

Part of an array

instead of full array.

Examples:

  • Merge Sort
  • Quick Sort
  • Binary Search
  • Sliding Window
  • Two Pointer Techniques

Common Bugs​

Bug 1​

void func(int arr[])
{
    cout << sizeof(arr);
}

Expecting:

Array size

Wrong.

Gets pointer size.

Bug 2​

void update(int *ptr)
{
    ptr++;
}

Expecting original pointer to move.

Wrong.

Only local copy moves.

Bug 3​

Thinking:

int arr[]

means full array passed.

Wrong.

Pointer passed.

Bug 4​

Forgetting array size parameter.

void func(int arr[])

Array length is lost.

Interview Questions​

Q1​

What gets passed when an array is passed to a function?

Answer​

Address of first element.

Q2​

What is Array Decay?

Answer​

Automatic conversion of array into pointer to first element.

Q3​

Why does:

sizeof(arr)

become 8 inside functions?

Answer​

Because arr becomes an int*.

Q4​

Difference between:

ptr++;

and

(*ptr)++;
Answer​

ptr++ changes pointer.

(*ptr)++ changes value.

Q5​

Why does modifying array inside function affect original array?

Answer​

Function receives address of original array.

Q6​

Why is passing arrays efficient?

Answer​

Only an address is passed, not the entire array.

Cheat Sheet​

void func(int arr[]);

void func(int *arr);

sizeof(arr);

arr[0] = 999;

ptr++;

(*ptr)++;

getSum(arr+3,3);

Key Takeaways​

  • Functions receive copies of parameters.
  • Pointer parameters are also copied.
  • Modifying pointer variable does not affect original pointer.
  • Modifying dereferenced value affects original memory.
  • Arrays decay into pointers when passed to functions.
  • int arr[] and int *arr are equivalent function parameters.
  • sizeof(arr) inside functions gives pointer size.
  • Arrays are not copied during function calls.
  • Passing arrays is efficient because only an address is passed.
  • Subarrays can be passed using pointer arithmetic.
  • Array decay is one of the most important C++ concepts.

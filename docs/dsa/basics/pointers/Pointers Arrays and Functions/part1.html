Arrays and Pointers (Integer Arrays)

This chapter is one of the most important chapters in C++.

Most developers learn:

int arr[5] = {10,20,30,40,50};

and

int *ptr = arr;

but never truly understand:

  • Why array names behave like pointers
  • Why arrays are not pointers
  • Why arr[i] works
  • How compiler computes indexing
  • Why arr = arr + 1 fails
  • Why ptr = ptr + 1 works
  • Why sizeof(arr) and sizeof(ptr) are different
  • What actually gets stored in memory

This chapter builds the foundation required for:

  • Arrays
  • Pointer Arithmetic
  • Dynamic Memory
  • STL Iterators
  • Function Parameters
  • Advanced C++

How Arrays Are Stored In Memory​

Consider:

int numbers[4] = {1,3,4,5};

Memory Allocation​

Each integer typically occupies:

4 bytes

Array size:

4 × 4 = 16 bytes

Memory:

Address      Value

1000         1
1004         3
1008         4
1012         5

Important Observation​

Array elements are stored:

Contiguously

Meaning:

One after another

without gaps.

Visual Representation​

numbers

┌───────┬───────┬───────┬───────┐
│   1   │   3   │   4   │   5   │
└───────┴───────┴───────┴───────┘

1000    1004    1008    1012

What Is Array Name?​

Consider:

int numbers[4] = {1,3,4,5};

Question:

What exactly is "numbers"?

Many beginners answer:

Pointer

This answer is:

Partially Correct

Correct Answer​

Array name is:

NOT a pointer

However:

In most expressions it behaves like a pointer

More specifically:

Array name decays to pointer
to first element.

What Does Array Name Represent?​

numbers

represents:

&numbers[0]

in most situations.

Example​

int numbers[4] = {1,3,4,5};

cout << numbers;

Possible Output:

0x7fff1000

Equivalent​

cout << &numbers[0];

Output:

0x7fff1000

Important Rule​

Most of the time:

numbers

behaves like:

&numbers[0]

Memory Diagram​

numbers

Address      Value

1000         1
1004         3
1008         4
1012         5

Array name:

numbers

evaluates to:

1000

arr vs &arr[0]​

Example:

int arr[4]={1,2,3,4};

cout << arr << endl;

cout << &arr[0];

Expected Output:

Same address value

Example:

0x1000
0x1000

Are They Actually Same?​

No.

This is an extremely important interview concept.

arr Type​

arr

after decay behaves like:

int*

&arr[0] Type​

&arr[0]

type:

int*

So Why Do They Look Same?​

Because both represent:

Address of first element

arr vs &arr​

Many developers incorrectly believe:

arr

and

&arr

are identical.

They are not.

Example​

int arr[4]={1,2,3,4};

cout << arr << endl;

cout << &arr;

Possible Output:

0x1000
0x1000

Looks identical.

Hidden Difference​

arr​

Type:

int*

(after decay)

&arr​

Type:

int (*)[4]

Read as:

Pointer to entire array

Why Does This Matter?​

Because pointer arithmetic behaves differently.

Example​

Assume:

arr = 1000

Array size:

4 integers

arr + 1​

1000 + 4

=

1004

Moves to:

Next integer

&arr + 1​

1000 + 16

=

1016

Moves past entire array.

Visual​

arr

1000
 ↓
[1][2][3][4]



arr+1

1004
 ↓
[2]



&arr+1

1016
 ↓
After whole array

Internal Working of Array Indexing​

Most developers know:

arr[3]

But very few know what compiler actually does.

Example​

int arr[4]={1,3,4,5};

cout << arr[2];

Output:

4

Internal Compiler Conversion​

Compiler converts:

arr[i]

into:

*(arr+i)

Formula​

arr[i]

=

*(arr+i)

Example​

arr[2]

becomes:

*(arr+2)

Memory​

Address      Value

1000         1
1004         3
1008         4
1012         5

Evaluation​

arr+2

Address:

1008

Then:

*(1008)

Value:

4

Result​

4

Example​

cout << arr[2];
cout << *(arr+2);

Expected Output:

4
4

Important Interview Question​

Why does:

*(arr+2)

work?

Because:

arr

decays to:

&arr[0]

and pointer arithmetic becomes possible.

Strange But Valid Syntax​

Most developers don't know this.

Example​

int arr[4]={1,3,4,5};

cout << 2[arr];

Expected Output:

4

Why?​

Compiler converts:

arr[i]

to:

*(arr+i)

Addition is commutative.

Therefore:

*(arr+i)

=

*(i+arr)

Therefore:

i[arr]

works.

Example​

cout << arr[3];
cout << 3[arr];

Expected Output:

5
5

Arrays vs Pointers​

This is one of the most misunderstood topics.

Array​

int arr[10];

Memory:

40 bytes allocated

because:

10 × sizeof(int)

Pointer​

int *ptr=&arr[0];

Memory:

Only pointer variable created

Usually:

8 bytes

on modern systems.

Memory Diagram​

Array:

arr

40 bytes

Pointer:

ptr

8 bytes

contains address

Example​

int arr[10];

int *ptr=&arr[0];

cout << sizeof(arr) << endl;

cout << sizeof(ptr);

Expected Output (64-bit):

40
8

Why Different?​

Array:

Actual storage

Pointer:

Stores address only

Important Difference Table​

FeatureArrayPointer
Own MemoryYesNo
Size FixedYesNo
Can Change TargetNoYes
sizeof()Full Array SizePointer Size
Stores ElementsYesNo
Stores AddressNoYes

Why arr = arr + 1 Fails​

Very common interview question.

Example​

int arr[10];

arr = arr + 1;

Compilation Error.

Why?​

Array name is:

Not a modifiable variable

Important Concept​

Compiler symbol table stores:

arr

↓

Starting address of array

This mapping is fixed.

Visual​

Symbol Table

arr

↓

1000

Compiler does not allow:

Change array identity

Therefore​

arr = arr+1;

fails.

Why ptr = ptr + 1 Works​

Example:

int arr[10];

int *ptr=&arr[0];

Memory

ptr

Address: 5000

Value: 1000

Operation

ptr = ptr+1;

Before

ptr = 1000

After

ptr = 1004

Why Allowed?​

Because:

ptr

is a normal variable.

Normal variables can change values.

Important Distinction​

Array name:

Not assignable

Pointer variable:

Assignable

sizeof Examples​

Example:

int arr[10];

cout << sizeof(arr);

Expected Output:

40

Why?​

Array owns:

10 integers

Example​

cout << sizeof(arr[0]);

Expected Output:

4

Example​

int *ptr=&arr[0];

cout << sizeof(ptr);

Expected Output:

8

Example​

cout << sizeof(*ptr);

Expected Output:

4

Why?​

Because:

*ptr

is:

int

Common Bugs​

Bug 1​

arr = arr+1;

Compilation Error.

Bug 2​

Assuming:

sizeof(arr)

and

sizeof(ptr)

are same.

Wrong.

Bug 3​

Thinking:

arr

is literally a pointer.

Wrong.

Array and pointer are different entities.

Interview Questions​

Q1​

What does array name represent?

Answer​

In most expressions:

arr

decays to:

&arr[0]

Q2​

How does compiler evaluate:

arr[i]
Answer​
*(arr+i)

Q3​

Why does:

i[arr]

work?

Answer​

Because:

*(arr+i)

=

*(i+arr)

Q4​

Difference between:

arr

and

&arr
Answer​

Different types.

arr      -> int*
&arr     -> pointer to entire array

Q5​

Why does:

arr = arr+1;

fail?

Answer​

Array name is not modifiable.

Q6​

Why does:

ptr = ptr+1;

work?

Answer​

Pointer is a normal variable storing an address.

Cheat Sheet​

arr

&arr

&arr[0]

arr[i]

*(arr+i)

i[arr]

sizeof(arr)

sizeof(arr[0])

int *ptr=&arr[0];

sizeof(ptr);

sizeof(*ptr);

ptr = ptr+1;

Key Takeaways​

  • Arrays and pointers are closely related but not identical.
  • Array name usually decays to pointer to first element.
  • arr[i] is internally *(arr+i).
  • i[arr] is valid C++.
  • Arrays own memory; pointers store addresses.
  • sizeof(arr) gives total array size.
  • sizeof(ptr) gives pointer size.
  • arr and &arr are different types.
  • arr = arr+1 is illegal.
  • ptr = ptr+1 is legal because pointers are variables.

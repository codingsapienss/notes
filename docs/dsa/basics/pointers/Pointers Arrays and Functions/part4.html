Advanced Array-Pointer Relationships

At this point, you understand:

  • Arrays
  • Pointers
  • Array Decay
  • Pointer Arithmetic
  • Character Arrays
  • Arrays in Functions

However, there are still several advanced concepts that frequently appear in:

  • Interviews
  • Competitive Programming
  • Legacy C/C++ Codebases
  • System Programming

This chapter focuses on the subtle but extremely important relationship between arrays and pointers.

Arrays Are Not Pointers​

One of the biggest misconceptions in C++.

Incorrect Statement​

Array is a pointer.

Wrong.

Correct Statement​

Array is NOT a pointer.

Array name often behaves like a pointer.

Example​

int arr[5] = {10,20,30,40,50};

int *ptr = arr;

Many developers conclude:

arr and ptr are same.

Wrong.

Why They Look Similar​

Both allow:

arr[2]

and

ptr[2]

Both allow:

*(arr+2)

and

*(ptr+2)

Both produce:

30

Why They Are Different​

Arrays and pointers have different identities.

Memory Diagram​

int arr[5]={10,20,30,40,50};

int *ptr=arr;

Memory:

Array Memory

1000 -> 10
1004 -> 20
1008 -> 30
1012 -> 40
1016 -> 50

Pointer Memory:

ptr

Address: 5000

Value: 1000

Visual​

┌────┬────┬────┬────┬────┐
│10  │20  │30  │40  │50  │
└────┴────┴────┴────┴────┘

Key Observation​

Array:

Owns storage

Pointer:

Stores address

sizeof Comparison​

Example:

int arr[5];

int *ptr = arr;

cout << sizeof(arr) << endl;

cout << sizeof(ptr);

Expected Output (64-bit system):

20
8

Why?​

Array:

5 integers

Memory:

5 × 4 = 20 bytes

Pointer:

Stores only address

Memory:

8 bytes

Assignment Comparison​

Pointer:

ptr = ptr + 1;

Valid.

Array:

arr = arr + 1;

Invalid.

Why?​

Pointer:

Normal variable

can change.

Array Name:

Fixed identity

cannot change.

arr vs &arr​

This is a very common interview question.

Example​

int arr[5]={10,20,30,40,50};

cout << arr << endl;

cout << &arr;

Possible Output:

0x1000

Looks identical.

Important Reality​

Values appear same.

Types are different.

Type Of arr​

In most expressions:

arr

decays to:

int*

&arr

Pointer To Entire Array​

One of the least understood concepts.

Declaration​

int arr[5];

int (*ptr)[5] = &arr;

Understanding The Syntax​

Start from variable name:

ptr

Move right:

Nothing

Move left:

*

means:

Pointer

Move right:

[5]

means:

Array of 5 integers

Final Meaning:

Pointer to an array of 5 integers

Memory Diagram​

ptr

↓

arr

┌────┬────┬────┬────┬────┐
│10  │20  │30  │40  │50  │
└────┴────┴────┴────┴────┘

Accessing Elements​

int arr[5]={10,20,30,40,50};

int (*ptr)[5] = &arr;

cout << (*ptr)[0];

Expected Output:

10

Why?​

*ptr

gives:

arr

Then:

(*ptr)[0]

means:

arr[0]

Pointer To Entire Array vs Pointer To First Element​

Pointer To First Element​

int *ptr = arr;

Type:

int*

Points To:

First Integer

Pointer To Entire Array​

int (*ptr)[5] = &arr;

Type:

int (*)[5]

Points To:

Whole Array

Array Of Pointers​

Another very important concept.

Declaration​

int *arr[3];

Read It Carefully​

Many beginners confuse:

int *arr[3];

with:

int (*arr)[3];

They are completely different.

Rule​

Array notation:

[]

has higher precedence than:

*

Meaning​

int *arr[3];

means:

Array of 3 pointers to integers

Example​

int a=10;
int b=20;
int c=30;

int *arr[3]=
{
    &a,
    &b,
    &c
};

Memory​

arr[0] -> &a

arr[1] -> &b

arr[2] -> &c

Visual​

arr

┌─────────┐
│  &a     │
├─────────┤
│  &b     │
├─────────┤
│  &c     │
└─────────┘

Access Values​

cout << *arr[0];

Output:

10
cout << *arr[1];

Output:

20
cout << *arr[2];

Output:

30

int (*ptr)[3];

Traversing Arrays Using Pointers​

Most developers traverse arrays using indices.

Traditional Method​

int arr[5]={10,20,30,40,50};

for(int i=0;i<5;i++)
{
    cout << arr[i] << " ";
}

Output:

10 20 30 40 50

Pointer Method​

int arr[5]={10,20,30,40,50};

int *ptr = arr;

for(int i=0;i<5;i++)
{
    cout << *(ptr+i) << " ";
}

Output:

10 20 30 40 50

Why Does This Work?​

Compiler converts:

arr[i]

to:

*(arr+i)

Pure Pointer Traversal​

int arr[5]={10,20,30,40,50};

int *ptr=arr;

while(ptr != arr+5)
{
    cout << *ptr << " ";

    ptr++;
}

Output:

10 20 30 40 50

Memory Trace​

Initial:

ptr -> 10

After:

ptr++
ptr -> 20

Then:

ptr -> 30

Then:

ptr -> 40

Then:

ptr -> 50

Common Bugs​

Bug 1​

Confusing:

arr

and

&arr

They may print same value but have different types.

Bug 2​

Confusing:

int *arr[3];

and

int (*arr)[3];

Completely different declarations.

Bug 3​

Assuming arrays are pointers.

Wrong.

Arrays are separate language constructs.

Bug 4​

Forgetting parentheses.

Wrong:

int *ptr[5];

Array of pointers.

Correct:

int (*ptr)[5];

Pointer to array.

Interview Questions​

Q1​

Is an array a pointer?

Answer​

No.

Array name often decays to pointer, but an array is not a pointer.

Q2​

Difference between:

arr

and

&arr
Answer​

Different types.

arr     -> int*
&arr    -> int (*)[N]

Q3​

Difference between:

int *arr[5];

and

int (*arr)[5];
Answer​

Array of pointers vs pointer to array.

Q4​

Why does:

&arr+1

move farther than:

arr+1
Answer​

Because it points to entire array.

Q5​

Can arrays be incremented?

Answer​

No.

arr++

is illegal.

Cheat Sheet​

int arr[5];

arr;

&arr;

arr+1;

&arr+1;

int *ptr = arr;

int (*ptrToArray)[5] = &arr;

int *pointerArray[3];

*(arr+i);

i[arr];

while(ptr != arr+5)
{
    cout << *ptr;
    ptr++;
}

Key Takeaways​

  • Arrays and pointers are closely related but not identical.
  • arr and &arr often print the same address but have different types.
  • arr+1 moves by one element.
  • &arr+1 moves by the size of the entire array.
  • int *arr[3] means array of pointers.
  • int (*ptr)[3] means pointer to array.
  • Arrays cannot be reassigned or incremented.
  • Pointer traversal and index traversal are fundamentally equivalent.
  • Understanding array-pointer relationships is critical before learning double pointers and dynamic memory.
